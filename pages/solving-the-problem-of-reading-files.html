I recorded a class where I teach how to use Mocks, Spies, and Stubs for testing JavaScript applications. The task was to manually create an .CSV file and transform it into an Array. The example was something like `csvString.split("\n")` I noticed a common problem. /2

People using newer Macs and other Linux distros weren't having issues but people using Windows had an additional character '\r\n' and older Macs (up to 9) the '\r' newline terminator in their CSV files. Those characters weren't written by them but by the OS they're using. /3

Modern Operating systems would use different styles of newline and you'd see often the CR (Carriage Return or "\r\n") or LF (Line Feed or "\n") /4

After a bunch of attempts people realized that they'd split those lines using a regular expression /\r?\n/ that could get both symbols '\n' and '\r' or one or another. /5

What a few people noticed is that if you're the one who created the file you'd instead use the EOL function from the native 'os' method from @Nodejs You'd face a lot of surprises when processing files and this one you definitely should pay attention ðŸ¤© /6
