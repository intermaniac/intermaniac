A lot of people do not include/update ReadMe files for their projects - but it is absolutely necessary to add and regularly update the ReadMe for your project! #OpenSource #GitHub Here's why, and steps to create a plausible ReadMe -üßµ

1. A ReadMe file is the 'FIRST' look for your project. - When someone arrives at your project repo, they wish to know more about what the project is about and who is it for. - Explaining in brief - what the project is about is mandatory. ‚úîÔ∏èHere are some examples of good ReadMe https://t.co/Nt54ymgJw1

2. There would be a lot of people who would wish to test your project. They won't be able to install/use/test the project on their own. - Make sure you add installation steps or a guide on how to test the project. - Further, if there are other steps to be followed, mention them! https://t.co/MNxuFopHGR

3. Then, there are people who wish to help you build the project better. - As mentioned, the people who would want to contribute would need installation steps. - Further, please link or mention contributing guidelines in the ReadMe file. https://t.co/fXdCCub3WY

4. Tech stacks are optional but appreciated for developers or contributors to know about the skills they need if they wish to contribute.

5. Add a License. Licenses denote the things that can be changed or used by everyone else around the world. To know which License fits your project the best, you can read this official Doc by GitHub https://t.co/clW2nSZrew

You don't need a long ReadMe. You need one which gives enough information about the project. You are free to play around and find the best layout that works. These are just the basic stuff you can add or take care of.
