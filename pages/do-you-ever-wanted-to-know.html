To be able to trace the lines a piece of code for instance a unit test is executing, coverage tools need to insert so called "probes" into the loaded bytecode of the jvm.

A probe is similar to a log statement which makes it afterwards possible to reconstruct the control flow of the executed unit test.

The simplest approach would be to insert a probe at every statement the code is executing and reconstruct the control flow from the generated log. However as we have to consider the speed of execution usually probes are only inserted at special points of interest.
