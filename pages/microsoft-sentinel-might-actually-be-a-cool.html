1. Multitenancy aspect exists between customers, but not in the subscription. Partially because MS made the RBAC Reader role hard to interpret. When does it allow access to the data plane? When doesn't it? There's really no telling until you test or read the exact permissions.

The answer to this by the way, is Reader can access it. They can't read App Service contents, that requires Write permissions. They can't read Storage Account Container contents, that requires ListKeys. But LAW? Yep!

2. Elasticity. The cloud storage can scale, probably endlessly. The analytics capability are all met with hard and strict limits. Time and time again, everyone will say "Optimize your query!". But no, there's a memory allocation done per query that if exceeded...

it fails the query. There's no finding the "Needle in the haystack" when the memory allocation trips and MS gives you no context as to how or why, or what else you can try. I've used MS's default queries to the same result. Sentinel's/LAW's compute Does Not Scale.

3. Pay Per Use. Also not present, as Microsoft will bill you on duplicate logs coming from their services that are stored in your environment. MS will also duplicate logs if they feel like metadata they inject is important, such as with risk detections in AAD.

4. On-Demand Self Service. Azure Monitor has too many recent service health alerts to make this the case. Not to mention issues listed above, which there is no provisioning that would assist in any of that. Purely dependent on MS to fix their mess.
