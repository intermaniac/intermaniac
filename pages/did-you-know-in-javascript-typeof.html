Did you know? In JavaScript typeof null is "object" and typeof NaN is "number". But wait, null is not an object and NaN stands for Not A Number! Why's that? Let's find out below üßµ https://t.co/HsbMnlEL62

- At first, type tag and value were used in JavaScript for getting the values. - A type tag for the objects was 0. - null was represented as the NULL pointer. - Like most platforms, null had 0 as a type tag in JS too. - Consequently, it returns "object" for the typeof operator.

But what's the reason under the hood? - null is in contrast with undefined. - undefined were used when an object appears. - And null is often pointing to an empty reference for an object.

- At first, Brendan Eich the creator of JS also followed the same paradigm and made it return to "object". - However, the ECMAScript declared null as the primitive value that reflects the absence of any object value!

Now coming to the typeof NaN. Why does it return "number"? - Same as null, NaN is used where the numbers appear. - And it reflects the absence of any number value.

- The only difference is, that NaN was originally implemented as a "Number" object, and null is implemented as a primitive value. https://t.co/pvZNG4Iq69
