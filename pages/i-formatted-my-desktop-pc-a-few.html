I formatted my desktop PC a few days ago and tonight spent some time installing Docker on Ubuntu for the first time in a long while. My experience was... Interesting. Thought it'd be fun to recap the onboarding experience for someone who's a Linux power user. (thread)

BTW: I'm running Ubuntu 22.04 (LTS), nothing fancy. First, I navigated to the Docker website homepage and clicked the massive "Download Docker Desktop" for Linux button. https://t.co/p0tbaz7jgC

This button took me to the Install on Linux (https://t.co/zMBSxbU41h) page of the site, which outlines how to install Docker for Linux. https://t.co/EJXZD2Af1O

After reading the notices at the top, I scanned through the lengthy list of system requirements and realized a few things: 1. I need to check to ensure I've got all the KVM virtualization support enabled by following the KVM virtualization and support instructions.

2. I need QEMU installed (after some Googling, this was no biggie: `sudo apt-get install qemu`) 3. I need the systemd init system (a quick Google to confirm that Ubuntu is using systemd) 4. I need to use the Gnome, KDE, or MATE desktop environments (check!)

5. I need to install the AppIndicator Gnome extension. NOTE: I've used Gnome extensions before and luckily had the proper browser extension installed, but had I been unfamiliar with this, I'd find it very confusing.

6. I need to ensure `/etc/subuid` and `/etc/subgid` exist and are configured properly, according to the Docker file sharing guide. Luckily, it looks like these files existed and were already properly configured, so nothing to do there.

The KVM virtualization stuff was hard. After visiting the Docker KVM instructions and running the `modprobe_kvm` command and getting no output, I did some digging as there are no instructions for how to get this working, and the suggested `modprobe kvm_amd` command errored out.

After substantial Googling, I came across a StackOverflow thread (https://t.co/qBTDHTz9AS) which said that virtualization support might be disabled in my BIOS. So I rebooted my computer and entered the bios, learning that I'm using MSI Click Bios 5.

After Googling how to enable kvm support on MSI Click Bios 5, I stumbled across the official MSI instructions: https://t.co/Jue4KkyWXG

Unfortunately, my BIOS did not have these options, and the official MSI docs were wildly inaccurate, so back to Google I went, eventually stumbling upon this Reddit thread (https://t.co/Wc8WaxLVE9) which saved the day.

One of the commenters explained how to enable SVM support by going into Advanced mode -> Overclocking settings (why is this in overclocking?) -> OC Explore Mode (Expert) (not at all obvious) -> CPU Features -> Enable SVM Mode.

Once I set this, I rebooted back into Ubuntu and ran the recommended `sudo modprobe_kvm` and `sudo modprobe kvm_amd` commands, but this time I didn't get errors (yey!). I then ran `lsmod | grep kvm` and saw some positive reinforcement, as it now looked like I had kvm working.

Next, I set up the kvm permissions as instructed by the Docker tutorial, and those commands worked as expected with no issues. Once I had KVM working, I headed to the Install on Ubuntu page of the docs and downloaded the Debian package at the top of the page. https://t.co/tU050vqIll

Once I had the package downloaded, I followed the next set of instructions on how to set up Docker's package repository (which is weird, as usually, if I'm prompted to do something like this, I wouldn't need to manually download a .deb package). https://t.co/OVEYRCVBXg

Unfortunately, once I had set up the Docker page repository, when I went to run `sudo apt-get update`, I had some errors -- I was getting a 404 from the Docker mirror. Luckily, this StackOverflow thread came to the rescue and after fixing the `docker.list` file I was good! Whew!

Finally, I installed the Debian package I had downloaded and was able to install Docker Desktop and all the required dependencies. Once the install finished, I opened the Docker app and after 3 minutes it started up. (I'm using a Ryzen 5950x CPU with 64G of RAM). https://t.co/NVi8LqOFQN

This entire process took me approximately 1.5 hours to complete, which is quite a bit of time, considering how simple I thought this was going to be.
