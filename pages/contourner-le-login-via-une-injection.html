Détection : Il y a plusieurs façons de découvrir une injection SQL, dans le cas le plus simple un guillemet simple ou double va casser la commande SQL 2/10

C’est le cas ici : l’ajout d’un guillemet donne l’erreur suivante ￼ 3/10 https://t.co/Ws28UIoLB0

Nous pouvons en déduire que la requête envoyée à la base de données est : 4/10 https://t.co/vY5DWjprKM

Mais pourquoi il y a une erreur ? ￼ Le guillemet que nous avons saisi a donné lieu à un nombre impair de guillemets, ce qui a provoqué une erreur de syntaxe SQL 5/10

Maintenant que nous avons compris la portion vulnérable, nous pouvons essayer de contourner le système d’authentification Pour cela, il faudra que la requête envoie « true » quels que soient le nom d’utilisateur et le mot de passe #login #motdepasse #mdp 6/10

Ce cas est possible en injectant les valeurs suivantes : Identifiant : Admin’ or ‘1’=’1 Mot de passe : demo 7/10

La requête sera donc la suivante et retournera bien la valeur « true », ce qui aura pour effet de nous connecter en tant qu’admin 8/10 https://t.co/xLaLJDSByQ
