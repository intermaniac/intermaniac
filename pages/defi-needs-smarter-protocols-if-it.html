DeFi needs smarter protocols if it's going to succeed. The collaboration with the pioneering StarkNet-native oracle @EmpiricNetwork is an important step for building infrastructure for a new generation of #StarkNet protocols! What's new and why? Here's the thread. 1/

Empiric leverages the power of validity rollups to be the leading oracle on StarkNet (live with 20+ price feeds). Empiric Network is completely on-chain, so protocols that use Empiric can access data that's as verifiable, transparent, and secure as smart contracts. 2/

Why are Oracles so important? There's no real-world information in pure smart contracts. To be useful, these smart contracts need to react to the state of the real world - that's where oracles come in. 3/

Oracles are critical infrastructure that can be manipulated, so building a good one is hard. Oracles aren't always verifiable 4/

Some existing oracles sacrificed decentralization and transparency for security. They pull data from unknown sources (not verifiable) and aggregate it (not verifiable) before posting the answer on-chain. That final number can't be verified, smart contracts have to trust it. 5/

So what's new? Empiric Network gives smart contracts access to data that's as verifiable, transparent, and secure as the contracts themselves. 6/

First-party In contrast to intermediaries that obtain data from public APIs, they use first party data sources with proprietary, high-quality market data. Partners sign their data and send it directly on-chain, where anyone can verify it. 7/

Decentralized: It's not just a few sources that they rely on. Their smart contracts get data from many high-quality sources. Empiric doesn't have off-chain infrastructure, so censorship attacks can't bring it down. 8/

Transparent. From origin to destination, their on-chain approach makes all data verifiable. Its SDKs and contracts are all open source and are being audited. 9/

What makes StarkNet and Empiric's collaboration so special? 10/
