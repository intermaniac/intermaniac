Debouncing in #Javascript is a powerful way to improve the performance of your site. It does this by significantly reducing the amount of times a specific action is taken such as an API call, which is performance intensive. Let's break this concept down ðŸ§µ

Let's use the example of a search bar on an e-commerce site. Say you want to use it to search for a particular product. The HTML for this search field may look like this. https://t.co/qbRDzdWCb3

This input element has on onChange event handler which means that every time a user enters in a character to the search field it will execute the searchProduct function.

The searchProduct function may look something like this: Where we extract the new value within the input field and pass it as a parameter to a function (getSearchResults) that makes an API call to a database. https://t.co/Hl55EzviOS

This as it stands presents a problem where if a user were to enter a product name such as "Apple Airpods Max" then our searchProduct function would run a total 17 times. Once for every character inputted. This in turn makes 17 API calls or 17 database trips. A big no-no

No worries, we can optimize this by using debounce logic. We will create a debounce function that takes in our searchFunction as the first parameter and the time delay in ms that we want to wait between each character input as our second input. Something like this: https://t.co/mwfUyCEiID

Let's break down what's going on here. ðŸŸ¢ We have a function that takes in a callback function and a delay. ðŸŸ¢This function declares a variable timeout which will hold the countdown for our callback function to execute.

ðŸŸ¢ We then return another function that is responsible for resetting our timeout variable ðŸŸ¢It then reassigns it a new timeout

Essentially, we can now use this debounce function as our input elements event handler which will call our searchProduct function only and only when there is a delay between inputs that meets the threshold we defined.

The debounce function here is a closure. If you do not know what a closure is you can check out the thread I wrote about them here; https://t.co/qm4MSWWayj
