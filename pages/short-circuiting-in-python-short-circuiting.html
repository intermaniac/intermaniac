short-circuiting basically happens when two or more expressions are joined together with a boolean operator there are two boolean/logical operators in which short-circuiting happens. these are the "and" and the "or" keywords in python

let's take the "and" keyword and start with it to see how it works https://t.co/soHgQEhjxE

As you can see, the output of the above code is "True" because "and" only evaluates to "True" if all the expressions are "True" in this case 2 == 2 is "True", 3 > 1 is "True", lastly, type(1) is equal to type(2) since they are all "int"

since we understand how and works let's take a look at another example and then talk specifically about short-circuiting https://t.co/4I2vnGdxZx

as you can see the output of the above code is "False" and the reason is that 2 == 3 is "False", although the rest of the expressions are all "True". since there is a "False" expression and it "and" the output has to be "False"

Here is where short-circuiting kicks in, python doesn't have to look at each and every expression in our last code above to determine it's "False" immediately after it meets that first "False", it will not check the rest of the expressions.

because even if they are all "True" the output will still be "False", there is already a False so basically that is how short-circuiting works, let's see it in action with the "or" keyword.

"or" keyword is basically kind of the opposite of "and" keyword in an "or" statement, if atleast one of the expressions is "True", even if the rest are "False" the output becomes "True" let's look at an example: https://t.co/evMAnimIP8

As you can see, the above statement evaluates to "True". although both 3 < 1 is "False" and type(1) = type(2.0) is also "False", there is one expression that evaluates to "True" That one "True" is what makes the statement "True"

so short-circuiting in "or" statement happens, when "True" is met, python doesn't have to check the rest of the expressions to evaluate to "True" because even if the rest are "False", that one "True", makes it "True"
