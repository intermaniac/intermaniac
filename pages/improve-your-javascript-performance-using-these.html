1. Reduce activity loop Each statement in a loop, including the for statement, is executed for each iteration of the loop. https://t.co/dAlIBb8PGc

2. Reduce DOM access Accessing the HTML DOM is very slow, compared to other JavaScript statements. If you expect to access a DOM element several times, access it once, and use it as a local variable: https://t.co/sor6zmv63g

3. Prefer async and defer Javascript is parser blocking. It blocks DOM construction when browser encounter the script tag. async and defer are strategies to tell the browser that a script is not render blocking. https://t.co/txmHCZFFOh

3.1 Adding "async" attribute to the script tag. Async attribute makes sure the script does not block the CRP(critical rendering path). It tells the browser to not block the DOM construction. As a result the browser dispatches the fetch request and continues parsing the DOM.

3.2 "defer" attribute to the script tag. defer attribute is similar to async. The difference is, with defer the script will wait to execute until after the document has been parsed. Whereas the async let the script execute in the background while the document is being parsed.

5. Use local variables When we declare a global variable, it remains in memory throughout its lifecycle. However, if we declare variables in a local context, a block scope is applied, and the variables are removed from memory after the block executes. https://t.co/ovVzFDp4UH

6. Defining class methods When using the function constructor to create a class, make sure to move the class methods to the prototype object. The methods inside the constructor, gets created each time an instance of the class is constructed. https://t.co/EXMGivL5Xk

7.2 - Creating a closure is significantly slower than creating an inner function without a closure and much slower that reusing a static function. https://t.co/sR40eDupno
