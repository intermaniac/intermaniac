We first have to understand these three terms; * Iterables. * Iterators. * Iterations. Let's define these one by one.

Iterable is an object on which '__iter__' or '__getitem__' method can be applied e.g. a string or list. In simple terms, an iterable is an object which can be traversed.

Iterator is an object on which the '__next__' method can be applied or in other objects an object that can traverse through a data container or a sequence of data types without storing those values in that exact sequence in memory.

Iteration is quite simply the process of traversing through an object.

Generators are the kind of iterators which can only be traversed once. Generators can be considered the more advanced or expert level features in Python, although they are not overly complicated and can be understood with some minimal effort on your part.

If we write a program like below and run it, it will not generate all 95 values at once and then print them rather it'll generate and print values on the go. This behavior is useful for when we have a very large range. https://t.co/Khbdeb6BkQ

Yield is a keyword in Python that is used to generates values on the go i.e. it creates a generator object. It returns from a function without destroying the states of its local variable and when the function is called, the execution starts from the last yield statement. https://t.co/XzV5lg2Px1

'__next__' gives us the next value in the generator object. In case of a for loop, when the last value is reached in the generator object StopIteration Error is raised and the iteration stops. Returning a generator object is a good practice if we are dealing with large ranges.
