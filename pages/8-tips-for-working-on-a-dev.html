1. Make your code easy to understand. Your team should know by a glance who wrote the code and for what purpose. This can easily be done by writing comments while coding on who wrote the code and for what reason.

2. Document as you code Simple and redundant but so important I stated it twice.

3. Keep the team up to date on what you are working on. This will prevent duplicate code and wasted time in addition to getting feedback from your team. This goes hand in hand with #4.

4. Prioritize your tasks Keep a list of upcoming tasks in order of importance. This allows a smooth transition of work from one project to another and keeps the team organized.

5. Tackle together Stuck on a something or is there a big task. Work as a team to get it done. Update each other every day to determine if there is something you need to TT, Tackle Together.

6. Have a team lead There are many times when team members will have differing opinions, this is where a team lead comes in handy, to make the final call with no if, ands, or buts about it.

7. Review Each Others Code Sometimes this is mandatory but regardless you should always have a second pair of eyes to do some quality control. This can help everyone grow.

8. Pair Programming Two people, one keyboard. This is a great way to develop more stable code. One person checks for errors and quality, the other types the code.
