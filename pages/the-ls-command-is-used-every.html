The ls command is used every day by 99.9% of Linux users 🐧🐧 And yet most people don't utilize its full potential 🤷 Here are 7 ls options you probably didn't know (but you should)👇🧵

-S: sort entries by size With the -S option, ls will display entries sorted by size, largest file first. In case of a tie, the entries are sorted according to their pathname. https://t.co/wQAIczION5

-F: append a classification character to the filename of some categories of file system objects to identify them - \ for directories - @ for soft links - * for executable files - | for FIFO (named pipes) https://t.co/TEbi2pmGpP

-t: sort entries by modification time Display directory entries sorted by modification time (most recent first) instead of sorting them by their name https://t.co/8k3G0x65XU

-r: reverse sort order I find it most useful when associated with the `-t` option to have the least recently modified file displayed first or with the `-S` option to display the smallest file first. https://t.co/3qvtg1pean

-R: recursively list subdirectories By default, the ls command only displays the content of the directories explicitly given on the command line. With the -R option, it will also recursively display the content of the directories encountered. https://t.co/Cf1ymSTebR

-A: display almost all files The famous -a option includes the hidden files in the display. The -A option does the same except excluding the `.` and `..` entries. https://t.co/au1ESoCPM9

-i: display inode number With the -i option, the ls command will display its inode number alongside each path. https://t.co/XC98Z5C2Ca
