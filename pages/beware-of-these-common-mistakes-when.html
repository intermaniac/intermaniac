As beginners of JavaScript Promises, you may make a few common mistakes. This thread is to help you identify them and start using them in the right way.

1. Looping with Promises ‚Üí The most common mistake is handling promises inside a loop(for, forEach, and all other cousin loops). ‚Üí When we handle multiple promises within a loop, we need to be a bit careful.

‚Üí Some promises may take longer to resolve. ‚Üí The rest of the code inside the loop may finish execution early and may cause undesirable behaviours. ‚Üí Example- I want to fetch a few GitHub details of my three dear friends along with mine: https://t.co/h6NfnOEkzH

‚Üí We use the for-loop to loop through the user id array and call the fetchData() method. ‚Üí The fetchData() method returns a promise with a response. So we get the response value using the .then() handler method. ‚Üí The response value is another promise. https://t.co/5khedyxHNj

‚Üí Hence we need to invoke .then() one more time to fetch the intended data. ‚Üí The fetchData() performs an asynchronous operation, and you can not ensure the sequence of output in this case. ‚Üí There are chances we get the output in a different order than the user id passed https://t.co/cF67SUGNpt

‚Üí Now we will change the loop function a bit to use our favourite async/await keywords. ‚Üí In this case, the control waits when it encounters the await keyword. ‚Üí We have an assurance of getting the first user data and then move to the second one, then the next one, and so on https://t.co/Xe8vAobu53

‚Üí But, still, there is a problem! Fetching each of the user details should be an asynchronous activity. ‚Üí Also, these are unrelated promises, and they must run in parallel to produce a result. ‚Üí In the example above, the promise execution is synchronous.

‚Üí To fix that, use the Promise.all([...]) or Promise.allSettled([...]) APIs ‚Üí They both take an array of promises, run them in parallel, and return the result in the same order of the inputs. ‚Üí The total time taken depends on the max time taken by any of the input promises. https://t.co/fbLkpApKb0

Spoiler- Use promise APIs(.all or .allSettled) to handle multiple unrelated promise executions than using loops. https://t.co/vSSE1dkCl1
