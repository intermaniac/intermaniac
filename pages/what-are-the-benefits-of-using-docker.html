1/7 Replicate production environment locally: If you use Docker, you can make sure that your local development environment runs on the identical OS and Python runtime that you use in your production server.

2/7 Use any database without the need for local installation and maintenance: With Docker, you can just run the database in a separate container. It just works out of the box. There are Docker images for all the major databases.

3/7 Get new team members up and running quickly: A new team member only needs to install Docker on their machine. Then they can just download the code and start working on it right away!

4/7 Quickly experiment with dependencies: Need to change a Django version in a project? All you need to do is to rebuild your Docker container. You can even swap out the database entirely if you like. All without the need to install or setup anything locally.
