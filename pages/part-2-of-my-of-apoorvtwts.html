How is Git different from other VCS? Reasons: 1. Snapshots: - Most of other VCS keep a tab about their files in the form of list of changes that are made to a particular file. This Version Control System is also known as Delta based VCS. https://t.co/P9xp0g2rN2

(continuing...) - In the diagram, V1,V2...V5 are the versions of files and delta is the change made to the base file(the original file(version 1) - Other VCS maintain the changes that are made to the base file over the period of time.

(continuing...) - So for file A, delta 1 is change context to file A. And delta 2 is also change context to file A not delta 1. Similarly for other files.

How does Git works? - Git thinks of this data as series of the snapshot for a miniature file system. With respect to Delta VCS picture. For file A, A1 is first change. For third commit, it will remain A1(because not change made) and for fourth commit, A2 will be there. https://t.co/Cy7fSY2uTb

File System(brief explanation): We have Version V1 And V2 V1 V2 -> App -> App -> a.txt -> A1.txt -> b.txt -> b.txt -> c.txt -> c.txt 27 sept 22 28 sept 22 (time reference)

(continuing...) - Change of A1 has been made in Version 2. - Git takes the snapshot(picture of how does file look like and it's content) of Version of V1 and creates a reference to the point of time it has been commited.

(continuing...) - Whenever you make next changes with modification, it takes snapshots and create reference to it.

(second reason) 2. Nearly every operation is local: - Most of the operations performed by git is by reading all the data from the DB that is in the local system instead of making request to remote server(because git contains all the history or versions)

(continuing...) - So even if you are offline you can still perform git operation and commit changes(that can be saved in your local DB). You can upload them when you get your internet back.

3. Integrity: - Integrity means git will be aware of every change that you'll make. - The reason is that everything in GIT is checksummed. - GIT uses SHA-1 mechanism for checksumming. - SHA-1 is 40 character String composed of hexadecimal characters((0-9) and (A-F))

(continuing...) - It is calculated based on the content of file or directory structure. * If you add or remove a file that goes into the directory structure criterion and if you modify a file then it goes into the contents of the file criterion.

* this string is always going to be unique since it is based on directory structure or the contents of file. * GIT stores everything in the database by not the name of file but by these strings that are calculated based on the directory structure or the contents of file.

(fourth reason) 4. It only adds generally: - Whatever you do on git nearly all of it is only going to add some data into the database but it will never remove any data from database. - It's hard for us to make git erase some data.

That's all from the first video of Git(all covered in part 1 and part 2 of my thread).
