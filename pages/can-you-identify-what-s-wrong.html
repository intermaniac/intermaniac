𝗔𝗻𝘀𝘄𝗲𝗿: Whenever we use any outside variable, prop, state or function inside the useEffect hook, we should always include it in the dependencies list of useEffect hook. So when that dependency changes, our useEffect hook will execute.

In the above code inside the ProductDetails component, the getProduct function is not mentioned in the dependencies list like this: useEffect(() => { getProduct(); }, [getProduct]);

And the getProduct function is coming as a prop returned by useCallback hook which is using the 𝗽𝗿𝗼𝗱𝘂𝗰𝘁𝗜𝗱 which itself is a prop. So If we don't include the getProduct in the dependency array, then the code inside useCallback hook will not execute

even if productId is changed later even though we have mentioned the productId as a dependency to the useCallback hook. So whenever we call any function inside useEffect hook and that function is using some outside variable we should always mention the function as a dependency

for useEffect hook. 𝗣𝗦: If there is no outside variable used inside the function, then there is no need to include that function in the dependency list.
