Imagine you've developed a restful API and are looking to deploy it on a server for public access. What options do you have?

First you can deploy it over a normal public facing webserver (IIS, Apache..) and expose. In a cloud you can do the same thing via a Virtual Machine such as an EC2 instance - install Apache or IIS (for Windows), deploy your API and whitelist it for public internet.

This works fine... But you are also picking up the burden of maintaining these instances for software patching, downtime and maintenance. Also you are paying for the infrastructure, even when your API is not serving any requests / is down for sometime.

For the first issue, you can use a PaaS solution such as Elastic Beanstalk, where you don't need to maintain underlying infrastructure but just deploy your code.

But still you're using a fleet of EC2 instances here and still paying for your API idle time... This is where Lambda shines. AWS Lambda is a serveless function as a service offered by AWS. Here you deploy your code as "functions".

What is a function? It is just like your normal function, it takes an input and returns an output. Here the input is an "event" which contains your data and the return is a response with status and data.

The advantage with Lambda is that it is serverless - meaning you don't need to worry about the underlying infrastructure of the service, and you will be charged for only the computation time!

So if your function handled 10 requests for about 10 min in an hour, you will be charged only for the 10 minutes that it executed. How this is possible? Because internally lambda functions work as containers and are boot up only when it is invoked, and shuts down once done.

Due to this, it adds a very little time in total processing time which is called as ‚Äúcold start time‚Äù.

A Lambda function can run for a minimum of 3 seconds and a maximum of 15 minutes after which it times out. If you want to give more compute power to your Lambda, you can do so by adding more RAM, that's all!

Although for this context we are looking to deploy our API with a lambda function, in reality it can be hooked to various supported integrations within the cloud, such when a new user is signed up in a cognito, or a new message is put into an SQS queue and so on.

To invoke a lambda function, you need a trigger and so for a restful API you integrate with AWS API Gateway, which is another serverless API Gateway service. It transforms the request according to the lambda context and invokes the lambda internally.

Lambda supports most of the popular programming languages including Java, C#, Python, Go, NodeJS and so on. You just need to wrap your code with a function wrapper and ensure that you are returning the result in the same format as expected.
