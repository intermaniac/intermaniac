If you use them right, Flex & Grid are the most powerful tools in CSS. But... 98.8% people sucks at choosing between them. Here's how you can choose which one to use (in seconds). A thread ðŸ‘‡:

DISCLAIMER: I'm not going to tell you the exact cases where you can use any specific one out of flex & grid. Instead... I'll try to provide you with a mindset so that you can decide on your own when to use Flex and when to use Grid. So...

Firstly, Flexbox was added to CSS. But it had some limitations, and some things were complex to achieve. And because of that reason... The grid layout model was introduced to make things simpler. But... Does this mean that after the addition of Grid, Flexbox became useless?

Obviously NO! They both have their own specialties as well as their own limitations. That's why they both still exist. So the question is... Should you prefer one of them over the other?

The answer is : NO. If you're trying to decide which one is better and which one to use all the time, You're making a big mistake. The reason is... By doing so, you're limiting the possibilities. So... the first thing is:

Don't limit yourself to using only Flex or only Grid in a specific type of case. The second point is: Flex is a one-dimensional layout model, while Grid is two-dimensional. So does this mean you should use Flex for one-dimensional layouts and Grid for two-dimensional layouts?

Upto certain extent: Yes! But it's not completely true because... You can make Flex a two-dimensional layout by either using "flex-wrap" property or by using nested Flex. Also, you can make Grid a one dimensional layout by specifying only one row or column. In simple words:

Yes, you can create one-dimensional layouts with Flex and two-dimensional layouts with Grid, but also keep in mind the effect you want to achieve. Because... Flex is laid out based on the content, while in the Grid, the content is shaped based on the layout. Sounds confusing?

Simply put... You cannot specify a specific dimension for any track (rows/columns) in Flex. They are sized based on the content inside them. While in the Grid... You can specify the track size explicitly, and then the content inside it will adjust itself accordingly. But...

How will it help you choose between these two? As I said, it all depends on your requirements. So... If you want the content inside each cell to control the track size: Use Flexbox. While if you want the content to adjust inside a particular dimension: Use grid. Also...

Another way to decide is by comparing the no. of lines of code required using both the methods. Whichever has the longer (and complex) code... Try to avoid that one. But you might argue that this way, you'll need to write two pieces of code each time you use either of them.

The answer is: No. Because if you know Flexbox and Grid at their best... You'll be able to conclude which method will require more lines of code without actually writing it. But it can only happen when you know them perfectly and practice them enough. And that's why...
