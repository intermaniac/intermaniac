What is Git? Git is a Version Control System (VCS) for the files. It offers a bunch of things such as: -To revert a selected file -Compare Changes -Allows to see the modifications done in the file -Allows to know the complete details about the issues caused in a file etc.

Version control system and its need: -Version control, also known as source control, is the practice of tracking and managing changes to software code. -Version control systems are software tools that help software teams manage changes to source code over time.

Local VCS: -The first VCS built was a local VCS but soon discarded as it was a complete failure. -Since developers working on a project need to collaborate together and hence a centralized VCS having a centralized server was introduced over this one. https://t.co/RrXmY0hax5

Centralized VCS: -It had a centralized server that allowed different developers to collaborate together on a project that wasn't possible with Local VCS. https://t.co/C7JszQzMOC

But even this centralized VCS had some downsides: -It had single point of failure that means if the system on which the server is maintained went down somehow, whole of the thing would get disrupted. -Hardware failure that leads to the loss of data and files in the storage.

To overcome this, distributed VCS was introduced. Let's have a look at Distributed VCS: -Distributed version control is a form of version control in which the complete codebase, including its full history, is mirrored on every developer's computer. https://t.co/0hCOEHIA5D

Since Git is a form of Distributed VCS, now let's have a look on what led to it's creation. How was Git created? -During the early years of Linux Kernel maintenance, they used to roll out there new updates in the form of patches and archives.

-In 2002 the Linux Community started using a distributed VCS namely Bitkeeper. But soon in 2005 some problem led to the fallout between them and this prompted the Linux Kernel Community to create their own distributed VCS.

Alongside the problem between them, few other problems was also encountered by Linux Community while working with Bitkeeper that are as follows: -Speed. -Complicated design. -Not having a strong support for non-linear development. -Efficiency with large projects was an issue.

-It was not fully distributed and this was the demand. Linux Torvalds wanted all of this thing in a new distributed system and hence Git was created. Now the next question arises is that how is Git different from other VCS? We do have many reasons for that too:

-Snapshots: Git thinks of its data as a series of snapshots unlike other VCS to the file system that is there for project. The following picture illustrates this. https://t.co/6sbPHKw62v

-Nearly every operation is local that aids to speed and efficiency. -Integrity: It means if any sort of modification, addition, deletion etc. to file is made Git knows about it as in Git everything is checksummed. -Git only adds generally which means data won't be lost easily.
