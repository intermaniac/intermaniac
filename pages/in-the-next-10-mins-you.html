Also it may be obvious, but still, there can be only one [[Prototype]]. An object may not inherit from two others. Please note that __proto__ is not the same as the internal [[Prototype]] property. __proto__ just a getter/setter for [[Prototype]].

Now, [[Prototype]] works differently with functions. All functions in javascript comes with a default property attached with it, F.prototype. This default ".prototype" property points to an object with the only property "constructor" which points back to the function itself. https://t.co/Et6xNiUYNt

Remember, a new object can also be created using the constructor function, like new F(). When you use the new keyword to create an object, the new operator uses this F.prototype to set [[Prototype]] for the new object. https://t.co/snPXhPhszy

Note: In the case of Object, we used __proto__ to set the [[Prototype]]. In case of functions, objects created using the new keywords use the F.prototype property of the constructor function to set its [[Prototype]] by default. https://t.co/cpsOEJtVrt

It's needless to say that you can still uses the __proto__ to set the [[Prototype]] but with using the new keyword and the language default support to set the F.prototype to [[Prototype]] it becomes unnecessary. https://t.co/hJNpz5vxNM

F.prototype property is only used when new F() is called. If, after the creation, F.prototype property changes (F.prototype = <another object>), then new objects created by new F will have another object as [[Prototype]], but already existing objects keep the old one. https://t.co/Zn5hNYhCfu

The most important thing about "constructor" is that Javascript itself does not ensure the right "constructor" value. Yes, it exists in the default "prototype" for functions, but that’s all. What happens with it later – is totally on us.

In particular, if we replace the default prototype as a whole, then there will be no "constructor" in it. For instance: https://t.co/0lCHvI8Rmj

So, to keep the right "constructor" we can choose to add/remove properties to the default "prototype" instead of overwriting it as a whole: https://t.co/9jJnzTeoV7

Or, alternatively, recreate the constructor property manually: https://t.co/WDDUZcjvSh

Native prototype The "prototype" property is widely used by the core of Javascript itself. All built-in constructor functions use it. When  a new Object() is called or (object literal is used {...} to create a new object).

The hidden property [[Prototype]] of this newly created object is set to Object.prototype. https://t.co/nTUAzz9FZf

Similar thing happens when you create a new array using the constructor function new Array or simply create a array like this (let arr = []). The newly created arr inherit from Array.prototype which in turn inherit from Object.prototype. https://t.co/hMLZamAuUR

All functions inherit from Function.prototype, which in turn again inherit from Object.prototype. This is the reason you've access to .call, .bind etc for functions, and have access to .map, .reduce etc for arrays. https://t.co/I0skj2HRSM

Below is what happens when you create a custom object using the constructor function(using new F()) https://t.co/P8q5jh3yeF

Here's an example of function inheritance in Javascript. This is what happens behind the scene https://t.co/ubJdtINzIK

Here's the default prototype chain provided by Javascript. Arrays, Functions, Strings, Numbers, and all the built in type inherit from Object.prototype, which in turn inherit from null at the top. https://t.co/R66DRhEu40

To quickly recap: [[Prototype]] - a hidden reference to another object __proto__ - used to set/get [[Prototype]] of an Object prototype - the actual object to which [[Prototype]] references, used for inheritance.

F.prototype - property with function, used to set prototype of objects created using the new keyword Native.prototype - builtin prototype object, widely used by the core of javascript itself
