How to use Elastic File System (EFS) with AWS Lambda? ðŸ§µ ðŸ‘‡ https://t.co/RysdcEskaG

Why do you want to use EFS with Lambda? Let's say you're building serverless video processing application. You might want to share these large files between lambda functions. Some lambda functions will be writing files and other lambda functions will be reading the same files

You can use EFS which is a serverless , elastic file system which grows and shrinks as you add or remove files.

How Elastic File System (EFS) works with Lambda When `Elastic File System` is created, root directory `/` and root user (UID 0) with read, write and execute permissions will be available, by default. Of course, you can create multiple directories in EFS later, as you wish.

What is Access point? Access point is application specific entry point to the elastic file system. It is a directory at elastic file system for which client(lambda in our case) has access to.

What is Mount Path? Mount path is the directory at the client end (lambda in our case) using which you can use the access point at the EFS end. https://t.co/gI8HQFDtQg

POSIX permissions for access point In addition to the path, `access point` also defines the user who has access to the defined path. You can specify the `user id`, `group id` and access permissions in standard octal format.

Security Groups To access EFS from Lambda, the inbound port `2049` should be opened. This can be done through security group for EFS. If you're using `aws cdk` , AWS CDK would create this security group for EFS for you. https://t.co/aaiiW57AYL

Project: As a requirement, Lambda functions which uses Elastic File System should be in VPC. Because of security reasons - we want the lambda functions and elastic file system to be in private subnet of VPC so that no one external to VPC can connect to these services directly https://t.co/2lUW4JcSgJ

Creating VPC to host our EFS and Lambda: We're going to create 2 subnets â€“ a private subnet and a public subnet. In the private subnet, we're going to have lambda and EFS. When you create a subnet of type `PRIVATE_WITH_EGRESS` in AWS CDK, it will also create a NAT Gateway https://t.co/x5cdTqnDRo

Create Elastic File System (EFS ) https://t.co/rmmsgVx9fD

Create Access Point When you create an Elastic File System(EFS), you can have many directories in that file system. However, you want your lambda to access only specific path in that elastic file system. Access point defines that path and the associated permissions for the same. https://t.co/vtMGlpXlc0

Role, Permissions required for AWS Lambda to access EFS We create inline policy for allowing to mount, read and write to the elastic file system. Then, we use that inline policy to the lambda role. https://t.co/C9ROc5UJe8

Lambda function code to write to EFS: Lambda function code to write to EFS is pretty simple. Just define the file path (which should be part of your mount path) and write the file. https://t.co/QQwDHcvlrE
