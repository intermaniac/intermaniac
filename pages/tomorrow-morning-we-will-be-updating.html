Tomorrow morning, we will be updating the testnet. üöß Aleph Node will be updated to version 8.0, containing many important improvements and fixes, necessary for the upcoming Mainnet update (you will find a detailed changelog in the GitHub repo). Learn more below. üßµüëá

To make this update possible, we will need to push all of the external validators out of the committee for the duration of the update, so don't be alarmed if your node will not be validating for a moment: the committee will be restored as soon as the update is complete.

‚ùóIMPORTANT: Once the new version is out, you will need to update your nodes in the window between Thursday, Nov 3rd, 16:00:00 CET and Sunday, Nov 6th, 23:59:59 CET.

Since the update contains logic to suspend inactive validators, it is important you update within the provided window, as otherwise you risk suspension.

The update procedure for validators is the same as in the previous update: a) Validators using Aleph Node Runner will need to stop their containers (`docker stop`) and run the `run_node.sh` script again.

b) Validators using their own binaries will need to checkout the `r-8.0` tag in the `aleph-node` repository and build a new binary.

Note that additional config is required, which will be described in the guides as soon as the new version goes live (https://t.co/EDILVFaIaq). If you encounter any problems during the update, reach out to us on Discord and the team will be there to assist you.
