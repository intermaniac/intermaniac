GIT CONFIG (2/3) ` git config --global alias. <alias-name> <git-command> ` : Create a shortcut for a Git command. E.g. alias.glog ‚Äúlog --graph --oneline‚Äù will set ‚Äùgit glog‚Äù equivalent to ‚Äùgit log --graph --oneline.

GIT CONFIG (3/3) ` git config --global --edit ` : Open the global configuration file in a text editor for manual editing.

GIT BASICS (1/3) ` git init ` : Initialize the current directory as a git repository. ` git clone <repo> ` : Clone repo located at <repo> onto local machine.

GIT BASICS (2/3) ` git add <directory> ` : Stage all changes in <directory> for the next commit. ` git commit -m "<message>" ` : Commit the staged snapshot, but instead of launching a text editor, use <message> as the commit message.

GIT BASICS (3/3) ` git status ` : List which files are staged, unstaged, and untracked. ` git log ` : Display the entire commit history using the default format. ` git diff ` : Show unstaged changes between your index and working directory.

REMOTE REPO (1/2) ` git remote add <name> <url> ` : Create a new connection to a remote repo. After adding a remote, use <name> as a shortcut for <url> ` git fetch <remote> <branch> ` : Fetches a specific branch, from the repo.Leave off <branch> to fetch all remote refs

REMOTE REPO (2/2) ` git pull <remote> ` : Fetch the specified remote‚Äôs copy of current branch and merge it into the local copy. ` git push <remote> <branch> ` : Push the branch to <remote>, along with commits and objects. Creates named branch in remote repo if it doesn‚Äôt exist.

GIT BRANCHES (1/1) ` git branch ` : List all the branches in the repo. Create a new branch named "branch" by adding the "branch" option ` git checkout -b <branch> ` : Create and check out a new branch named <branch> ` git merge <branch>` : Merge <branch> into the current branch

GIT PUSH (1/2) ` git push <remote> ` : Forces the git push even if it results in a non-fast-forward merge. ` git push <remote> --all ` : Push all of your local branches to the specified remote.

GIT PUSH (2/2) ` git push <remote> --tags ` : Tags aren‚Äôt automatically pushed when you push a branch or use the --all flag. The --tags flag sends all of your local tags to the remote repo.

GIT PULL (1/1) ` git pull --rebase <remote> ` : Fetch the remote‚Äôs copy of the current branch and rebases it into the local copy. Uses git rebase instead of the merge to integrate the branches.

GIT REBASE (1/1) ` git rebase -i <base> ` : Interactively rebase current branch onto <base>. Launches the editor to enter GIT REBASE or how each commit will be transferred to the new base.

GIT DIFF (1/1) ` git diff HEAD ` : Show the difference between the working directory and the last commit. ` git diff --cached ` : Show the difference between staged changes and the last commit

GIT RESET (1/2) ` git reset ` : Reset staging area to match most recent commit, but leave the working directory unchanged. ` git reset --hard ` : Reset staging area and working directory to match most recent commit and overwrites all changes in the working directory.

GIT RESET (2/2) ` git reset <commit> ` : Move the current branch tip backward to <commit>, reset the staging area to match. ` git reset --hard <commit> ` : Same as previous, but resets both the staging area & working directory to match.

REWRITING GIT HISTORY (1/2) ` git commit --amend ` : Replace the last commit with the staged changes and the last commit combined. ` git rebase <base> ` : Rebase the current branch onto <base>. <base> can be a commit ID, branch name, a tag, or a relative reference to HEAD.

REWRITING GIT HISTORY (2/2) ` git reflog ` : Show a log of changes to the local repository‚Äôs HEAD. Add --relative-date flag to show date info or --all to show all refs.

UNDOING CHANGES (1/2) ` git revert <commit> ` : Create a new commit that undoes all the changes made in <commit>, then apply it to the current branch. ` git reset <file> ` : Remove <file> from the staging area, but leave the working directory unchanged.

UNDOING CHANGES (2/2) ` git clean -n ` : Shows which files would be removed from the working directory. Use the -f flag in place of the -n flag to execute the clean.
