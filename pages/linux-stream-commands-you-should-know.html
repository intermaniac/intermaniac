Linux stream commands you should know 🧵👇

There are 3 types of stream in Linux - Standard input stream - Standard output stream - Standard Error stream

Input stream - Carries input data from different input sources - keyboard - any file - Represented by '<' operator - "< input.txt" will take input from input.txt file

Output stream - Carries the output data to different output sources - Console/ Terminal - any file - redirect the output to standard input to another command - Represented by '>' operator - '> output.txt' outputs the data to the text file.

Error stream - Carries the error messages due to wrong commands to different output destinations - Console / File - Represented by '2>' operator - 2 is the code number associated with the error stream - '2> error.txt" shows the output to error.txt file.

Pipe operator | - the '|' operator redirects the output as input to another command - less command is used to view large files - 'ls -l | less' will output in less command.

env variables - list of env variables can be seen using 'env' command. - each env can be accessed by 'echo $env-variable'. - Each command has an executable file in certain directories. - So we are actually executing files internally.

Head command - displays by default the first 10 lines of the file - 'head output.txt' - 'head -n 7 output.txt' displays the first 7 lines.

Tail command - Same as head command - 'tail -n 7 output.txt'
