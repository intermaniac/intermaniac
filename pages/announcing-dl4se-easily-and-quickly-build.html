1/5 We continuously mine public GitHub repositories. The source code of the mined repositories is kept in line with its evolution on GitHub. This means that, pending a small time delay, you will always get fresh data as they appear in the projects' latest snapshot on GitHub.

2/5 Besides providing the source code of the mined projects, we also implemented several filters and processing steps frequently adopted in SE studies. For example, you can automatically remove from your dataset test code, unparsable code, or code including non-ASCII characters.

3/5 You can exclude exact and near duplicates from your dataset, ask for an AST-based representation of each code instance, and much more! The output dataset is in JSON format and includes metainformation about the repositories from which each code instance has been collected.

4/5 We provide forms specialized in the creation of different types of datasets (e.g., training DL models for the task of code completion). The forms differ for the type of processing operations available. Use the generic one in case you just want to collect data for an MSR study

5/5 Once you submit your request, an email will tell you when your dataset is available. Currently we only support Java, but multi-language support (Python, Javascript, C/C++/C#) is coming. Implemented and maintained by @ozren_dabic in the https://t.co/HXYM3FO54v group.
