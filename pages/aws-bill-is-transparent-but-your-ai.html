AWS Bill is transparent but your Ai Productive Systems are still too expensive for the foreseeable future? Then this thread is for you. How to reduce Ai Production Cost in 5 Steps: 1) Try using a CPU first:

• CPUs can be a great alternative for running models, especially if latency isn't a major concern. • CPUs are typically cheaper than GPUs and can be a great option if you don't need to run DNNs. 2) Try building your own GPU server:

• Building your own physical GPU server can yield significant savings compared to using cloud providers. (Ooops) • Downside: building your own server is a Capital Expense (CapEx), whereas cloud hosting is an Operating Expense (OpEx).

3) Make sure you're using GPUs as efficiently as you can: • GPUs can execute multiple functions at the same time. Adjust batch sizes to process as many inputs through a model at a time as your hardware allows.

• This will generate a cost savings because you won't be over-provisioning GPU capacity. • Shut down the GPU once you're done. 4) Take advantage of reserved instances:

• Reserved instances allow you to save 40-70% off regular cloud hosting prices if you pay the one year commitment upfront. • This isn't the best option for R&D and prototyping, though, since the estimation of usage isn't always accurate.

• Downside: You can't turn off these instances, which can be risky. 5) Take advantage of spot instances: • Spot instances can save up to 90% costs associated with running batch jobs.

• However, you can't always guarantee you'll get the infrastructure you want at the speed that you want so this is better for batch vs. real-time processing.
