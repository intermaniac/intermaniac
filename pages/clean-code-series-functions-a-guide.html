1⃣ clean code series: functions📒🧑‍💻 ☑️ use default arguments instead of short-circuiting or conditionals https://t.co/rpel3bfowm

2⃣ clean code series: functions📒🧑‍💻 ☑️ function arguments (2 or fewer ideally) 🔹zero arguments are the ideal case. one or two arguments are ok, and three should be avoided. anything more than that should be consolidated. https://t.co/ql17kknqav

3⃣ clean code series: functions📒🧑‍💻 ☑️ function names should say what they do https://t.co/fr3wvce0sq

4⃣ clean code series: functions📒🧑‍💻 ☑️ functions should only be one level of abstraction 🔹when you have more than one level of abstraction your function is usually doing too much. 🔹splitting up functions leads to reusability and easier testing. https://t.co/jxf6qck7vn

5⃣ clean code series: functions📒🧑‍💻 ☑️ don't use flags as function parameters 🔹flags tell your user that this function does more than one thing. 🔹functions should do one thing. 🔹split out your functions if they are following different code paths based on a boolean. https://t.co/pnyadcq0se

6⃣ clean code series: functions📒🧑‍💻 ☑️ avoid side effects 🔹a function produces a side effect if it does anything other than take a value in and return another value or values. try to avoid it. https://t.co/fdfma14lg2

7⃣ clean code series: functions📒🧑‍💻 ☑️ don't write to global functions 🔹polluting globals is a bad practice in many languages because you could clash with another library and the user of your api would be none-the-wiser until they get an exception in production. https://t.co/noqctxiiof

8⃣ clean code series: functions📒🧑‍💻 ☑️ don't use a singleton pattern 🔹singleton is an anti-pattern 🔹they violate the single responsibility principle 🔹they inherently cause code to be tightly coupled ... https://t.co/icqxs6tzqx

9⃣ clean code series: functions📒🧑‍💻 ☑️ encapsulate conditionals https://t.co/9mfb3vnwqx

🔟 clean code series: functions📒🧑‍💻 ☑️ avoid negative conditionals https://t.co/9xgvvua0uf

1⃣1⃣ clean code series: functions📒🧑‍💻 ☑️ avoid conditionals 🔹you can use polymorphism to achieve the same task in many cases. 🔹also, a function should only do one thing. https://t.co/qfnhmn982q

1⃣2⃣ clean code series: functions📒🧑‍💻 ☑️ avoid type-checking (part 1) 🔹consider using consistent apis. https://t.co/nnczdqvz6r

1⃣3⃣ clean code series: functions📒🧑‍💻 ☑️ avoid type-checking (part 2) 🔹if you are working with basic primitive values like strings, integers, and arrays, and you use php 7+ and you can't use polymorphism try considering type declaration or strict mode. https://t.co/msspvjnukb

1⃣4⃣ clean code series: functions📒🧑‍💻 ☑️ remove dead code 🔹dead code is just as bad as duplicate code. there's no reason to keep it in your codebase. 🔹if it's not being called, get rid of it! it will still be safe in your version history if you still need it. https://t.co/0p0g6qgn2i
