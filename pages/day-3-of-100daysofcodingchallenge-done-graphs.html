-NPM stands for Node Package Manager, it's world's largest software registry containing more than 8,00,000+ packages in it. -It puts modules in place so that node can find them, and manages dependency conflicts intelligently. It is used to publish, discover, install, and develop.

Let us try to understand the necessary of npm. -Let us say if we want to build a building, we will buy all necessary material required and build the building according to our requirement, instead of preparing our own cement and stuff. The same applies with node in real time.

-package.json is the main file where all the packages of NPM are defined. -If a project has a package.json file, by running npm install, it will install everything the project needs, in the node_modules folder, creating it if it's not existing already. #nodejs #js

-It mainly specifies the verison of modules, dependencies we are using, easy to share, read, and work with. -Version is a 3 number code which represents the latest change in package. -3.9.12 is where 3 is major change, 9 is minor and 12 is patch change in module. #javascript

-It also contains dependencies and devdependencies. -dependencies are libraries which are used at the execution of our project. -They contains modules which supports our project. -devdependencies contains libraries needed at some point in development process,not during execution.
