Suppose you're building an app and you want to show crypto coins prices and stats. Now, you don't have data to show, right? So, where will you get that data? From Someone who collects such data, right? Those who collect data, give you access to that via APIs(URLs)

To get data you need to make a request to their server using the API provided by them. This request goes through many networks to reach their server. Then take the data and travel back to you. This process is time-consuming obviously, isn't it?

JS is single-threaded. So, making it sit idle and wait for the result of the API call will cause serious performance issues. This is where promises come into the picture. We make a request using fetch(), this fetch returns a promise & we chain .then and .catch block with it

This network request made using fetch is processed by the browser. Meanwhile, JS engine continues doing other tasks. When that promise gets resolved, the browser puts the callback function(either then block or catch block, depending on its succeeded or rejected) in the task Que

Now, when the event loop finds that the js engine is free and not doing any other task then this callback function is taken from the microtask queue and given to the JS engine and which executes it finally. This is the whole picture of how a promise works.
