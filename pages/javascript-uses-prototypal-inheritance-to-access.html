In the beginning Javascript doesn’t have class keyword which was introduced later in ES6 which is basically a syntactic sugar and under the hood uses prototypes to structure our code as object oriented way.

𝗪𝗵𝗮𝘁 𝗶𝘀 𝗽𝗿𝗼𝘁𝗼𝘁𝘆𝗽𝗲 In Javascript, every object has an internal property denoted as [[Prototype]], which is simply a reference to another object which we call the prototype of that object. To test it let’s check it on the browser console. https://t.co/qGVMj5MwWU

As you can see we have an internal property [[prototype]] which has all the common functionality that we use in our code like isPrototypeOf or toString and also it has __proto__ which is linked to the parent built-in Object.prototype

Later in this thread, we will look into how we can link one object’s prototype to another custom-built object.

You might have heard in Javascript everything is an object and an Array is also an Object. Now if you ever wondered where all these methods for Array like length, push, filter, etc are coming from you got your answer, they are all stored in the prototype. https://t.co/LGjT5fcJ9r

𝗟𝗶𝗻𝗸𝗶𝗻𝗴 𝗢𝗯𝗷𝗲𝗰𝘁𝘀 (𝗢𝗯𝗷𝗲𝗰𝘁.𝗰𝗿𝗲𝗮𝘁𝗲()) One of the ways to create a link between objects we have object.create(); which creates an empty object and links(prototype) to another object that is passed in as a parameter. https://t.co/AYbCtxemwG

Let’s explain the above code. In the above code, we are creating newObject which is an empty object with a link to mainObject meaning now newObject’s [[prototype]] has the data and functionality of mainObject. Now if we try to access a property on newObject and it isn’t found

then it will check the prototype chain and find the property there and if let’s say the property is also not present in mainObject then it will check mainObject’s prototype which is link to the parent Object and our chain ends there then in this particular case Javascript creates

the property for us and sets it to undefined that’s why whenever the property is not present on the object we get undefined.

A constructor function is a normal function that is invoked with a new keyword and is used for creating objects. Let’s understand it using the above code.

In the above code, we are using the constructor function which is acting as a factory function meaning creating objects for us. The syntax to add properties to a constructor function is a bit different,

you use this.propertyName = whatever name we pass in at the time of invoking with the new keyword. Every Constructor function has an internal property prototype which is used to store common functionality and it is much more efficient

since we are only creating startEngine() function once in the prototype and it is linking to every instance of Car instead of creating it in the Car constructor which would’ve created for every instance.
