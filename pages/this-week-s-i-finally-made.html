This week’s thread… I finally made the switch to GitHub Copilot. For those that don’t know, this is an Artificial Intelligence system that generates code for programmers. I’ve been on it a few days now, and have some thoughts.

2/ There’s a tendency to go extremist around AI - people usually think it’s going to replace everything or ruin everything. This isn’t true.

3/ If you haven’t seen it, there’s a good video (https://t.co/bhqXfQjZcu) by @cleoabram that goes over what I’m ultimately talking about, so if you haven’t any background on this - I’d suggest you go watch that video first.

4/ So what I’ve found using GitHub Copilot mirrors what Cleo mentioned with art. There’s a certain level of “if you’re needing a bump up on a skill you’re deficient in, this will help”. But the clue is in the name.

5/ Yes, people have let this thing steer itself, but there’s a reason this thing isn’t called “Code captain”… it’s “copilot” and really shines when you can steer it.

6/ Oversimplified, it’s a “Garbage in, garbage out” scenario. But if you spend the time to set up the comments about what you’re really trying to do - then it does up with some solid code. Minus a few things.

7/ I’ve so far only used this with Python, but plan on trying it with Rust in the coming days. Whilst the code is good, it was still missing error handling. So ultimately, yes the code could run and do the job, but was still about 50% of what I need for production level code.

8/ Is it helpful? Hell yeah. Is it a crutch? It could be in the wrong hands. You still get the best out of it when you are able to understand what it is and isn’t doing.

9/ I did spend time watching many YouTube videos showing people generating code, and enthusing about how great that code was. Yes, it is incredibly good code - but it’s not production grade code.

10/ I spent a lot of time shaking my head, especially when seeing that people were accepting the first code it suggests. It’s like they didn’t understand you can embellish or see alternatives.

11/ And that’s where it really comes around to what was in the video I referenced about - in the hands of someone that knows what they’re doing, you get far better results than just accepting the low hanging generic results.

12/ In conclusion, I have so far found the experience positive. Remembering that it’s your “copilot” not your “line officer” is the key to success here. Even then, make sure you understand exactly what it generates. I’ll update when I’ve spent time in Rust with it.
