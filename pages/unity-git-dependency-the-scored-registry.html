Unity Git dependency, the scored registry, and the UPM thingy-thread! 1. Suppose you have a library that depends upon a bunch of other libraries like this in the case of my animation library called "vortex", https://t.co/8rOri5xG5V

2. Then you can have a "package.json" in your library that can contain your dependencies, like this. Then you can put vortex in the GitHub. https://t.co/efjXO7UxbT

3. Sounds fair and then you add the vortex in your project like this in the "manifest.json" file and you expect everything should go fine, except it IS NOT! https://t.co/2PWUg0hG7u

4. Unity will download only vortex and NOT any of its dependencies. So you have to tell your users to include the dependencies and now the manifest file looks like this: https://t.co/Nyp1AwcodW

5. Imagine the pain of managing a collection of libraries like this, I have 16 in my GitHub and it is a pure disgust of maintenance. So what could be the resolution?

6. Enjoy this forum thread if you would but the gist is "Unity will try to develop this feature in the future and no ETA is given". So this is it? https://t.co/yWZO8hw6J8

7. Actually if you can manage a package hosting server in EC2 or similar, you can use the following to create one and host there. https://t.co/QLnGoGe9kb

8. Then you can use the scoped registry as mentioned here. Or better! https://t.co/5doES2iHnU

9. You can use UPM. It has a command line interface, so your user will just write some commands to search, fetch and get packages instead of wandering around the manifest file. I think I like it! But... https://t.co/0e2VjTfj15

10. Please @poweredbyunity give a better way of managing git dependencies. It would be better for the small and indie teams.[/ThreadEnd]
