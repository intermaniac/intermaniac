WHAT IS REACT? React is a javascript library, It is a tool used by front-end developers for building single-page applications, which means that only a single HTML page is sent to the browser and routing and injection of content is handled by javascript in the browser.

You know all those sites where after the initial request to the server, in the subsequent ones you never see the browser reload when changing routes (an example would be This bird appüê¶, which is built with react), those sites are known as single-page applications.

the idea of React is fairly easy if you have used javascript to perform DOM manipulations. The idea is to have one HTML page and dynamically inject contents into the page based on the routes visited or interactions of the user

Since most, if not all of your HTML codes are writen in a javascript file, React takes advantage of JSX so users can create and write simpler easy to read HTML codes in javascript, which is then compiled down to raw javascript https://t.co/4WPnpEG3Iw

You can manage routing with React, so depending on the page visited, you can define what is to be injected into the browser page, this helps prevent those extra requests to the server, leading to a faster website

You should also note that react by itself can not be understood by the browser since it isn't pure javascript so it has to be compiled down by webpack before being sent to the browser

ADVANTAGES OF USING REACT > The user doesn't have to make request to the server when visiting other pages since routing is handled in the browser > React is component-based which reduces the stress of writing repetitive codes
