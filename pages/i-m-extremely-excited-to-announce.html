I'm extremely excited to announce the first-stable release of sigstore-python! https://t.co/UXIEkZ44pV this release comes exactly one year after we started working on sigstore-python at @trailofbits; this will be a thread on what you can expect now that we're at 1.0! 1/7

first: sigstore-python's 1.0 includes a stable Python API, which means that you can now `import sigstore` and do Sigstore signing and verification operations in idiomatic Python! docs are here: https://t.co/imm1oa2t6j 2/7 https://t.co/TQ3h5le5QI

next: we've stabilized the `sigstore` CLI, including a new `sigstore verify github` subcommand for GitHub Actions signatures! this allows for more powerful verifications, like pinning not just the identity but the exact trigger + git rev for the signing workflow: 3/7 https://t.co/ExhBRqOPQ5

i also want to highlight some key internal work: @jku added support for TUF with the 0.9 series, unblocking today's 1.0 release! this means that sigstore-python no longer pre-bakes its roots of trust, improving UX when fulcio etc. rotate their keys! https://t.co/nM8lrxaJ28 4/7

this work would not have been possible without @di_codes and the help and funding of the GOSST team at Google, as well as the broader @projectsigstore community! thanks especially again to @jku @AsraEntr0py, Zack Newman, and everybody else who helped us get to 1.0! 5/7

as always, sigstore-python is a single `pip install` away: https://t.co/LNKfSJLKVm 6/7 https://t.co/gxQ6sSzpp6

check out the Sigstore blog for the official announcement, plus the release notes for more details! https://t.co/a46mjE1Qhl https://t.co/99nkWWA1Hq 7/7
