Why is it called Bubble Sort? We call it bubble sort because in this the largest element bubbles up to the end and in the further passes we only consider the remaining elements.

What is actually happening in bubble sort? We compare the first element to its very next element and check whether it is greater or less if it is greater then we swap those two numbers then again we do the same with the other elements of the array. We understand it more, further.

What should be the approach? As we know we must check every element of the array, which means there is a loop running from 0 to the length of the given array. There are also sub-passes in one pass, which means there is an inner loop also.

First, let's talk about swapping: There are many ways to swap two numbers, but I use the ES2015 version(JavaScript): https://t.co/d7HseiBfnM

Now, the question is from where should the inner start running and where to stop? In a naive version, let's say the inner loop starts from 0 and runs till the length of the array. Let's see the solution: https://t.co/htyufLjlsp

This naive solution gives us the desired result but the problem is, why do we compare the last element in the second pass when we know that all we do in the first pass is putting the largest element at the end because it is its correct place.

So a better approach can be, to start the outer loop from i = arr[length] - 1 and end at 0 and start the inner loop from j = 0 and runs till j < i - 1 and now inside the inner loop compare if the arr[j] > arr[j+1], swap them. Let's see the solution: https://t.co/2oHu2J7HSe

Is there any optimization that we can do? Yes, in a partially sorted array bubble sort should take a very less amount of time to sort the array and in our solution, we haven't done anything specific for the partially sorted array.

Let's understand it with an example: Consider the array is [5, 1, 2, 3, 4]. Now, we know that after one pass 5 will be placed at the end position and the array is sorted but in our solution, the inner loop will keep comparing elements for 4 more passes.

What we can do is, check that if there is no swap happening in one complete pass of the outer loop then we break the loop and return the array. Let's see the solution: https://t.co/ShOJKeuLUt

Time Complexity: The time complexity will be BigO(n^2) because the number of iterations increases with the number of inputs.
