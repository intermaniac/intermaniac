Validation: It's not advised to send whichever data is entered by the user to the server. Hence validation is used for restricting and instructing users on what kind of data to enter in the form.

Types Client Side Validation: Performed by web browser before input has been sent to a web server Server Side Validation: Performed by web server after the input has been received.

In HTML, form validation is achieved using several attributes.

type - Restricting the type of data using type attribute on input. type="email" - only accepts email. Need to mention a pattern to validate an email. type="password" - entered characters will become dots as it is intended for password.

required - The input field must be filled in before submitting the form.

minLength - Specifies the minimum number of characters required to enter before submitting the form. Works on: text, search, url, tel, email, and password. Shows error message as Please lengthen the text to x characters are more (You are currently using 1 character)

maxLnegth - Specifies a maximum number of characters required to enter before submitting the form. Works on: text, search, url, tel, email, and password.

min & max: Specifies min and max values for input. Works on: number, range, date, datetime-local, month, time and week. Empty value is accepted.

pattern: Specifies a regular expression that the input value is checked against on form submission. Input type checks against given pattern and shows error message as Please match the format requested.

size: Specifies the visible width of the input.

multiple: Works on input & select to allow user to select multiple items
