Often it is possible to reuse an object rather than creating an equivalent one each it is needed. This can lead to much more efficient code! An object can always be reused if is it immutable

This is really bad example of what not to do. The argument "pasta" passed to the String constructor is itself a String. So if we aren't careful we could end up making a lotta pasta here! https://t.co/L6u5vtn8Pm

A good way to avoid creating unnecessary objects is to use Static Factory Methods (Item 1). Unlike constructors they are not required to create a new object each time they are called

Remember if you're creating an expensive object caching it can offer big performance gains!

You should also prefer primitives to boxed primitives and be on the lookout for unintentional autoboxing
