PATH variable is used to store the path of the binaries which are used by the system. e.g PATH on my system looks like : /usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin If your program calls a binary file, Linux will start to hunt from these locations, in sequential order.

Each `:` separated location is path to directory where linux will hunt for binaries. a:b:c say you call for `cat` binary, and if the binary is found in `a` path, the search would stop. If not search would continue. And my Friend, LINUX $PATH CAN BE CHANGED BY USER

So, we can exploit it !! Yes, but with constraints. To exploit this to fullest, you need to have 1. A binary with S bit set. 2. That binary must be calling a system binary. Next step ?

Identify that binary A and binary B that is called within. Exploitation Steps: 1. Create a vulnerable binary with name of B 2. May put in /tmp/ 3. Prepend /tmp/ path in $PATH 4. Execute the binary A and witness the magic.
