Been trying out GitHub code spaces with vsCode remote recently, which is pretty neat. It's pretty much the same as local development, even though Docker is running in the cloud, not on my machine. Thread collecting useful links and notes.

This is the Laravel template I'm using. If you want to try it, create a new repo from this template, then click the "Code Button" and select "Code spaces" https://t.co/KPLG6uv2z3 https://t.co/R0SDwpKwO2

The default option is to load vsCode in the browser. This is a no-go for me. I always open codespaces the desktop app. This is pretty straight forward, with the code space extension.

I have used cloud-based development with a few different setups. As long as it's not browser-based, I'm happy with it. My vsCode is way too customized, I struggle with the default config. Using the normal desktop ask makes all the difference.

I have used @gitpod in the past, which is really great, but only in browser. Easy to configure for Laravel https://t.co/NkmWDHunoS

There is a desktop integration for gitpod in beta. I will try it again. https://t.co/B5CxKP4D4l

The big problem with codespaces is the pricing. The on demand pricing seams fair, but for organizations you need a team plan, which is priced per team member and outside contributor. That's a hard for what I'm doing. https://t.co/Jxydxa6VJf

There is also Coder, which is an open source project for deploying your own vsCode remote. They also have enterprise options. https://t.co/iTnnJcaoI9

They are working on a V2, which is in Alpha https://t.co/3HR8e1UbX7
