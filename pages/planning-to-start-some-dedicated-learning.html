Planning to start some dedicated learning each week to a new topic/skill that I haven't touched before. But I'm finding it a bit of an internal debate on which to learn. It's between: -Docker -JS Framework (Vue) Very different categories. Why the debate? 1/ðŸ§µ

I feel a lot of outside pressure to learn and use Docker for my local dev environment. These days I hear a lot of "what, you don't use docker??" and the implication that it's a must-know technology. 2/

However, my dev machine runs Ubuntu and I have a LAMP stack installed directly. I can change PHP versions if needed with a couple of simple commands. Learning Docker has 0 current benefit for me, but the pressure to know it is real. I also like learning sys-admin type things. 3/

Learning a JS framework (I've settled on Vue) however has direct benefit to the projects I work on. I write a lot of vanilla JS and taking my JS work to a new level would be extremely helpful in a tangible way, to real client projects. 4/

However there's a 'stick to the backend' sort of pressure. I'm never going to master front-end design, CSS, etc. I have very little interest or time for having to setup and learn a lot of tooling for the front end like webpack, etc. 5/

Part of me feels like I *should* try to get really specialized in one area, like backend development in PHP, and I certainly am strongest there.... but I really do enjoy being a broad, multi-skilled jane-of-all-trades type developer. 6/

I know 'enough to do it well' in a lot of things, and this has benefitted me greatly as a solo developer who generally doesn't have a team member that I can pass things to when it's out of my wheelhouse. 7/

There is definitely less 'cred' for developers who don't have a single skill that they have achieved advanced mastery in. But I find that being a technology generalist is more useful to my clients, teams I've joined and for keeping away burnout. 8/end
