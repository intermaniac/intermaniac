@stripe is a powerful SDK (software development kit) that allows developers to create: - digital products - digital invoices - payment gateways - secure and compliant payment processes Imagine having all the financial capabilities of a @Shopify or @amazon but for YOUR own app ü§Ø

Thanks to the @stripe, we can handle all different parts of the payment lifecycle...meaning we can focus more on our code and less on the set-up. Aka a win-win (if you ask me)

In this guide, you'll learn how to: - create products in @dynamodb for a user to select - use @stripe to handle dynamic invoices for the products a user selects - use #useshoppingcart by @Dayhaysoos to handle the cart that a user sets up - run payments securely with AWS Lambda

This guide is particularly üî• because it breaks down how to run secure back-end code for payment processing using GraphQL. This is actually something I've been wondering about for a very long time, until I found @mtliendo's guide and built this out to #LearnWhileDoing

Here's how it works:

(1.) GraphQL Schema We first create a query that takes in AWSJSON (stringified cart items a user selects) as an input string. We then hook an AWS Lambda Function via @AWSAmplify onto that query which will handle our NodeJS script to process stripe securely in the background

(2.) NodeJS AWS Lambda Function We will run our @stripe code in the backend, securely with AWS Lambda. There are many vulnerabilities that can come from processing payments in static code. If it's part of the build, you will bake your private vars into the front-end...aka BAD

(3.) Front-End Pt. 1 In our virtual store, we see 2 "Products" which are 2 items saved to @dynamodb and stored in a database. Our admin group was able to create these items so others could see them in the store and purchase them. https://t.co/uxoAH2vHNi

(4.) Front-End Pt. 2 We will use #useshoppingcart to turn the items we see (available products) created by the admin user and stored in @dynamodb into stringified JSON we can send into AWS. When we click "Add to Cart" - it adds the item into a string that goes into our "Cart" https://t.co/YeJHnIUKrP

(5.) Front-End Pt. 3 When we click "Checkout", our Lambda Function will create a dynamic invoice for us with the products we selected on the previous screen with a checkout page. We can then make a secure payment via @stripe and get our items üòä https://t.co/9Ge7iMOayK
