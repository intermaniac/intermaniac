Did you know: Chrome, Firefox and Safari all disagree on how important certain types of files (JS, CSS, images, ...) are?! üòØ This can cause some major problems for optimal #webperf! More details in my article at https://t.co/5VxfrrbbMe And some highlights below üëáüßµ https://t.co/TSoWfR0j0m

In HTTP/2 and HTTP/3, the "resource prioritization" system allows the browser to tell the server which files on a web page are most important, and so which should get downloaded first. In HTTP/3, this can be done using the new "Priority" HTTP Request header: https://t.co/Mg4FTZAqLV

However, the browsers don't really agree on which resources are most important... For example, Chrome has a MUCH higher opinion of fancy typography than Firefox and Safari: https://t.co/Y1CI806aio

Chrome also gives CSS the same priority as the HTML, unless it's at the bottom of the document, then it's clearly less crucial. For Safari and Firefox, all CSS (except media=print üñ®Ô∏è) is equal and subordinate to the HTML. https://t.co/awltzdhfGJ

For JS, things are even more inconsistent, especially for async and defer (what is Safari thinking there... I wonder) Interestingly, Firefox doesn't really like preloaded JS all that much. https://t.co/rKtGp24MT3

For images, things are a bit more sane, though beware the gotcha: preloading your LCP image doesn't actually give it a higher priority! Use the "fetchpriority" attribute for that (in Chrome) and beware Safari üòë Kudos to Safari though for lowering priority on hidden images ü•∑ https://t.co/ZowICD84tK

Finally, fetch() is clearly a second rate citizen in Safari and Firefox but can easily delay other JS data in Chrome. Route splitters and lazy loaders beware! It does allow for manually setting the Priority header, so maybe there's hope for SPAs still? üòá https://t.co/Oc2qvWi1xf

In conclusion, it's a bit of a sad state of affairs, where not all browsers have all features, and none implement them consistently (I thought IE was deprecated? üò¢) If you're going for optimal #webperf, it might be a good idea to tweak resource loading on a per-browser basis!
