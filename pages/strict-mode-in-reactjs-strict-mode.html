Strict mode in #reactJs (A thread) Strict mode is a tool for highlighting the potential problems in a #react application. It does not render anything to the UI instead it activates additional checks and warning for its descendants.

Why is strict mode useful and why its more discussed after React 18? So basically after react 18 react made the strict mode such that it helps in identifying following things which may cause issue in react.

:- Identifying components with unsafe lifecycles. :- Warning about legacy string ref API usage. :- Warn about deprecated findDOMNode API usage. :- Detecting unexpected side effects. :- Detecting legacy context API. :- Ensuring reusable state.

Lets discuss some of the main points : Detecting unexpected side effects So react works in 2 phases

First is the render phase which determines what changes need to be made to DOM by calling render method and comparing results (Expensive and time consuming operation) Second is commit phase where react applies the changes.

Step 1 can be a bit slow so react plans to break step1 into multiple pieces and pausing and resuming works to avoid blocking. So to accomodate this change our components need to be working fine.

When this happens our component will be mounted and unmounted and remounted by strict mode just to check if anything breaks or any side effects are there or not in our components.

This will help to find any potential side effect of not returning functions on unmount and many more issues. (and a lot more) You can read more at : https://t.co/ENCgEQvym7 A detailed video lecture : https://t.co/lqFVxvm2Xz Hope you liked it : )
