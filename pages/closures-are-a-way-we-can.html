A closure is a feature in JS where there is an inner function within a parent function that encloses it while also having access to that parent functions variables.

This creates three scope chains: - Access to its own scope, this includes the variables declared and defined between its curly braces - Access to the parent function's variables - Access to the global variables outside of the parent function.

Let's look at a basic example: Here are two functions, a parent function with variable parentVar and a child function within it's scope that has a variable childVar https://t.co/vFqZ09GTZS

Lets go ahead and invoke the parent() function and store the result in a variable: var result = parent() When we call this function the following things happens:

- There is a scope created that is limited to parent. Within this scope parentVar is created and defined to the value of 10. - The following line is just a function declaration for child. Nothing is invoked here.

- The final line is a return statement that returns the child function we declared in the previous line. Here the entire body of the child function is returned.

- The contents returned are then stored in the result variable. - The parent function completes its execution and all variables that were in it's scope no longer exist. This includes parentVar.

That last step is important because once a function completes its execution, any variables that were defined inside it no longer exist and cannot be referenced. Thus, the lifespan of a variable defined within a function is the same as the lifespan of its functions execution.

Now let's look at the contents stored in our results variable which should be the child function. Since the type of the variable is now a function, we can safely invoke it with the () operator, such as result(). https://t.co/qNuSnPqX3R

Now let's go through the steps of result() execution: - Variable childVar is created and it has value 20 assigned to it. - JavaScript then tries to do the addition of parentVar + childVar.

But how is it able to add parentVar to childVar when we know parentVar no longer exists? This is the magic of closures, and how they allow access to variables of enclosing functions.

The child function is able to preserve (closure) the scope chain of the parent function at the time that the parent function was executed. This gives it the ability to access the parent function's variables.

I personally like to think of closures like this: Since functions are of type Object in JavaScript, we are essentially just adding a reference to a parent functions scope within our child function (object).
