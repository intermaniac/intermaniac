3. branch This command is used to list, create or delete branches. In git, a branch can be used to request new working directory, staging area and project history. Branches act like abstractions ( a term used in Object oriented programming). Here a programmer hides all....

but the relevant data about an object in order to reduce complexity and increase efficiency.

4. commit This command is used to record changes to the repository. It is an operation that sends of the source code to the repository.

Just in case. A "repository" is like a folder that contains all of your project's files and their revision history.

5. merge. To join two or more development histories together. The command let's you take the independent lines of development created by a git branch and integrate them into a single branch.

6. rebase. Does the same task done by the merge command. But is a little different as it creates a linear history while merge preserves the entire history of your repository. Only use it when the target branch is private otherwise you're better off with using merge.

7. reset Used to undo local changes to the state of the repository. I.e. changes that are on your local repository on your machine. The ones that you haven't pushed to the servers.

8 switch This command helps you switch/ shift from one branch to another.

9. tag This command is used to capture a point in history that is used for a marked version release. A tag is a branch that doesn't change.

10. fetch The fetch command is used to download contents from a remote repository.

11. pull The pull command is used to fetch and download content from a remote repository and immediately update the local repository to match that content.

12. push This command is used to upload local repository content to a remote repository. "Local" refers to the repository in your computer. "Remote" refers to the repository in the servers i.e. in the internet.

13. add By using this command, you tell git that you want to include updates to a particular file in the next commit. The changes can only be recorded if you run the git commit.

14 mv Used to rename and move files. It works by deleting the existing file and recreates the same file but under the new name or folder.

15 restore It helps us to unstage/discard uncommitted local repository changes. i.e. when you want to do away with changes that you haven't committed.

16. rm Used to remove individual files or a collection of files from the git index. An "index" is a critical data structure in git.

17. bisect Used to discover the commit that has introduced a bug in the code. 18. diff This command displays the differences between files in two commits or between a commit and your current repository.

19. grep Prints lines marching a pattern in any committed tree, the working directory, index of a string or a regular expression. 20. log Displays all commits in a repository's history.

21. show Used to view expanded details on git objects such as blobs, trees, tags, and commits. 22. status displays the state of the working directory and the staging area. Is shows which changes have been staged and which ones have not.
