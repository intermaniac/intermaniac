Ha! I have finally generated a "Django" static app with Cactus. Here's a thread of what I've learnt this morning. ðŸ§µ Cactus is powered by Python, which makes it easier for you if you have a Django background.

Unlike Django where all the HTML files are stores in a templates folder, cactus has two separate folders. 1 is called templates and it handles the base.html file. The 2nd one is called pages which handles the rest of the HTML files (index, about, etc).

Cactus has a build process and a serve process which I would equate to runserver with Django Web Framework. The build process collects all the changes you have made in the template and pages folders and then serves it to you on your browser.

Every time you make a change on your files, you have to cactus build and cactus serve on your cmd. There could be a process to automate this, kinda like autosave with VS code but I am yet to see it in the few hours I have interacted with Cactus.

There's a slight slight chance cactus handles backend stuff, things like models but it is highly unlikely. Cactus also uses the Django templating language (?) to handle HTML pages. So you'll have the standard 'extends' base.html' headers, etc.

Static files. Instead of doing something like {%static ' css/fhfhf/style.css' %} i did static/css/style.css.

There's still more to learn, of course. There isn't a lot of information online either so I'll probably have to write a more detailed article on all these processes. All in all, Cactus is not that bad once you get the hang of it. Took me about an hour to 'figure' it out.
