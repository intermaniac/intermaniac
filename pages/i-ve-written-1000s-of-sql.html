1/ How Joins Work When two tables have the same keys, the data can be appended together. Joins are used to bring two different data sets together in one view.

2/ Keys For a join to work between two tables, each table must have the same key. A key is a unique identifier in a row that makes that row of data unique. For example, this could be an account_id, an application_id, or a product_id.

Example: Table A has data from an application for a bank account, including the account_id once approved. Table B has details related to the account after it was created. I want to look both datasets in one view. I can join Table A and Table B using the key "account_id".

3/ Inner Join One of the most common join functions. Inner Joins are when you only want to view data that matches between the two tables. Ex: We could use an inner join to only look at account data for applications that were booked. So we would join on account_id. https://t.co/bBsNQKVZxf

2/ Left Outer Join Also a common join type. This is when you want to take your primary table, e.g. Table A, and append data for keys that match in Table B. Ex: Assume we wanted to look at all application data, and append account data for the applications that were booked. https://t.co/ll7sz1JyM5

3/ Right Outer Join A less common join type. Here we start with Table B and want to append data from Table A. Ex: Assume we want to take all booked accounts and join application data. https://t.co/XpegytZhUF

The difference with inner join is that here you might have booked accounts in Table B where there is no matching application data in Table A.

4/ Full Outer Join Also less common. This is where all data is combined into one table. When there isn't matching data between the tables, the non-matching attributes are filled with NULL. NULL is how the database signifies no data is available for a particular attribute. https://t.co/bXAR9hMIsW
