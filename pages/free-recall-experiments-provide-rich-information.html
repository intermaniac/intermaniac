Free recall experiments provide rich information about memory function, but analyzing free recall data can be complex and difficult. My Python package, Psifr, makes it easy to perform sophisticated analysis of free recall performance and organization. https://t.co/O3nPeUZ3tz üßµ https://t.co/HeYiC4nKch

In Psifr (pronounced like ‚Äúcipher‚Äù), data are input in a simple standard table format. This format is widely supported and compatible with many data analysis tools. Existing data in MATLAB format can easily be converted to standard format using a provided script. https://t.co/lxnMHshX1C

Raster plots, which display a marker for each recall attempt in each list, provide a quick overview of a dataset. Here, recalls are colored by serial position; lighter colors indicate items presented later in the list. Red indicates intrusions of items not on the current list. https://t.co/KENTeTIcZO

Many standard analyses are supported in Psifr, such as recall by serial position and probability of Nth recall. Recall order can be analyzed to test for clustering based on temporal, categorical, or semantic features of the study list. https://t.co/Uv531Atsff

Each analysis is highly customizable. For example, if there were multiple categories of items on a list, you may want to conditionalize an analysis based on whether each transition was within the same category or across different categories. https://t.co/WhYpAFEetI

Each analysis in Psifr is checked for correctness using automated testing. These tests, which are defined based on published test cases when available, are automatically run every time the codebase is changed to ensure correctness and stability. https://t.co/a0qHsjJ4bo

Psifr is designed to be extensible, and new analyses are being added over time. The codebase for Psifr has been peer-reviewed and is published in the Journal of Open Source Software. Special thanks to @cMadan and @paxt0n4 for their helpful feedback! https://t.co/JQttlEBNQm

Psifr is written in Python, but the in-development {psifrr} R package makes it possible to call Psifr functions directly from R. Feedback on both Psifr and {psifrr} is welcome! https://t.co/GXjhj1rRKn https://t.co/lQNVDRwiam

The analyses in Psifr are based on measures developed by @PolynSean, @KahanaMichael, myself, and many others. See the user guide for references. https://t.co/H5xQNnRKSD
