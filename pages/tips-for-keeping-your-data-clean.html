1. Use normalized formats where possible. For example, make sure all dates are in a standard format like yyyy-mm-dd or mm/dd/yyyy. This makes it easier for other scientists to work with your data, and it also makes it easier for you to track changes over time.

2. Try to keep your data as close to real world values as possible. This means making sure all values represent something that can actually happen in the world (for example, using measurements from weather stations rather than making up numbers).

3. Try not to use too many uncommon datatypes in your dataset. This can make it harder for other scientists to work with your data, and it can also lead to more complex code cleanup challenges down the road.

TL;DR: 1. Use normalized formats 2. Keep your data values close to real world values 3. Not use too many uncommon datatypes
