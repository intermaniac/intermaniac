Introducing LangChain (🦜🔗): a python package aimed at helping build LLM applications through composability `pip install langchain` https://t.co/3V3myaAOPG A thread on why we built this, what you can do with it, where it's going, and other tools in the ecosystem: 🧵

Why we built this: Large Language Models (LLMs) are a powerful new machine learning technology. But using these LLMs in isolation is often not enough to create a truly powerful app. 💪 The real power comes when you are able to combine them with other things.

LangChain aims to help with that by creating: 🧩 a comprehensive collection of pieces you would ever want to combine ⛓️ a flexible interface for combining pieces into a single comprehensive "chain" 🏗️ a schema for easily saving and sharing those chains

Before diving in: please note that this library is still in the early stages of development. 💥 This field is still exploding and so we expect this library to evolve as feedback comes in (some please give some!) 💁‍♂️ And if you'd like to contribute - please reach out!

What you can do with this: 🧩 LangChain already supports a number of components: OpenAI API, Cohere API, SerpAPI (a search API), python interpreter 🔮 Working on adding Wikipedia support, vector database support (FAISS/pinecone), and others

Why is this important? ✨ Abstracting out what LLM API/Search provider you are using can let you easily experiment and switch between them. 🧱 Having a collection of pieces makes it easy to create more advanced logic/apps

⛓️ These pieces make it possible to recreate some popular chains, including the ones listed below: Self Ask With Search LLM Math NatBot More on those below

Self Ask With Search Implements the paper from @OfirPress where you ask an LLM to reason through a complex problem by asking the model to ask simpler questions and then answer them with Google Search https://t.co/PG2OxkxY7g https://t.co/GcrXByPmXa

LLM Math Implements a cool use case from @goodside and @amasad of prompting an LLM to answer math questions by consulting a Python REPL https://t.co/oCLKmznAoq https://t.co/ECLGUFgoSX

NatBot Implements @natfriedman's using GPT-3 to control a browser https://t.co/hjBdGSdT8p

And of course, you can just use it for normal prompt templating https://t.co/k0I9iZ9hB2

🏗️ All of these chains will soon be able to be saved to and loaded from json files, allowing you to easily save, share, and edit configuration.

🚀 What is next There are a few things in the works, including: - ReAct from @ShunyuYao12 @_akhaliq - WebLM from @aidangomezzz - Hooking into a vector database for a search engine from @_brivael_ - More LLM APIs (@BananaDev_ ? @huggingface ) - More Search engines - Wikipedia

🔧 Other tools There are a bunch of other great tools for working with LLMs, with have similarities/differences with LangChain If this seems close to what you want, but not quite there, you may want to check out some of the below

@dust4ai from @spolu - a no code/low toolkit for designing and deploying large language model apps! Also has a great focus on composability

@everyprompt from @evanjconrad - a prompt management tool for working with language models. It has a super intuitive UI/UX that I am very jealous of!

Manifest from @laurel_orr1 - a python framework that focuses on making prompt programming simple, interactive, and reproducible https://t.co/amvyydKrnM

@UnstructuredIO from Brian Raymond - a bit different, focused not on LLMs but rather on data preprocessing. I hope to integrate their capabilities into LangChain at some point!

Before I wrap up, shoutout to @PredragGruevski and other Kenshins for setting a good example with the python packages they opensourced - whenever we had a question about the best way to develop this, we tried to copy what was done there!

That's all for now - stay tuned for any updates, and reply with any ideas of things you would like to see!
