Python 3.11 comes with a new tomllib package in the standard library. It allows us to read toml configuration files. Never heard of toml? Then lets dive into it Thread üßµüëáüèæ https://t.co/zcPCpwzoBR

TOML stands for Tom's Obvious Minimal Language. Its a format for (mainly) writing configuration files. You might think why we need another configuration format. Whats wrong with using xml, json or yaml? or even python itself? https://t.co/y5PKOh1M3M">https://t.co/y5PKOh1M3M

The issue with XML is that it is simply too verbose. You have to type so much for simple configurations. You have to match open and close tags. Empty tags with no children have a weird syntax. It's really easy to mess it up. https://t.co/q7WC8Ywy6f

Unlike XML, JSON has a clean and minimal structure. It is easy to read a JSON file. Unfortunately, JSON has limitations. You can't add comments. JSON is very strict and will fail if you use single quotes instead of double quotes, or leave a trailing comma at the end of an array. https://t.co/hgzqxo2nQU

That brings us to YAMLüò∂ At first it seems to be easy enough to read and write YAML. But YAML is really, really complex with huge amount of bloat. Did you know that there are NINE different ways to write multi-line strings in YAML? It's a mess. Avoid. https://t.co/DEhvfy1LZB

One approach that is taken is to put all the config into a python file. That way you can use the full power of the language. While it sounds good at first, it has one serious problem: it allows one to write and run arbitrary and potentially dangerous code in the config file.

And that's where TOML comes in. TOML has been designed to be simple to read and write. It supports things like comments and multi-line strings. It is minimal, having learnt the lesson from YAML. Still, it can easily represent complex nested structures that are used in app configs

Here is a toml file. We can define sections in square brackets like [build-system]. When parsed, they will become python dictionaries. Sections can have subsections separated using . Below we have one section tool.poetry and a subsection tool.poetry.dependencies. https://t.co/O6YI1eYav2

Each section has key = value pairs. Values can be strings (4 types), numbers, arrays, objects and a few more. The configuration file is easy to read and modify by hand. There is no verbosity or deep indentation. Check out more examples in the spec https://t.co/y5PKOh1M3M">https://t.co/y5PKOh1M3M

Python now uses pyparsing.toml for configuration, and use of toml is increasing so it makes sense to add tomllib to the std lib. tomllib works similar to the json module, only thing you need open the file in "rb" mode. https://t.co/ZSZ3EIdPaq

You will get this output after parsing https://t.co/an4Wg70AoB

One gotcha - tomllib can only read and parse toml files, it can't write them out. If you need to write toml output, check out tomli-w https://t.co/EzgvQn35w2
