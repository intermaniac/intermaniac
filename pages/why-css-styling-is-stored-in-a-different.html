Why CSS styling is stored in a different file? ðŸ‘‡ðŸ§µ

If you ever wondered why CSS rules are stored in a separate file here is the answer.

HTML elements can store styling rules inside the opening tag. https://t.co/d8HgoU13g0

Can you imagine how long and hard to navigate each element will be if it stores its own rules? Not a great idea.

The CSS rules can be grouped under the <style> tag inside the <head>. https://t.co/HLVV2HD4jr

This is a better approach since we are grouping all the selectors in one place. But CSS files can get pretty long easily so this approach will add complexity to the HTML file making it harder to read, search and understand it.

The CSS rules can be stored in an external file and referenced by using a <link> within the HTML <head>. https://t.co/79F4Do6yR7

This approach has the most advantages. It will make the HTML and CSS files easier to read, search and understand. This is the default practice when working with HTML and CSS.
