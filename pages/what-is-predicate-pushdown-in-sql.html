What is Predicate Pushdown in SQL? ðŸ¤” Predicates are statements that result in True or False. In SQL, you can think of WHERE/HAVING as a predicate. Now, let's see what is the predicate pushdown ðŸ§µ

Consider a schema as shown in the image. We want to find the total_amount spent by customers whose name starts with 'A'. https://t.co/masIfMgFai

So, here's the query along with its execution plan: https://t.co/C6LtSS4q5G

As you can see, the WHERE clause is executed as the first step, while reading the customers from the disk. Even before the JOIN. This helps in reducing the amount of data to be processed at a later stage of the query execution. Let's take another example.

We want to find the total_amount of customers having odd customer ids. Here's the query and the execution plan for the same: https://t.co/01OsWj69sY

Even though HAVING is used to perform filtering after the aggregation, SQL is smart enough to realize that filtering the records before the aggregation will not be a problem in this case. Hence, the records are filtered while reading from the disk.

However, if you have a condition on aggregation like sum(total_amount) > 50 or likewise, the filtering will be done after the aggregation. I hope you get the idea. Let's put it more formally. https://t.co/fvo7hUMHBi

Predicate Pushdown is a behaviour according to which certain predicates/filters in a SQL query can be performed as early as possible so that further processing can be reduced. That's it for predicate pushdowns!
