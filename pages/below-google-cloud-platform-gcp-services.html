Below Google Cloud Platform (GCP) services explanation. A Thread. Cloud & Identity Access Management Cloud Load Balancing Google Compute Engine(GCE) Google App Engine Google Cloud Functions Cloud Run Cloud SQL Cloud Storage Instance Groups Snapshots VPC Networks & Subnets

Cloud & Identity Access Management - Identity and Access Management (or IAM) is your frontline defense for your cloud services. Security will be at the top of mind for basically everyone running a cloud operation—and it should be on top of mind for you as well. 1/19

Cloud & Identity Access Management - Google manages permissions and identity and access management with Cloud IAM and Cloud Identity. Cloud IAM is a unified resource access management system for both users and services. 2/19

Cloud Load Balancing - load balancers route the traffic and balance it across all instances. It’s a fully managed incoming traffic service responsible for distributing traffic across several VM instances. 3/19

Cloud Load Balancing - It is autoscaling, set by policy, CPU utilization, or serving capacity (or all of them). It supports heavy traffic and is instrumental in routing traffic to the closest instances, and can detect and remove unhealthy instances. 4/19

Google Compute Engine(GCE) - GCE is basically a virtual machine, on these VMs we run our all applications. In other words, everything happens on GCE (Virtual machine) Google Cloud offers several machine types that vary in CPU, RAM, memory, and cost. 5/19

Google Compute Engine(GCE) - Example- think of your laptop, if your laptop is managed by google in any of it's data center, and google is providing a service to use your laptop using google cloud platform (GCE), then you can refer your laptop as virtual machine. 6/19

Google Compute Engine(GCE) - Example continue- you can do all kind of operation on your laptop, this laptop will be known as instance (VM) in google cloud platfor, google offer several virtual machine(VM) that vary in CPU, memory, cost etc.. 7/19

Google App Engine- App Engine is a platformAs-aService or PaaS. So it requires no manual server setup and provisioning. It provides automatic scaling and load-balancing. It’s great for websites, mobile apps, and line of business apps that need to get up and running quickly 8/19

Google App Engine - Developers use App Engine for HTTP applications. it's Google serverless product, Google bills you for the time you spend processing, and you can scale up as much as you need. 9/19

Google Cloud Functions - Tt is serverless product of google , developer will write the code only and entire infrastructure code deployment will be taken care by Google. It scales automatically and is extremely low cost. Google bills you for execution time. 10/19

Cloud Run - it's another serverless product by google, Cloud Run operates on event-driven code. Instead of App Engine, something has to trigger them. Cloud Run is best for containerized apps, you use can use any language, library, or binary in cloud run. 11/19

Cloud Run for Anthos - It enables hybrid multi-cloud apps. Cloud Run for Anthos runs on Google Kubernetes Engine. You can use custom domains, integrated logging, and monitoring services. 12/19

Google Cloud SQL - CloudSQL is used to manage data with SQL. It’s a relational database service. CloudSQL manages, maintains, and administers PostgreSQL, SQL server, and MySQL. 13/19

Google Cloud Storage - Cloud storage is the unstructured data service on google cloud based on buckets and objects in those buckets. Those buckets can hold, well, anything that’s digital pictures, files, or entire databases. It has practically infinite size, 14/19

Google Cloud Storage - You only pay for the amount of data currently in that bucket (storage), and you don’t have to pre-allocate space in advance. It just abstracts away the need to deal with hardware, and you can focus on storing the data. 15/19

Instance Groups - if you have hundreds of google compute engine instances, then it's tough to manage complexity So we use instance group, by batching together instances into groups—buckets of instances where you can try to make changes to a bunch of them all at once 16/19

Snapshots - The primary backup method in which a compute engine takes a snapshot of an instance or attached disk, creating a backup of that disk. You can take that snapshot in any disaster recovery situation and create a new instance or disk. 17/19

VPC Networks - It is a virtualized network that provides ipv4 connectivity for GCP resources, Within your VPC, you can create all the Google services. A Google Project can contain one or more VPC networks. 18/19

Subnets - Subnet is sub section of network In a VPC, we can consume resources in all regions by creating a subnet within one region. Subnets use the Google global network, building subnets within a zone. We use the VPC to assign a subnet to one, multiple, or all regions 19/19
