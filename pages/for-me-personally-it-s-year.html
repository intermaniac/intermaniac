For me personally it's year of linux. I mean that kind of year, when I hit something I would consider terribly broken, and even with my experience I managed to work around it only barely and in very fragile way. (details in ðŸ§µ, involves color emojis in file vs gitk vs X11) \

To work with git repositories, I use 95% of time "git cola" GUI client, which can also visualise branch (call `gitk` tool). https://t.co/RJQeILpbYc is shutting down in Dec 2022, as it now exists as CodeQL at github directly, github suggest configuration script to set it up \

Suggested config script contains color emojis in comments (I committed it to project directly from github web interface). gitk started to crash immediately with "BadLength (poly request too large or internal Xlib length error)" it's due to bug in `libXft`. \

Suggested work-around "remove fonts-noto-color-emoji package". Sounds good to me, I can live w/o emojis, but I need gitk working. But I have KDE neon distro. Apt insists `neon-desktop` package depends on color-emoji font. *here* if you are new to linux, you are LTT-fucked-up \

Me knowing better, I didn't uninstall half of my OS, but used `dpkg` to uninstall only the font, run git cola + gitk again, reset the project into state before the script was added, re-done the script without emojis, and force-pushed new history to github. \

And then reinstalled the font back to have non-broken state of OS packages. -- about bug: https://t.co/ACEPQxni7O ("LTT" refers to Linus Tech Tips uninstalling desktop environment while trying to install steam during linux challenge) EOF
