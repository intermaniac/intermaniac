I spend a lot of time working with #SQL - not just writing queries, but transforming the parsed language, or working inside #postgres. A thread on the projects I use every day to make my job easier: 1/6

Need access to the raw #postgres SQL parser? @pganalyze’s libpg_query is the secret sauce: https://t.co/jHHsLsGINY As it is written in C it’s fast, and with FFI it’s pretty easy to use in whichever languages you prefer. 2/6

If you are working with #postgres protocol in #golang, you want to use @J3C10’s pgproto3 - https://t.co/tGsidN59BG - which turns protocol messages into golang objects. If you’re building any sort of postgres proxy you want to use this library. 3/6

Transforming from one SQL dialect to another? @Captaintobs has you covered with sqlglot - https://t.co/BG2P1P4Hrw - which I use to transform #sqlite SQL into #postgres SQL. Also, the response time on issues is insanely fast! 4/6

Speaking of #sqlite - sometimes you need tools that understand the liberal interpretation of syntax & types of sqlite. @simonw has you covered with sqlite_utils - https://t.co/ID5E1t8izj - a library that gives you programmatic access to the entities in a SQLite database. 5/6

Of course: #postgres itself. World-class database? Check. Easy to read, understand, and extend despite being a 20+ year old C code base? Check. Awesome community? Check. https://t.co/961JxUDYyq Thanks to all the contributors to these tools. They make my life easier every day! 6/6
