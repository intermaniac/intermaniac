JavaScript-powered websites are more exposed to an excess of code among the lines of their rendered HTML (DOM). In turn, this may cause render delays during the second wave of indexation 2/

What sorts of unused JavaScript can cause render-blocking on your websites? 1) Non-critical JavaScript 2) Dead JavaScript 3/

1) Non-critical JavaScript refers to elements that are not needed for above the fold but are being used elsewhere on the page (e.g an embedded map at the bottom of a page‚Äôs content) 4/

2) Dead JavaScript refers to code that is not being used at all anymore on the page. These could be elements belonging to a legacy version of the website.

Should you solve this site speed ticket? IMO the trade-off resulting from this choice is ridiculously high, as a tiny change would imply: - Discussing room for amendments with a web dev - Accounting for unforeseen sunk costs - Rebuild the entire site architecture 6/

In a nutshell, consider assessing carefully whether to remove unused JS. You may well choose to look at page loading time or render-blocking issues from a different perspective. This could involve delay or deferring unused JS 7/
