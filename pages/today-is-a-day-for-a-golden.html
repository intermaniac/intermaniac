Have you had to force a Node.js process to stop using `process.exit()` because it simply wasn't stopping by itself after finishing processing? /2

There's a useful NPM package created by @mafintosh called `why-is-node-running` which uses the Node.js async hooks API to inspect what handles are still running after an execution /3

It shows all the handles and which line in your code would have caused the problem and all other related async operations running in the background. /4

If you use Node.js to create console applications and you have to force it to finish the program using `process.exit` be careful! This could mean that some operation is still running in the background and you'd lose data that is currently being processed ðŸ˜± /5

If you're a curious person, I'd recommend you take a look at the package's source code. It uses the Node.js raw API to handle and get information about async operations. I think you'd learn a lot from there! /6
