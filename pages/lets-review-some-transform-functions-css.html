Lets review some transform functions #css A #Thread 🧵

🔸 translate() 🔸translate3d() to translate an element is just to move along one axis or along more axis (X,Y,Z). 🔸 accepted values: ± [length], ±[percentage (%)] except translateZ() and 3d , which do not accept percentage values since it has no reference point.

🔸 scale() 🔸 scale3d() to scale is to either increase or decrease the element's size proportionally, an the value it takes are multiplier 🔸accepted values: only positive integers (no %)

🔸rotate() 🔸rotate3d() rotation function causes an element to rotate around an arbitrary vector in space (3d) 🔸 values: only accepted angle formats (deg,rad,grad,turn) 🔸 rotate3d() hides some powerful features read more 👇🏾 https://t.co/jGgebGavDK

🔸skew() 🔸skewX() 🔸 skewY() NB: there is no skew3d() hence no Z-axis to skew an element is to slant it over any of the both axis. 🔸values: only accepted angle formats (deg,rad,grad, turn)

🔸 perspective () it gives the appearance of a front to back depth. to an element (when viewed in the Z-axis) 🔸 values: only positive [length] values. 🔸lower numbers create more extreme depth and opposite for higher numbers. 🔸 should always appear first in your list.

🔸matrix() 🔸 matrix3d() it's meant for the gods and the PC 🙂

🔸 choosing a 3d style is a very crucial point in creating 3d arts with css. 🔸 transform-style: value; 🔸values: flat, preserve-3d ✅

🔸 Moving the origin often elements are transformed at it center what of it corners? 🔸 transform-origin: value; values: left, center, right,top,[length],[%].

so some functions not all accept 3 arguments such as those with 3d. 🔸 so we have 🔹functionX() 🔹functionY() 🔹functionZ() 🔹and function3d( combination)
