1. Scope your CSS to components. *Always* avoid global stylesheets. This avoids naming collisions and specificity hell at the same time. As an added bonus, you won't have to spend time thinking about class names, if you even need them.

2. Make composable and reusable components. This avoids duplicating styles. For example, maybe you have similar styles between a profile card and a product card. Make one shared card component and compose it to make specific layouts for profiles and products, respectively.

3. Keep semantics agnostic. This decouples a11y features and SEO from aesthetics. For example, it's often necessary to share styles between <a> and <button>, or <h1> and <h2>, etc.

4. Build atomic components. You'll rarely see a scrollbar in a component that only styles an element or two. This lets your component apply to more use cases, and keeps the impact scope for changes small. In other words, it's highly maintainable.

5. Don't expose style directly; expose use cases instead. This gives the component freedom to change its appearance from a single location, vs all over the app where it's used. For example, avoid font="bold" and favor importance="urgent".

6. Use the right tools. The best is native shadow DOM by far. Tied for next best is CSS Modules or CSS-in-JS solutions like StyledComponents or Emotion. Avoid CSS frameworks that break the aforementioned rules - remember that utility classes are global (see 1st rule)

If you obey all these rules, and practice them until they become second-nature, I think you'll find your problems just fall away. CSS is not hard for me anymore. Actually, I think it's really fun! Hopefully this thread helps you get there too.
