2. Multithreading Vs Multiprocessing When we say: "Running parallel CPU Bound operations", we're saying MultiProcessing. When we say: "Running parallel I/O Bound operations", we're saying MultiThreading CPU Bound -> Multiprocessing I/O Bound -> MultiThreading

3. Python GIL - The infamous feature By default, python uses a single thread ran under the Python Global Interpreter Lock (GIL). The GIL allows only a thread to hold the control of the Python interpreter. Hence, only one thread can be in a state of execution, any point in time.

4. Python GIL - Functioning: The Global Interpreter Lock states the rule: "The execution of any Python bytecode requires acquiring the interpreter lock." By doing that we're solving deadlocks while locking the different reference count variables.

5. The cons of the GIL in Python: Bytecode needs the interpreter code to execute, making any logic single-threaded. BUT: I/O operations spend part of their time waiting, giving the ability to share the GIL. CPU-bound operations don't, hence they can't share the GIL.
