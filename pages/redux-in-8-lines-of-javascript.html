2/7 Using 'dispatch' to update state. What is interesting is batching updates to multiple stores with signal's 'batch' function. Using 'effect' to subscribe to store changes. In this particular example, there are 5 batched dispatched actions but 'effect' executes only once. https://t.co/sxcSkLnYs4

4/7 @preactjs signals have integration with Preact/React with zero weirdness. I bet this code just works without using selectors or wrapping components into other components: https://t.co/c4d6ndQqt6">https://t.co/c4d6ndQqt6 https://t.co/jWvbe9yt6B

5/7 The way how @preactjs signals work is very similar how hooks work but without the React/Preact part. It solves the 'how to subscribe to multiple stores' problem by using the same ideas from hooks to know which 'store/signal' is accessed within a 'computed' or 'effect'.

6/7 I personally don't think Redux is bad but I think the 'subscription to the store' part could be better if based on something like signals (+ remove the 'global store single source of truth' idea)
