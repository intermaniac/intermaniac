Some very neutral, beginner-friendly observations on #Typescript This is very much aimed at those that are struggling or can't see the point, not the dyed in the wool experts... This is also aimed at those trying to build apps and software, not websites... üßµ

For context, I'm building a digital product that has three environments so to speak ‚Äì two force Typescript on me (Deno and Ionic/Capacitor) and the third where I could have used TS but I'd already started without (a standard Vue SPA) so I'm stuck without doing a crazy refactor

Caveat, my Typescript level is pretty basic ‚Äì I largely get the basics but I struggle when faced with typescript-max codebases. I'm more of a "write #Javascript until Typescript complains" and then add some Typescript. (Which I think is the best way so far)

Initially, the Deno and Ionic envs where I *have* to use Typescript were frustrating to work on. I constantly felt like I was doing things to appease typescript rather than what I needed to do right now ‚Äì it felt anti-productive

This was especially apparent when dealing with third-party API data and having to go off and make interfaces or types for things when *I* knew what they were and just wanted to get on with things.

Incidentally, google "json to typescript interfaces" ‚Äì there are loads of good tools you can dump some sample json into and they'll have a go at making typescript interfaces for you ‚ù§Ô∏è

So at this point, you'd think I fall into the anti-typescript camp. Not quite. I've been doing a lot of work in my Vue app since spending a lot of time in Deno and it's driving me mad not having typescript.

The great thing about appeasing typescript, VS code knows all my code, my types and interfaces. Without it I'm constantly hunting around my code base looking for what that random object is made up of, or console logging it just to see what a certain property is call.

Sure there's more to typescript but it's the editor level stuff that stops you making the majority of the mistakes the compiler catches anyway, and the productivity of not hunting around and console logging is a huge win.

So honestly, if you're making a large javascript app ‚Äì I genuinely don't think you'd regret using typescript. I think you can over do it, but aiming for the bare minimum isn't that tricky to learn and makes life so much easier ‚Äì I really regret not having it in my Vue app.

But if I'm just building a website. You'll find me over with the static site folks using things like 11ty and as little vanilla javascript as I can get away with üòä

Overall, I don't *like* typescript but I can see the benefits in the right projects. End
