Most Used Commands in Git. Part 1 - [A Thread]

git –version - This command is used to check the version of the git.

git init - This command is used to convert a local repository to git repository.

git clone repo_link - This command is used to bring a repository hosted on Github to our local machine.

git status - This command is used to see the modified and untracked files.

git add - This command is used to track the untracked and modified files. Adding files for tracking is also commonly known as staging the files. - git add index.html will start tracking the index.html file. - git add . will start tracking all the untracked and modified files.

git commit -m “message” - This command is used to save the staged files. - git commit -m “Added description” will save the staged files with the message “Added description”. - git commit -m “Added description” -m “Updated index.html” will save the staged files with two messages.

git remote -v - This command shows all the repositories connected to the git repository on the local machine.

git remote add origin - This command sets the origin to the location to a repository on Github. - git remote add origin repo_location will add the location to the origin. This way if a git repository created on a local machine runs git push origin master, it won’t throw an error.

git pull - This command will download the commits from the git repository on Github to the local machine. - git pull or git pull origin master (if the upstream is not set) is used to pull the latest commits.

git push - This command will push the commits from the local machine to the git repository hosted on Github. - git push origin master will push the commits to the git repository hosted on Github.

git push -u origin master or git push –set-upstream origin master will make the origin location as default location and the next time, the user won’t have to write git push origin master. Just writing git push will work.

git log - This command will show the previous commits.
