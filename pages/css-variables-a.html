CSS variables are entities that help define a value and be able to use it throughout a document if you ever encounter "--my-color: green;" in a CSS code base, that's how a CSS variable is defined these kinds of properties are accessed through "var()" function, let me explain

Fully-fledged websites have things like primary color, font family, or even border-radius that will be used thousand times so instead of trying to remember complicated hex color codes a thousand times it will be easier to define it as something readable that we can remember

you get the idea, now let's declare a CSS variable to define it, choose your desired custom property name and make sure to prefix it with a double hyphen as (--) and give it a value example --primary-color: #fgh44, --default-font: arial

like all other CSS properties make sure it's written inside a ruleset as: https://t.co/jF9PkTeGPb

from your knowledge of CSS, you know the above CSS ruleset will only be applied to an HTML class named "my-class" but we don't want that we want that, we want to be able to call our custom variable anywhere and assign it to anything

so the ":root" pseudo-class is used so that custom property can be applied globally, throughout an HTML document as https://t.co/UuwM4HJLwY

now we are all set let's say we have a header with the id="header" in an HTML file, and a footer with id="footer" that you wanna make the background color the default you don't have to remember the hex code, all you have to do is remember the variable name

then use the "var()" function and pass the variable name all in a target ruleset as follows https://t.co/5PXsOMV1HJ

ignore the rest of the CSS properties in each ruleset and just focus on the background-color as you can see I used the var function with the name of the custom variable and our footer and header are now #ffff without actually having to remember the hex code
