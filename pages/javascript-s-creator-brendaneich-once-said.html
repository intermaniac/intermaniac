@BrendanEich JavaScript can get weird when adding different data types together. JavaScript's type coercion handles these additions. For example, adding a string and a number returns a string, while adding a number and a boolean return a number. https://t.co/W7FgKuY5LV

@BrendanEich A few years ago you may have seen a tweet saying that `('b' + 'a'+ + 'a' + 'a').toLowerCase()` gives you a `'banana'`. Which is odd... We're missing a couple of `'n'`s. This works because of an `'a'` being added to nothing.

@BrendanEich If you were to console log `+ 'a'`, you'd get `NaN` (Not a number). String values can't be coerced into being a number. Numbers can be coerced into a string, so what ends up being added is something more like this: `'b' + 'a'+ NaN + 'a'`. Oddly `+ []` returns `0`. https://t.co/o4atURWTuc

@BrendanEich Let's get weirder; what happens when you add two arrays filled with numbers? Well, the two Arrays are coerced into strings and then combined. You're probably better off using the spread operator if you really want to merge them ðŸ¤­ https://t.co/LeqQIJ4XnM

@BrendanEich Here's another fun one! What's the result of `0.2 + 0.1 === 0.3`. I'll give you 5 seconds. 1... 2... 3... 4... 5... I hope you guessed `false`! And now you're left wondering why and don't worry, dear reader, we'll explain it to you.

@BrendanEich Floating point numbers in JS are stored as double-precision floating-point numbers. Numbers are represented as whole numbers multiplied by a power of two. Rational numbers whose denominator is not a power of two cannot be exactly represented this way.

@BrendanEich The double closest to `0.1` ends up actually being `0.1000000000000000055511151231257827021181583404541015625`. The double closest to `0.2`, like that of `0.1`, is a little larger than `0.2`. And the double closest to `0.3` is a little smaller than `0.3`.

@BrendanEich Knowing `0.1` and `0.2` are larger than what they should be, and `0.3` is smaller than what it should be, it makes sense that they aren't the same. https://t.co/YHC5YVTCE5

@BrendanEich Finally, let's talk about `parseInt`. Check out the image below and guess what the value of all of these `parseInt`s are. https://t.co/SN1vO274TI

@BrendanEich If you guessed that they all resulted in 0 as they're all less than 1, you'd be mistaken. The reality is that `parseInt(0.0000005)` is the outlier as it returns `5`. Why?

@BrendanEich `parseInt` will convert a number input into a string and then back into a number. It then takes the first character from the string of the number it converted into. `String(0.0000005)` ends up being `'5e-7'`. So, `parseInt` takes that `'5`' and turns it back into the number `5`. https://t.co/60wlba1BPt
