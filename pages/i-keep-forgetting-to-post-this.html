I keep forgetting to post this so wanna do whatever I remember now before I forget again. This is for my future reference. Installing (and messily uninstalling) a couple versions of #dotnet on #linux created a bit of a mess. Ran into an issue. Thread about issue and solution.

I had .Net5 and .Net Core, wanted to install .Net6. I didn't want confusion between them so decided to remove the former two. Wasn't able to find a decent way to uninstall them, so ended up trying to remove it using dpkg. This is kinda what created a mess I guess.

Decided to install .Net6 via apt. Where I ran into an issue was, removing .Net5 and Core the way I did, didn't clear up the symlinks, and just left broken ones. More specifically, the one that Omnisharp looks for, was a broken link. This is why intellisense wasn't working.

I think at some point in time previously I may have had to make that symlink manually to get intellisense working. Anyways, after some research and looking around at the various dotnet locations, I found the symlink and found out it was broken. Here are some dotnet locations. https://t.co/zm93ALVJtv

I had to create a symlink from /usr/share/dotnet/dotnet to /usr/local/bin/dotnet. "sudo ln -s /usr/share/dotnet/dotnet /usr/local/bin/dotnet" incase anyone needs it. Excuse any possible spelling mistakes.

That's how I solved the issue. Sharing this for my own future reference, just incase I run into the same issue in the future with .Net7. But if this info helps you in any way, great. This is a solution to a very specific problem I had.

Side note, I tried installing dotnet via snap, but that seemed to have it's own set of issues. The issues could be because I had previously installed dotnet via apt, or it could be something else. Trying various solutions mentioned online didn't help. So gave up on it.
