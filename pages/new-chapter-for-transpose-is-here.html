This announcement includes several major releases: - Full release of the SQL API - Performance upgrades to the REST API - Significant improvements to SQL playground - Revamped pricing system - Improved user dashboard

First, the Transpose SQL API is leaving its pilot phase and will now be included in your existing Transpose plan. Head to the Transpose Playground at https://t.co/vg0Px0tB6u to start drafting queries for free.

Explore the Transpose Atlas in the Playground to leverage performant queries written by other users. Some of our favorites: - Smart floor prices for all NFT collections - OHLCV USD prices for any token over any interval - Full historical PnL for a wallet

In full release, the SQL API will see higher limits across the board: - 500kb query size for bulk lookups by account, transaction, block, etc - 10MB response size for bulk downloads - Up to 15 minute query timeouts - High query concurrency for use at scale

Additional SQL API upgrades include parameters, which you can use to publish generic queries in the Atlas and run parameterized requests in your code. Also, you can now specify options to include custom timeouts for long queries and number stringification for rendering uint256s.

In addition to the SQL API, the REST API also received massive performance improvements: - Up to 10 thousand rows returned in each request! - ~20ms request round-trip for <100 rows - ~500ms request round-trip for 10k rows
