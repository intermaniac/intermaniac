Working with C++ (GDExtensions in @godotengine 4.0) is quite interesting but a lot of things are still difficult for me, as someone coming from Java: ðŸ§µ

1) I do not use Visual Studio but mostly VSCode + command line to write C++ Perhaps that is my first mistake and I should move to Visual Studio for a full "dev experience"

2) debugging the C++ code is quite difficult. For GDExtensions especially, I first have to build a .dll and then load it with Godot Engine to see if it actually works. Am I missing a trick to improve the feedback loop?

3) when attempting to load a GDExtension that contains errors, it prints me a stacktrace into the command line showing at which file/line it failed exactly. However, compared to Java, the stacktrace is not really readable and does not contain all the stackframes

4) sometimes, the DLL just fails to load with a message like this: how am I supposed to know what the issue exactly is? https://t.co/THeN3MCAEC

5) to investigate most of loading issues around DLLs, I have to rely on 3rd party tools such as https://t.co/M0UsD9IejV

6) I am thinking about writing up a proposal for GDExtensions specifically to improve the dev experience on them: - better stacktraces - debug mode where Godot is able to load GDExtensions from C++ directly (with breakpoints in C++ even) - make DLL dependency scan part of Godot?
