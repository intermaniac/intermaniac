When you execute a piece of code you want to know who invoked the code(who triggered the code execution) It is called invocation context. Why is it important ?

It's important because: - First, how else would you get the stack trace ? Knowing who invoked the code, helps in identifying what caused the error. - 2nd, It help in designing reusable code. A piece of code that can be invoked from multiple contexts can be reused. https://t.co/RYtBHgAFfR

Now, how does javascript maintain the invocation context ? It maintain it using a special object know as "this" keyword. In javascript, "this" object refers to the object that invoked the function. https://t.co/Y9lwE6rvqg

In most cases "this" simply is the object on the left side of the dot(.) operator. In few cases it refers to the GLOBAL/WINDOW object. And in other few it refers to the parent's lexical scope(read arrow functions). And a few other cases it refers to event.currentTarget. https://t.co/FAg850t5xY

And in a few other cases, you can explicitly override "this" usingW .apply/.call/.bind. Well, that's a lot of fews. I'll teach you all. You'll learn four contexts where a function can be invoked from: 1- Global context 2- Object method 3- Constructor function (new keyword)

4- In a DOM event handler You'll also learn about how "this" can be explicitly overridden using .apply/.call/.bind I'll talk about arrow functions and .apply/.call/bind a separate post.

Before you hit the follow @happydecoder, let's first understand "this" in detail. First, "this" in global context: - here, 'this' refers to the window obj. in browser, and global obj. in node environment. - however, under strict-mode, 'this' is undefined in global context. https://t.co/HO6OQL5mu4

Why "this" is undefined in global execution context under strict-mode ? https://t.co/Uxsyf4yDR1

"this" in Object method context: Remember, when I told you 'this' mostly always refers to the left side of the dot(.) operator. Object method context is the simplest way to understand that. https://t.co/iW5za7n8EY

What happens if you access the function using the dot(.) operator but not invoke it ? https://t.co/MLzDtqxRbZ

In the above case, we are simply accessing the function not invoking it. Later, when we execute the function, it behaves just like another global function. In other words, it gets executed in the global context. Therefore, the "this" here refers to the window object. https://t.co/a9HTpJM09q

Let's add .who property on the window object and validate it. logThis() is equivalent to window.logThis() in browser environment. logThis() is equivalent to global.logThis() in node environment. https://t.co/iRKqX7MW34

Often times, you would want to reuse the function defined on one object on another object. https://t.co/YEuPVVXoi7

In such cases, you can use .apply/.call/.bind to explicitly specify the value of "this" keyword. .apply, .call, and .bind is mostly used to share the function across different objects. https://t.co/81awdF43YF

We'll learn about .apply/.call/.bind in detail in a separate post, for now simply understand that the first argument in each of these function is the new "this" object that you want to explicitly use. https://t.co/gd22neMYZ0

Now, lets understand "this" in constructor: Let's understand how "this" behaves with the new keyword. When you invoke a function using the new keyword, the following things happen: 1- An empty object is created and referenced by "this" variable

2- Properties/methods are added to the object referenced by "this". 3- The newly created object referenced by "this" is returned at the end implicitly (if no other object was returned explicitly). https://t.co/FdPtN5p5tY

"this" inside event handler: "this" will refer to event.currentTarget. More often than not, developers will simply use https://t.co/ev1sRRsrAu as needed to access elements in the DOM, but since the this reference changes in this context, it is important to know. https://t.co/kPqcqUIj3W

One last thing, note that "this" behavior is not at all affected by how or where the function was defined. In the example below, we defined the function inline(as a member of obj). However, we could have just as easily defined the function first and later attached it to obj. https://t.co/oc9x5PaWZm
