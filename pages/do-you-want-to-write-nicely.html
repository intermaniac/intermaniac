Big projects like Foundry use conventional commits to make automatic releases seamless. Take a look at the commit messages. They follow this specific structure called conventional commits. Read more about this here: - https://t.co/ycl6Dfh0IA - https://t.co/oECh5Qin1A https://t.co/6sTOSKlt3g

This simple hook uses regex to check every commit message that you are about to commit. No external dependencies are required! You can quickly add it to your project with a couple of commands: ‚¨áÔ∏è https://t.co/HeHkt2ziYq

1. Create a "commit-msg" hook with "touch .git/hooks/commit-msg" 2. Open it in nano "nano .git/hooks/commit-msg" 3. Paste the hook 4. On MacOS: "control + O" to write to the file 5. "command + return" 6. "control + X" to exit 7. Make it executable "chmod +x .git/hooks/commit-msg"

If you don't make it executable, you will get an error: "The '.git/hooks/commit-msg' hook was ignored because it's not set as executable." These commits pass the regex: - fix(scope): commit message - chore: commit message This one will fail: - ugly commit message

You can automate the process with @keyboardmaestro as I did, and it will take 5 seconds to set this up in your project. Save the gist for future use. Enjoy! https://t.co/R8hZy73SBe">https://t.co/R8hZy73SBe

There are also downsides to using git hooks that you need to be aware of. You might disrupt the contributor's local workflow if you enforce them in your project. You can always squash commits on your PR if asked to do so. Consider the pros & cons and decide for yourself!
