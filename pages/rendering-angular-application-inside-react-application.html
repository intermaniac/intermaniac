Rendering #Angular application inside #React application. Four days ago I hosted the Angular-IL meetup at the @FronteggForSaaS offices. Thank to @eventhandlerPro for organizing this wonderful night. (thread) #meetups #MicroFrontend #FrontEnd #SaaS #frontend

(2/10) In my previous Front-End position at CheckPoint. Frontend Architects decided to host all company's frontend applications in a single portal.

(3/10) The Portal contained three main parts: 1) NavBar to display the logged user, notification, etc. 2) SideBar to display available applications. 3) AppRenderer where the application should be mounted. https://t.co/LaHyjkSdv0

(4/10) The biggest issue was that the Portal built with React and our application built with Angular, thats mean there is one option to continue; build with React from scratch.

(5/10) While the team starts working on the tasks to convert the project from Angular to React, I was trying to learn how Angular application bundling works.

(6/10) The first step was to understand what is the purpose of main/runtime/polyfills/chunks bundles mean? After one week I came with POC that can render Angular application inside React,

(7/10) The first impression was, “What about the performance?”. I answered, The performance is still the same. How?

(8/10) I built connector to sync states and routers in effective way to synchronize changes between application. Wrapped Angular application inside mount function

(9/10) And exposed option to download JS files from CDN and mount it inside a specific container.

(10/10) for PDF slides and working example visit https://t.co/UGKM2eeZyB
