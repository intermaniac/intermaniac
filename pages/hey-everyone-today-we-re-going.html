Let's start with 'git clone'. 'git clone' is used to create a local copy of a repository that exists on GitHub. This is useful when you want to start working on an existing project, or when you want to contribute to an open-source project. #git #commandline

Next up is 'git add'. This command is used to stage changes that you've made to your code. Staging changes allows you to selectively commit changes, rather than committing all changes made to the repository. #git #commandline

'git commit' is used to save changes to the local repository. It's a good idea to include a descriptive message with each commit, so you (and others) can understand what changes were made. Use '-m' to add a message. #git #commandline

'git push' is used to upload your local commits to a remote repository, such as one hosted on GitHub. This allows other collaborators to access your changes. #git #commandline

Finally, 'git pull' is used to download and merge changes from a remote repository into your local repository. This is useful when you want to incorporate changes made by other collaborators. #git #commandline

Those are some of the most commonly used Git commands through the command line. Of course, there are many more commands available, but these are a good place to start. #git #commandline
