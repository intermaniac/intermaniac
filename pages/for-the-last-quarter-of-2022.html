For the last quarter of 2022 I am starting my journey of learning data structures and algorithms. There's a lot of good reasons but I believe this is finally going to help me explain to ppl why tailwind is better than vanilla css. Will update this thread as I go. #SlightlyTechie

Recap ðŸŽ‰ Today I started with an introduction to Big O notation. I learnt that Big O is basically how we compare the efficiency of different algorithms.

I did wonder why efficiency matters if the code works and I learnt that in most situations depending on the scope and context of the project you are working on, you may not care about the efficiency of your code (apparently nobody cares about the performance of my todo app)

But in situations where you are working on applications that is going to be used by lots of people, it really matters.

I don't know about you but If I'm logging into your website and the piece of code that runs every time a user logs in takes a long time to do its job, I'm gonna wanna to talk to your manager or you know whatever the bad yelp review version of software development is.

So yes it matters that your code works, but it also matters how fast it works or how efficient it is, especially in large projects. Writing better and more efficient code can save you a lot of money and significantly improve the user experience of your users.

People apparently don't like slow and buggy applications for some reason.

So to recap I learnt: 1. We use Big O Notation to analyze the performance of an algorithm. 2. Big O Notation can give us a high level understanding of the time or space complexity of an algorithm.

3. Big O Notation really doesn't care about precision, just the broad strokes is the algorithm linear? is it quadratic? or is it constant?

4. Time or space complexity (as measured by Big O) depends only on the algorithm, not the hardware used to run the algorithm. I learnt that manually timing our code with the https://t.co/BMloNawdTb() method is not a very good approach when measuring time complexity.

I also learnt a couple of Big O shorthands like the fact that arithmetic operations are constant. How to determine space complexity and a brief intro to logarithms.
