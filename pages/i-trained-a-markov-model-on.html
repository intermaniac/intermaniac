Some others that I'm fond of that weren't seeded that way: 25) 'Many Python modules are to be frustrating, arcane and non-obvious.'

26) 'The title says it all -- this PEP proposes the introduction of a raw bytes sequence object and the unification of the current str and unicode objects.' (this one is an alarmingly good mixture of real PEPs, from some experimentation with a larger state size)

30) 'This PEP proposes to deal with ambiguous time specifications during DST changes.' (unfortunately, the model was not more forthcoming, this sounds novel!)

Note that these are more cogent than the average because they've been collated by me, it outputs more nonsensical stuff fairly regularly as well, e.g.: 'This permits build systems were able to freely invoke class methods avoids that ambiguity.'

Switching tenses in particular is a fairly unavoidable hallmark of this sort of model, because tense is a much longer-scale correlation in a sentence than an n-gram model will usually have. Possible to do it with very large n, but then you often just regurgitate the corpus...

Another long-distance correlation problem that shows up a lot in this corpus is the presence of unpaired (), [], {} (the ones given above are verbatim, it just does this a lot) Possible to clean this up in postprocessing, but the author is lazy :)
