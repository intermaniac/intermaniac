I’ve been having a lot of success landing AWS roles paying $100+/hr and I want to share with you all on how you can too. Here’s how you can position yourself for AWS developer/engineer jobs making $100+/hr [thread]

Algorithms is what you’ll be writing to solve problems, it’s well defined rules or steps that are created to obtain a specific result/output.

Past projects developing applications in AWS From my experience you don’t need a Github portfolio to show potential employers, you need to be able to discuss your past projects at a high level, discuss your success, and any problems that you faced and how you overcame them.

You’ll also stand out if you’re able to discuss a project that is somewhat relevant to what the potential employer is attempting to build. They want to see that you can add value to whatever they’re building so past experience will definitely make you stand out.

Scripting Experience scripting will help you out so much when you’re automating tasks or need to transform any data that you’re working with for an application.

Some scripting languages that are good to know are Bash and PowerShell but you can also use programming languages such as JavaScript or Python to automate tasks as well.

When working with AWS you’ll find out that scripting is used a lot in the industry to automate builds, configure instances, and to move data from one system to another.

Experience with Infrastructure as Code If you know how to code you’re at a unique advantage for AWS roles simply from the fact that you can write code to provision resources to the cloud by writing code.

A lot of engineers and administrators in the past would provision resources manually in the console but now with tools such as Terraform and AWS CDK you can deploy resources quickly by writing code.

So if you’re a developer that builds applications I highly recommend that you take the time to start learning how to provision your own resources by writing infrastructure as code.

AWS Certifications It’s great to have AWS certifications on your resume because it shows that you know how the different AWS services work and you’re aware of the best practices. AWS certs will make your resume stand out to potential employers.

To prepare for exams my process is pretty straightforward and works every time for passing. - Download the AWS exam objectives - Purchase acloudguru subscription and take exam course - Take practice exams on Whizlabs - Take notes and study AWS documentation - Take the exam

If you’re just starting in AWS I recommend taking the exams in this order: - AWS Cloud Practitioner - AWS Certified Developer - AWS Solutions Architect - Associate - AWS Certified Security Specialty

System Design System Design is the process of defining all the components, databases, APIs, and security measures you’ll have in place for your application.

System design should always be conducted before the development of any application so that you have a baseline to work with, never hop right into code map out everything first before performing any development.

Soft skills I hate to break it to you but you need to talk to people, you need to be able to communicate effectively and collaborate with others.

Soft skills is so important because you’ll be working with so many different people with different backgrounds, you need to be able to resolve conflicts and get your point across in a respectful manner.

Any training on emotional intelligence will definitely help you out significantly in this area and you’ll be able to dominate in this field.

Supreme Confidence You really have to have that mindset that no one is fucking with you and that you really get shit done. Supreme confidence goes a long way because companies are you going to pay you what you think you’re worth not what you’re really worth.

Walk into these interviews and show that you know what you’re talking about and you’re that guy/woman, show how them how intelligent you are and how you can add value to their organization.
