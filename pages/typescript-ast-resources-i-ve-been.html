Let's get a little more generic for a bit because ASTs are super important and fun for way more than just ts! In case you don't know what ASTs are, they're the tree-shaped /data structure/ that the compiler turns your source code into That tree is what gives meaning to text

Let's look at an example real quick The attached gif shows how to use the TS AST Viewer. Type some code and then explore the nodes in the tree Clicking within the code selects the corresponding node The right panel shows metadata for the selected node https://t.co/3QGCnrF76T https://t.co/wzEvsQuYii

It's this tree structure and metadata that gives meaning to our code! When the compiler parses that code into a syntax tree, it has all of the data it needs to understand that "greeting" is the Identifier of a const VariableDeclaration with a value of StringLiteral "'hello'"

Syntax trees are an important concept for developer experience (DX) because they're the basis for so much of the tooling that gives us a good DX Since it's code as data, tools can be built to analyze and modify that data to do different things

@tseslint parses code into an AST that it then uses to detect and report violations of the different linting rules we specify @babeljs parses code into an AST that it uses to support different features enabled by presets, plugins, transforms, etc.

The TypeScript Language Server utilizes the AST to power code actions, renaming utilities, refactor analysis, formatting, etc. And then VSCode, neovim and other apps use the Language Server to attach those features to their interfaces

@unifiedjs is an entire ecosystem of ASTs and tooling for content-related languages like markdown and html that make it easy to customize the output content: - `unist` defines a UNIversal Syntax Tree spec that other syntax trees in the ecosystem follow https://t.co/kW5Lo7NEvA

- `mdast` is a specification for a MarkDown Abstract Syntax Tree - `hast` is a specification for an Html Abstract Syntax Tree - `remark` handles plugins that use `mdast` to manipulate markdown - `rehype` handles plugins that use `hast` to manipulate html

Thanks to @unifiedjs and the many ASTs it defines, we're able to do really cool shit like use MDX, turn static code blocks into interactive, type-aware interfaces, generate page metadata from frontmatter, etc. Just an awesome ecosystem https://t.co/Kqz2oo73Ti
