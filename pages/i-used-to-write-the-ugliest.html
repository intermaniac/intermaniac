[USE FORMATTER] When you write CSS, you don't always give consistent new line gaps and indentations. which, in return... Makes your CSS messy, and this can confuse you a lot of times. So... Use code formatters like "prettier" or any other one. But... always use one!

[ADD COMMENTS] Not always necessary, but... Whenever you use any trick or anything that could possibly confuse you or anyone reading your code, add comments.

[USE SHORTHANDS] This one is VERY important. So, the first step is... Learn about all the shorthands in CSS then whenever you use properties related to them... Use their shorthands instead. https://t.co/i0T9D8Cmp1

[USE DRY PRINCIPLE] DRY = Don't Repeat Yourself. Aways try not to repeat any piece of code. If the navbar and a button have 4 properties in common... Write them all in a separate class and then add that class to both elements.

[AVOID "!important" TAG] Why should you avoid it? Simply because it'll break all the hierarchy rules in CSS. When you use this tag, that property will always get highest priority. And... If you try to overwrite it somewhere else, it won't work. And will create confusion.
