Arrow functions are the simplest way to represent a function expression in Javascript. Not only are they concise, but they are also easier to write and offer some advantages as well. Let's look at arrow functions in detail in this thread:

Let's first see why is it called 'arrow' function. If you see the simplest arrow function, with just one line of code in the function's body, it resembles the shape of an arrow. This is why it is called an arrow function. https://t.co/VmeWy5SmGm

Now, let's break it up into different parts and see what pieces is it made up of. So, if you look the the left hand side of the equal to sign of the image attached above, we've assigned the function expression to a constant called 'func'.

We use constants for arrow functions as they're widely used in functional programming. Then, between the equal to sign & the equal to and greater than sign, we have a pair of parentheses. That parentheses contains the argument require by the function.

- If there's no argument, just put empty parentheses like above - If there's one argument, just mention that with/without parentheses - For more than one parentheses, put them all inside the parentheses with a comma between them. All three cases have been illustrated here: https://t.co/2x9XWfI9hl

Lastly, we have the stuff at the right hand side of the greater than equal to sign. In here, we've the function body. Till now, we've seen how to declare the function and how to pass argument to the function. We'll now see where to manipulate that data and how to return it:

So after the greater than equal to sign comes the body of the function. Again there can be two cases: - body contains one statement - body contains multiple statements Based on it, the 'return' keyword and braces are either optional or mandatory as illustrated in the image: https://t.co/7XG8tCpMPw
