2/ VENDOR BENEFITS: + It is easier to control and secure a platform as a centralised instance - as opposed to controlling distributed/downloadable copies. +Intellectual Property (code) is less likely to be compromised (through reverse engineering).

3/ + Discovery of a major security vulnerability can be quickly fixed at the source (in the cloud). + As the product evolves into new versions, customers do not have to update/upgrade their software - it is all just contained in the cloud and managed by the vendor.

4/ CUSTOMER: + Avoid capex on purchasing a new server, and having a secure place to store it + Eliminate energy costs/consumption used to run a physical server. + No underlying operating system that needs to be maintained or supported (SaaS)

5/ + No operating system licence fees (Windows Server), or licence fees associated with other supporting software (such as enterprise anti-virus) (SaaS)

6/ +Less wastage of compute resources; a physical server has finite compute resources; this means that when a server is under-utilised the unused compute resources are wasted, and during busy periods compute resources are maxed out (resulting in performance bottlenecks).
