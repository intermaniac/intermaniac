What is CSS? CSS stands for Cascading Style Sheet. It is a stylesheet language that is used to selectively apply styles to HTML elements. And what is it made up of? CSS is made up of rules. And, what are CSS rules?

CSS rules are made up of selectors and a property-value pair. The selectors specify which HTML element we're targeting and the property-value pair specifies what is the property that we're changing (property) and what are we changing it to (value). See the attached image ðŸ‘‡ https://t.co/8gmtzez3gw

Let me explain selectors in a little more detail. Selectors are simply a way for us to refer to the HTML element to which the specified style should apply. And how do we specify those styles? By the property-value pair.

And how many types of selectors are there? Selectors can be of many types depending on how they're classified. Let's look at some common selectors ðŸ‘‡

1. Universal selector: Universal selector basically acts as 'select all' in CSS. Using it, we can select the entire HTML document. So, for example, if we want to increase font size of the entire page, we'll use this. https://t.co/sKgd2389gV

2. Class selector: Class selectors are selectors using which we can target an entire class of HTML elements. We can create custom classes as attributes in HTML tags and then target all elements of a particular class using this selector. https://t.co/wJXBMkncD3

3. ID selector: ID selectors are similar to class selectors and help us target the HTML elements with custom ID attribute set by us in the HTML document. One thing to note is that there can only be one ID of an element and an ID can't be repeated on a page, while classes can: https://t.co/f1Wp8R9bp4

4. Type selector: Type selectors are used to apply styles to all HTML elements of a kind. For example, say we want all our H1s to have a background of blue. We can do this simply by using the type selector as shown: https://t.co/Ex7nB0lUH9

We also have several other selectors like pseudo selectors, group selectors, and more like descendant selectors but the above-discussed ones are sufficient to get started with. Tomorrow, we'll discuss even more interesting things in CSS like specificity and cascade.
