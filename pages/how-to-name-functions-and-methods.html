Function names should be verbs. ‚ùå calculator(...), greeting(...), rotation(...), area(...) ‚úÖ calculate(...), greet(...), rotate(...), compute_area(...) This way you won't mistake them for variables. Also, you can read the code more naturally.

Function names should be precise. ‚ùå load_data(...), calculate(...) ‚úÖ load_sales_csv(...), calculate_daily_volume(...) This way you don't have to remember what your functions do. Your brain can focus on solving problems, rather than deciphering your own code.

In Python, by convention, function names should be lowercase and in snake_case. ‚ùå doNonPythonicThing(...) ‚ùå Do_non_Pythonic_thing(...) ‚úÖ do_pythonic_thing(...)

...but it's just a convention, and it's okay to break it if it makes sense: ‚úÖ crystalise_NaCl(...)

Class methods usually follow the same rules. ‚ùå Car.acceleration(...) <-- this could be an attribute, not a method ‚ùå Car.Accelerate(...) ‚úÖ Car.accelerate(...)

...but there's an interesting exception: properties. Properties are methods which pretend to be attributes. And attribute names should be nouns. https://t.co/uwzhsk3tKE
