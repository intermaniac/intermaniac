ðŸ§µ high leverage things on the horizon that need to be worked on: - social graph/contacts protocol - hoon generated UI of any flavor - git client for urbit or similar, esp with some kind of integration with groups (think github) - file serving from the runtime (until new mars)

social graph/contacts protocol @basileSportif has talked about this we need some relatively universal way to expose data about the relationships between ships that is inspectable by all agents. it would be nice if it was easy to persist parts to a global consensus layer too.

hoon generated UI one of urbit's powers is composability. we'll have some nods in this direction soon, however, true composability needs an ability to accept UI from other agents so that the place where composition happens isn't required to generate UI for every integration.

git client and github urbit is a dev's dream, if we had git + github, that would even further make this true. it would mean that devs could simply push there repo up and have it instantly deployed. there are literally a handful of SaaS's that do this as a business for $$$$$

file serving from the runtime pt 1 it's pretty embarrassing to rely on services like S3 when we're literally running a server already. so we need some way to serve files. we won't have new mars for a while (fully utilized ram + disk) so we have to do something until then.

file serving from the runtime pt 2 one thing we could do is serve files out of the put directory, straight from the runtime. this would require a decent amount of effort across the entire vertical of the stack, but it would enable a ton of applications that are held back rn

this is by no means exhaustive, lots of other high leverage things to do too. these are just top of mind for me. if I had the time I would do them all. each of these is a really important piece to unlocking urbit's true potential. if you want to work on any of these please DM
