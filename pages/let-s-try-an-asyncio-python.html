What is asyncio? It's a way to run python code concurrently, which gives us a way to write I/O-intensive code in a non-blocking manner. This is generally for networking requests such as HTTP and databases. File I/O isn't natively supported by OSes (thread for another day).

Example: I need to do laundry, dishes, and clean. I can do each of these chores serially (one at a time) which is how python code executes by default. Alternatively, I could switch between chores while the machines (washer, dryer, dishwasher, etc.) do some work for us.

I can start the laundry, then start the dishes, then clean while those are running. After a while, I can decide to take a break and move the laundry to the dryer, and put the dishes away. This is an example of concurrency which is what asyncio is in python.

Concurrency allows for more efficient use of CPU time. Instead of blocking the main thread while waiting on a network request or database response, that waiting can be done "in the background" and other code can be executed.

Parallelism is different and allows code to be executed simultaneously. In the previous example of chores, this would be 2 people doing chores at the same time. In Python, this is represented with processes.

Follow me for more #asyncio tips, tricks, and explanations!

Bonus: depending on the workload, asyncio/concurrency can be faster than parallelism! For CPU-intensive tasks that aren't I/O bound, such as large calculations, parallelism is the way to go. For anything using a network socket (and even subprocesses), asyncio is the way to go.
