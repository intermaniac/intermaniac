AWS SAM can do so many things for you when building serverless applications on AWS. ⚡️ It can be used to build, test and deploy Serverless applications on AWS quickly and efficiently. ⚙️ In this thread, you will find All AWS SAM CLI Commands & explanation ↓

Before getting started... Installing SAM CLI on Linux and Mac. $ brew tap aws/tap $ brew install aws-sam-cli If you use Windows, you can download the setup file.

$ sam init → Initializes the SAM project with a SAM template. The template contains everything you need to get started such as project structure, the template file, etc.

$ sam build → Builds a serverless application and prepares it for subsequent steps in your workflow, like locally testing the application or deploying it to the AWS Cloud.

$ sam local start-api → Allows you to run your serverless application locally for quick development and testing.

$ sam local invoke → Invokes a local AWS Lambda function once and quits after invocation completes.

$ sam local start-lambda → Enables you to programmatically invoke your Lambda function locally by using the AWS CLI or SDKs. This command starts a local endpoint that emulates AWS Lambda.

$ sam package → Packages an AWS SAM application. This command creates a .zip file of your code and dependencies and uploads the file to Amazon Simple Storage Service (Amazon S3)

$ sam deploy → Deploys an AWS SAM application.

$ sam publish → Publish an AWS SAM application to the AWS Serverless Application Repository. Takes a packaged AWS SAM template and publishes the application to the specified AWS Region.

$ sam sync → The `sync` command deploys your local changes to the AWS Cloud. Use `sync` to build, package, and deploy changes to your development environment as you iterate on your application.

$ sam pipeline init → This command generates a pipeline configuration file that your CI/CD system can use to deploy serverless applications using AWS SAM.

$ sam pipeline bootstrap → This command generates the required AWS infrastructure resources to connect to your CI/CD system.

$ sam validate → Verifies whether an AWS SAM template file is valid.

$ sam local generate-event → Generates sample payloads from different event sources, such as Amazon S3, Amazon API Gateway, and Amazon SNS.

$ sam delete → Deletes an AWS SAM application by deleting the AWS CloudFormation stack, the artifacts that were packaged and deployed to Amazon S3 and Amazon ECR, and the AWS SAM template file.

$ sam traces → Fetches AWS X-Ray traces in your AWS account in the AWS Region.

$ sam logs → Fetches logs that are generated by your Lambda function.
