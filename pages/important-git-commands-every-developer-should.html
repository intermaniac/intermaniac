10 important #git commands every #developer should know It's a #thread

1. git clone <https://name-of-the-repository-link> its a #command for downloading existing source code from a remote #repository. In other words, Git clone makes an identical copy of the latest version of a project in a repository and saves it to your #computer.

2. git branch <branch-name> By using branches, several #developers are able to work in parallel on the same project simultaneously. We can use the #git branch command to create, list and delete branches. git branch or git branch --list (list) git branch -d <name> (delete)

3. git checkout <name-of-your-branch> We use git checkout mostly for switching from one branch to another. We can also use it for checking out files and commits. This #cmd that allows you to create and switch to a branch at the same time: git checkout -b <name-of-your-branch>

4. git status The Git #status command gives us all the necessary information about the current #branch.

5. git add <file> or git add -A When we create, modify or delete a file, these changes will happen in our local and won't be included in the next commit (unless we change the config). We need to use the git add command to include the changes of a file(s) into our next commit.

6. git commit -m "commit message" This is maybe the most-used command of Git. Git commit is like setting a checkpoint in the dev process which you can go back to later if¬†needed. A short message is needed to explain what we have developed or changed in the source code.

7. git push <remote> <branch-name> After committing your changes, the next thing you want to do is send your changes to the remote server. Git push uploads your commits to the remote repository. Important: Git push only uploads changes that are committed.

8. git pull <remote> It's used to get updates from the remote repo and immediately apply the changes to local. This command is a combination of git fetch and git merge. NB: This operation may cause conflicts that you need to solve manually.

9. git revert <hash code> A safer way that we can undo our commits is by using git revert. To see our commit history, first we need to use git log‚Ää--‚Ääoneline Then we just need to specify the hash code next to our commit that we would like to undo

10. git merge <branch-name> Git merge basically integrates your feature branch with all of its commits back to the dev (or master) branch
