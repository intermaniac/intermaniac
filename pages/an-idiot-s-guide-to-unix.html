An idiot's guide to UNIX domain sockets I have been coding on UNIX for 27 years. Let me share some facts about UNIX domain of local sockets that you perhaps have always wondered about. ðŸ§µ

1. a UNIX socket is not visible outside the local machine or the file system A UNIX socket is just a simple UNIX file. You can read and write to it just like you can read or write files. But the socket can be written into or read from using any process running on same machine.

2. You can start a UNIX or local socket like this: $ nc -l -U /tmp/sock and you can send data like this. $ echo hi | nc -U /tmp/sock It is that simple.

3. It is a very simple IPC mechanism for unrelated processes to communicate seamlessly One of the best IPC methods in the UNIX universe, UNIX or local sockets are used by databases like postgres and several network protocol servers to implement their command interface.

4. You obtain statistics about operation of a server(daemon) by querying the socket You are not aware of this , but this is what happens behind the scenes. You can theoretically implement a different method to respond to questions about state but this is most common.

5. UNIX sockets can't be attacked by outsiders Since they are not visible to attackers across other machines or other networks on the Internet they do not pose a security threat.

6. It is better and easy to use than other IPC schemes Compared to other inter process tools this one is very mature and friendly and common, also widely used. Moreover the semantics are oddly similar to TCP or UDP sockets.

That is a wrap. TL;DR 1. Only within a host/node 2. nc example 3. simple IPC 4. stats from servers 5. not visible to attackers 6. easy to use Follow @girish1729 for network protocols lowdowns and UNIX topics.
