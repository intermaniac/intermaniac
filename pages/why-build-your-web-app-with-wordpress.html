Don't use WP if... 2. Flexibility Is NOT Important to You One of the primary benefits of WP is its flexibility (more later), but if you need an app that is rock solid, does just one thing really well, and is unlikely to need to change much... consider something other than WP.

Don't use WP if... 3. Your App Needs to Be Highly Real Time At it's base, WP uses PHP to render content and serve it based on the URL requested, aka a typical web site. Great for blogs (and a lot of apps too), but if you need lots of real time updates, there are better tools.

REACT was built for highly real time UIs, and WordPress itself uses REACT for its editor. Many plugins use REACT. But if you're just going to use WP to just load REACT, that's probably a waste.

Don't use WP if... 4. You Plan to License or Sell Your Site's Technology WordPress uses the GPL, which has restrictions on how you distribute software that you build with it. Namely, you cannot restrict what people do with your software once you sell or distribute it.

The GPL is a complicated topic, but the basic idea is if you are only selling or giving away *access* to your application, you won‚Äôt need to worry. However, if you are selling or distributing the underlying source code of your app, the GPL will apply to the code you distribute.

Alright, if you're still here, here are 6 reasons to use WordPress to build your web app. If these land with you, you should use WP and be confident in your decision.

Use WP if... 1. You Are Already Using WordPress The #1 reason to use WP for your app is if you are already using WP for your site, you are already a good WordPress developer, or your team already knows how to manage a WordPress site. Build with what you know.

Use WP because... 2. Content Management Is Easy with WordPress This is the use case even detractors concede. If your web app focuses on bits of content, the Custom Post Types API for WordPress makes it easy to quickly set up and manage this custom content.

WordPress was built with content in mind. Features, plugins, and systems are already available and vetted to help you edit, publish, restrict, and moderate content.

Use WP because... 3. User Management Is Easy and Secure with WordPress With WP, you start with secure logins and authentication. Further, the Roles and Capabilities system in WordPress is extensible and allows you to control what actions are available for different users.

Use WP because... 4. There are Over 60k Plugins There are plugins for everything. When that next new social media site takes off, there will be a plugin for it. Some plugins are essential. Some are ready to use. Some you will want to fork and fix up.

For almost any feature, integration, or bit of functionality you need, there is likely an existing plugin that will give you a head start. #WordPress

Use WP if... 5. Flexibility is Important WP is built on top of PHP which is used on all kinds of sites and apps, from the smallest websites to the biggest. It's all just code running on servers and browsers. Any tool for a specific piece of your app can be bolted into WP.
