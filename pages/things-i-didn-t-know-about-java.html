First, what is an inner class? An inner class is a nested class that is non-static. In the picture, `NotInnerClass` is *not* an inner class because, even though it is nested (i.e. declared inside `Outer`), it is declared with the `static` modifier. https://t.co/tWYATwW6Aq

An inner class instance *must* be associated to an instance of its enclosing class. In other words, to create an instance of an inner class, we must first have an instance of the outer class. To illustrate, let's declare the Outer/Inner duo in a jshell session. https://t.co/eJ3ejXjdpr

Next, let's try to create an `Inner` instance as if it were a static nested class. We can't. The error message says we must provide an instance of the `Outer` class. https://t.co/QRI1az6a5m

So let's do that. First we create an instance of `Outer`. And from the outer instance, we create an instance of `Inner`. Notice the syntax to create the `Inner` instance. https://t.co/F9prAFi6O2

We could have also written it in a one-liner, like in the picture. The takeaway here is: - in order to have an instance of `Inner` we must start with an existing instance of `Outer`. https://t.co/vhrU1GOyqX

What if we need to create a subclass of `Inner`? Say we need a class `InnerChild` that extends `Inner`. An initial (and wrong) attempt would be to do like in the picture. https://t.co/I5eEtAeQ7W

An instance of `InnerChild` is also an instance of `Inner`. Therefore, an instance of `InnerChild` must be associated to an instance of `Outer`. That's why the previous example failed to compile: it needs an instance of `Outer`. One way to fix is illustrated in the picture. https://t.co/UQPJLoPzBm

The previous is an example of 'Qualified Superclass Construction Invocation'. Note that `InnerChild` is a top-level 'static' class. In other words, we can instantiate it directly. In its constructor, a new `Outer` is created and the `Inner` constructor is immediately invoked. https://t.co/1UliIwQFRl

An alternative form of the qualified superclass constructor invocation is shown in the picture. In this form an instance of `Outer` is directly passed to the `InnerChild` constructor. The superclass constructor, i.e `Inner`, is then immediately invoked. https://t.co/jqe63ltU4W

To better understand both forms of the qualified superclass constructor invocation, let's redefine our classes. https://t.co/NnzBY3PtwE
