My goal for this weekend is to create a fully automated tweet. i'm starting with some python knowledge and a little experience with API requests, but i believe the hardest part for me will be tackling the automation of oAuth. follow along on my weekend project. ðŸ§µ

Once you have your Keys and Tokens, there are several paths to get started. Since my goal is to send a Tweet, let's take a look at the documentation for managing Tweets: https://t.co/87dXljn0JM https://t.co/3hoLLvauvH

Twitter offers a Graphical User Interface (GUI) for their APIs via the the API Explorer or API Playground (Live Request). These GUIs can help us understand the API flow before we start writing code. I sent the attached Tweet using the API Playground https://t.co/8t6aON0hIn https://t.co/TeHSERaMUu

There are three main parts to this example Tweet. 1) The Endpoint. They serve as the recipient of your request, and Twitter has separated theirs by function. Since I want to upload a Tweet, I use POST /2/Tweets. Check out the other Endpoints: https://t.co/YkptQ4V3E1

2) Request Body. This is where I put the data that I want Twitter to act on. The data is JSON formatted and for a simple Tweet, I only need to supply one key-value pair. The key: "text" is what I am tweeting and the value "This tweet was sent ..." is the value.

3) Response. This is how the Twitter API reacts to my request. Usually the response will let you know if your request succeeded or failed and may provide some additional data. In my example, an additional "id" key-value was given to me. This is the unique id for my tweet.

Here is an example of getting an error due to formatting. I removed the double quotes from the VALUE in the request. Doing so and running the request will give the following error. https://t.co/jMrpfMg6Fx

Now that I know tweeting via API is possible, I want to create my own python script to do it. First, I'm going to try using the example code from Twitter on their GitHub page https://t.co/zNkjL81cpY

I put in my API key and secret, and set my tweet to "test #3". I run the script, but instead of the tweet just going through... it wants me to further Authorize this request. https://t.co/qzk5RIRvob

I'm not very familiar with this process, but this is called oAuth and as stated in the beginning, this will be my biggest hurdle. Remember, I want this to be fully automatic. But this example script is wanting me to go to a site, get a PIN, and put it back in the terminal. Boo

when i go to the website, I have to manually click "authorize app" and then the PIN is revealed. https://t.co/DlU9LykGzG

I copy and paste the PIN into my terminal and magic happens and the Tweet gets posted. https://t.co/Fdmoi6VVXJ https://t.co/ldjwj8E19w

I need to solve two things at this point. 1) Automate this process so I don't have to click on anything or copy/paste PINS 2) Dive deeper into what is being done so I can understand what is going on. (note, this should be a by-product of completing task #1)

There's a bunch of hoops my tweet has to jump through. I may be able to automate these jumps, which may end up being janky code, but i don't mind. I start by reverse-engineering the final call made and i add a line of code to output my 'access token' and 'access token secret' https://t.co/PwwQJp7rbe

i copy and paste the 'access token' and 'access token secret' and comment out all of the code that was used to find them. i then declare these variables directly and run my script. it worked! https://t.co/3Sh2iUTEYM https://t.co/TmblTC28cy

it's likely possible to create a semi automated tweet now. i've skipped jumping through all of the hoops, but i believe the 'access token' and 'access token secret' have a two hour expiration. so, once they expire, i'll need to go through the manual process again.

Okay... I may have gotten lucky. It appears the access tokens do not expire. This makes sense because of all the steps involved in getting them. I am going to proceed with a little bit more oAuth education, but will assume that I don't have to renew my tokens. https://t.co/rpYbIZPl8J

Success! My first fully automated tweet is attached below. I thought it was going to take me longer to get to this step, and as long as my access token doesn't expire as Twitter says it shouldn't, then everything should be good to go. https://t.co/9tTcR8zTuq

this is the final code. pretty simple, right? i just need my API keys from my dashboard and the Access Tokens from my earlier authentication. I transferred this code to my Raspberry Pi and set it to automatically run via crontab. I scheduled it to run 40 minutes after the hour https://t.co/WZF6PuTreH

now that i have basic tweeting working, I'll be looking at some of the other API functionality and working on a bot that will show availability for Flipper Zero products

this project was highly inspired by @rpilocator. If you're looking to pick up a raspberry pi, follow that account, because it tweets when a unit becomes available!
