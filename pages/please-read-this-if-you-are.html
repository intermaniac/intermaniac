Please read this thread if you are a beginner or novice who is not yet fluent in Python and are wondering how you should plan your path.

First off, decide what you want to code long term. Do you want to develop web apps? Do you want to work in AI? It's ok to have multiple interests, but if you're learning python for a career you should pick a specific field to start out with. Don't jump around to various fields.

Second, and I must emphasize this, do not think that becoming fluent in Python will allow you to just code whatever you want all by itself. For example, you can't code your own ciphers if you don't know cryptography. You can't code networking tools if you don't know networking.

This fact is something that most noobs don't understand, and non-coders simply don't get. If a person can code new ciphers and make flask applications, that person knows A LOT. Like, a lot a lot. Because in order to be able to do that, you need to know multiple fields.

Cryptography is a field of its own, and it takes time to learn. 3D Graphics design is a field of its own, and it takes time to learn. Forensics is a field of its own, and it takes time to learn. Security is a field of its own, and it takes time to learn. Seriously.

This is where I see some new people messing up, they think that by becoming fluent in python they'll be able to write just anything without having any knowledge in its respective field. After they learn python, they tend to have a "what do I do now" feeling.

Ok, so you've decided your long term study/career field, and now you're studying python and those fields. Be sure to study the packages affiliate with that field, find out what exists and what doesn't. The Python package index is massive, so most of what you need already exists.

Now you need to do something too many aren't doing. Practice, practice, practice. I see way too many people who are only reading books, or only watching youtube, or only reading articles/guides.

When you read a book and see a code example, TYPE the code out! If you type it out, you're more likely to remember it. For some reason when you read a code example, it tends to make sense, and then when you go get a soda, you've forgotten it by the time you're back at the laptop

To clarify, I'm not against people reading books and watching youtube videos etc, but you should be combining that with PRACTICING, otherwise you become a copy/paste coder who relies on stackoverflow. For simple stuff that copy/paste works, not for developing in production.

After you practice, gain experience by writing a new project or a new program. Create something new. Since it will be new and original, this FORCES you to exercise mental faculties you'll need as a developer, it'll be something you can add on your github, and it'll be YOURS

You won't be able to just copy/paste someone else's program when you make something new. You might find bits and pieces, but the code you need for your own unique project will have to come from you.

Repeat this process of learning, practicing, coding. Do this again and again and again. After a while, you'll be able to read any python code, and know what it means (fluent). You'll also become experienced with the packages and knowledge of your field.

Hope this helps, if it didn't please comment or ask a question. If it did help, please like and say what part helped you.
