How to push your personal project to GitHub using Git commands? ðŸ”¥ ðŸ§µðŸ‘‡ #webdeveloper #git #opensource

First, make sure you have a repository created on GitHub that you want to push your project to. Open your terminal and navigate to the local directory where your project is stored.

Initialize the directory as a Git repository by running `git init` This sets up the local repository and prepares it to track changes to your project files.

Add all of your project files to the staging area by running git add . .The staging area is where you prepare files to be committed to the repository. The . tells Git to add all files in the current directory and its subdirectories.

Commit the files to your local repository by running `git commit -m "Initial commit"` The `-m` flag allows you to add a commit message, which should be a brief description of the changes you're committing.

Add the URL of your GitHub repository as a remote by running git remote add origin github[].com/USERNAME/REPOSITORY.git, replacing USERNAME and REPOSITORY with your own GitHub username and repository name. A remote is a connection to a remote repository.

Push the committed files to your GitHub repo by running git push -u origin master The -u flag sets the upstream branch, which links your local repository to the remote repository. The origin is the name of the remote, and master is the name of the branch you want to push to.
