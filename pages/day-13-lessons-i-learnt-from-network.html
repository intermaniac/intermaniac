Day 13 - lessons I learnt from "Network penetration testing & ethical h*cking" LIVE training organized by: @thecyberzeel #spinthehack • Compromising SQL Servers and Password Hashes: - performing lateral movement ० check out the following thread: #infosec #cybersecurity

• In previous phases of INPT, we discovered a lot of hosts. We also found Database servers (like MSSQL). • Now, we need to compromise this database server and move laterally towards the main Windows machine (target).

• We can do this with the help of 'Stored Procedure' - a kind of backdoor, that allows us to move from DB server towards the Main asset.

• We already know the ID/password of the MSSQL server (as we cracked it earlier). • Now, login to this server using a mssql_login auxiliary of metasploit.

• Next, use mssql_enum auxiliary to find/locate those 'Stored Procedures' or 'sp'. • You'll notice that there is an executable procedure called: xp_cmdshell that is 'Not Enabled'.

• to enable it, make the use of mssql_cli. • try to connect to the target through this CLI (Command Line Interface). • after connecting, configure the advance options. • also reconfigure this change made in the file that stores the changes.

• Now, it's time to enable the 'xp_cmdshell' using the following command: sp_configure 'xp_cmdshell', '1' • again reconfigure this change made in the file that stores the changes.

• Now, go back to metasploit and run the mssql_enum auxiliary again. • You'll see that xp_cmdshell is enabled now. Therefore, you can execute commands on mssql server.

• now, you've successfully performed the lateral movement towards the main Windows machine from the MSSQL server. • to check your identity in mssql server, execute the following command: exec master..xp_cmdshell 'whoami' - Output: nt service\mssql$sqlexpress

• to check your privilege: exec master..xp_cmdshell 'net localgroup Administrators'

• Till now, you've successfully moved laterally from the MSSQL server towards the main Windows machine. • You're ready to crack into the SAM database of the Windows machine.

• But, the passwords are stored in hashed format. - We'll look into what to do about these hashed passwords in next phases of INPT.

• Well, this Sunday was a funday, but more of a day with lot of learnings. So, all thanks to: @thecyberzeel
