Ease of coding: Arguably Java is easier to start coding and setup, so beginners can find it easier to start with. This factor becomes irrelevant as u become more experienced. A win for java 2/n

Inherent Performance: C++ is compiled with no runtime baggage, Java is compiled to bytecode and then is run by JVM, with a garbage collector and garbage collectors are slow That's why C++ wins in inherent Performance (there are many more reasons tho) 3/n

Library Support: Since in CP platforms we can't use external libraries, we'll have to compare inbuilt ones. Fortunately, both have a strong set of features. C++ -> STL Java -> Collections 4/n

STL is richer in features than Java Collections. It even has Policy-Based Data Structures which are extremely useful at mid levels in CP Both languages have fair share of pitfalls like Arrays.sort() being O(N^2) worst case in java for primitive types 5/n

and unordered_map being O(N) worst case look up in C++. Looking objectively, C++ wins this as well 6/n

Community: Most people use c++ in CP community by a huge margin, so getting help, finding reference codes etc is easier in c++. Although less , Java still has many high rated coders. While Objectively C++ wins, if you love Java, Too bad, facts don't care about your feelings 7/n

Wider community: While C++ is used a lot in the CP community, in the wider community, Java is more widely used, so finding non-CP-focused answers should be easier in java. Another win for java 8/n

Coding Speed: While C++ does allow for macros to shorten code length, similar is possible in java by making a bunch of functions. And Coding speed really depends on the individual coding style, so it's a matter of preference. Draw here 8/n
