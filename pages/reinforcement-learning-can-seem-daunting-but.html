1/ First, why should we care? #ReinforcementLearning can learn complex behaviour - it has seen huge breakthroughs in recent years. An RL algorithm, #AlphaGo, famously beat the world champion in the game of Go. Thereâ€™s a movie about it, free on YouTube: https://t.co/7QUTmzKNmi

2/ This is impacting the real world. Here is OpenAIs RL trained robot arm solving a Rubik's Cube: https://t.co/TYH35qJE39

3/ How does it work? By interacting with the environment This feels intuitive. When an infant plays or waves its arms, it has no teacher. However, it interacts with its environment by using its senses (e.g. vision, hearing) and taking actions (e.g. moving, speaking)

4/ This connection produces a wealth of information about: - Cause and effect - The consequences of actions - What to do in order to achieve goals.

5/ Reinforcement Learning (RL) refers to both a class of problems & set of solutions: RL problems are sequential decision-making problems. RL solutions are computational approaches to learning from interactions with an environment that solve sequential decision-making problems

6/ The technical name for the set of problems solved by RL is Markov Decision Processes They are discrete-time processes. This means time progresses in discrete steps. We can compare discrete (in red) and continuous time (in gray) in the diagram https://t.co/WpVbTVabuw

7/ At each timestep t: 1. The agent takes an action, a 2. The state, s, updates as a result of this action 3. And a reward, r, is given Each are defined in the image below: https://t.co/rLCWoNAxxY

8/ Diagramatically, this diagram from Sutton and Barto's book summarises the an RL MDP Let's delve into each of the state, action and reward next https://t.co/C43ghp0Bkl

9/ STATE The key aspect of the state in a Markov Decision Process is that it is 'memoryless'. This means that the current state fully summarises the environment. You gain no information about what will happen in the future by knowing preceding states (or actions).

10/ ACTION These are the choices the agent can make. At each timestep, the agent takes one action. Action can have multiple independent dimensions. E.g. a robot can decide its acceleration/deceleration independent from deciding its change in turning angle.

11/ REWARD Defines the goal of a reinforcement learning problem. On each timestep, the environment sends a single number called the reward to the reinforcement learning agent. The agentâ€™s objective is to maximize the total reward it receives over the long run.

12/ For the last bit of terminology, let's look at the POLICY A policy is a function that maps from a state to an action to be taken when in that state. The policy is the core of a reinforcement learning agent in the sense that it determines the agent's behavior.

13/ The Optimal Policy is the mapping from state to action that gets the highest possible reward over time. It can be represented in different ways: a table, a dictionary, a neural network - or many others The images contain an MDP and it's optimal policy (written in Python) https://t.co/XCqlzLg8en
