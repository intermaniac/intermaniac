üç≠ Lexical Scope: üí° Similarly: - A function that is lexically associated with another function, can access the scope of the parent function. - In other words, child function(Inner fun) can get access to their parent functions variable. üö© BUT vice-versa is not true!!!!

- The parent function can't get access to its child function(Inner function) variables. - Or, the inner function's variables are inaccessible to the parent function.

üç≠ Eg 1: - I've created 2 variables called "global" but their scope is different. - var global = 69; -> Global scope - var global = 6969; -> Function Scope - The "lexical scope" of the test( ) function is the global scope which means it can get access to the global variable.

- If we run this function then it tries to find the "global" variable in the local scope first. - JavaScript found a variable called "global" in the local scope, so the output will be 6969. - If it is not found, then the JS engine tries to look up its lexical(parent) scope. https://t.co/D7d2HifV5h

üç≠ Eg 2: - In this example, I've created 4 functions that are a(), b(), c(), and d(), and one variable that is "val" has global scope. - Inside the d() function call the value of "val" variable.

üí° JS engine lookup: 1st - Inside the d() function(Locally) 2nd - If not found inside the d() function then the lexical scope of d() function that is c() function. 3rd - If not found inside the c() function then the lexical scope of c() function that is b() function.

And so on... - The lexical scope of a() function is the global scope. üç≠ This type of behavior for looking at variables are known as "variable lookup" in JavaScipt. https://t.co/RbeZSur8UK
