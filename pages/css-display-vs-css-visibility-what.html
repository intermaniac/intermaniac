The CSS display property is one of the most powerful and useful properties in all the CSS. It can be very useful for creating web pages that looks in a different way, but still follow the web standards.

Let us describes the most commonly used CSS display values. ¬ª Block ¬ª Inline ¬ª Inline-Block ¬ª None

Display BLOCK The block value of the display property forces an element to behave like block-level element, like a <div> or <span> element. The style rules in the following example displays the <a> and <span> elements as block-level elements. https://t.co/jUAAHcjW1w

Changing the display type of an element only changes the display behavior of an element, NOT the type of element it is. For example, an inline element set to display: block; is not allowed to have a block element nested inside of it.

Display INLINE The inline value of the display property causes an element to behave as though it were an inline-level element, like an <a> or a <span> element. The style rules in the following example displays the <p> and <li> elements as inline-level elements. https://t.co/fUxyAaVZ0C

Display INLINE-BLOCK The inline-block value of the display property causes an element to generate a block box that will be flowed with surrounding content i.e. in the same line as adjacent content. The following style rules displays the <div> and <span> elements as inline-block. https://t.co/G1HSIk7FMo

Display NONE The value none causes an element to generate no boxes at all. Child elements do not generate any boxes either, even if their display property is set to something other than none. The document is rendered as though the element did not exist in the document tree. https://t.co/hiPUJ2mP52

The value none for the display property does not create an invisible box ‚Äî it creates no box at all. See the live demo given inside visibility vs display section.

CSS VISIBILITY The visibility property determines whether an element is visible or hidden.

We can use the visibility property to control whether an element is visible or not. This property can take one of the following values below:

VISIBLE Default value. The box and its contents are visible. HIDDEN The box and its content are invisible, but still affect the layout of the page.

COLLAPSE This value causes the entire row or column to be removed from the display. This value is used for row, row group, column, and column group elements.

INHERIT indicates that the value of the visibility property should be inherited from the parent element i.e. takes the same visibility value as specified for its parent.

The style rule visibility: collapse; however removes the internal table elements, but it does not affect the layout of the table in any other way. The space normally occupied by the table elements will be filled by the subsequent siblings.

If the style rule visibility: collapse; is set for other elements rather than the table elements, it causes the same behavior as hidden.

CSS Visibility vs Display The display and visibility CSS properties appear to be the same thing, but they are in fact quite different and often confuse those new to web development.

visibility: hidden; hides the element, but it still takes up space in the layout. Child element of a hidden box will be visible if their visibility is set to visible.

display: none; turns off the display and removes the element completely from the document. It does not take up any space, even though the HTML for it is still in the source code.

All child elements also have their display turned off, even if their display property is set to something other than none.
