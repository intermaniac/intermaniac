1. Var Keywords - Variables were previously declared using "var" which had function scope and were hoisted to the top within its scope. It means that a variable can be used before declaration.

2. Let Keyword - The "let" variables and constants have block scope which is surrounded by curly-braces "{}", they are not hoisted & cannot be used before declaration. Example: https://t.co/qtD6ZGvLji

3. Const Keyword - The new const keyword makes it possible to define constants. Constants are read-only, you cannot reassign new values to them. Example: https://t.co/Da8wp3ZbqI

4.1 Arrow Functions - It provides a more concise syntax for writing function expressions by removing the "function" and "return" keywords. - Arrow functions are defined using the fat arrow (=>) notation. - Unlike ordinary functions, it does not have their own this keyword.

4.2 - The value of this inside an arrow function is always bound to the value of this in the closest non-arrow function. - Arrow functions are not hoisted. They must be defined before they are used. Example: https://t.co/USilYt4Eas

5. Multi-line Strings Users can create multi-line strings by using back-ticks(`). In ES5 we needed to use '\n' for multi line statements. Example: https://t.co/M1v9UTUDd8

6. Template Literals ES6 introduces very simple string templates along with placeholders for the variables. The syntax for using the string template is ${PARAMETER} and is used inside of the back-ticked string. Example: https://t.co/dPYfd4N15R

7 . Default Parameters ES6 allows function parameters to have default values. But, in ES5, OR operator had to be used. Example: https://t.co/tOvVqNfhBD
