I'm going to assume that the reader already has the basics in place so I'm not discussing things like why we use 'let' to declare variables in place of 'var' and the like. This is geared more towards people who have basic proficiency in the language.

- Playing with Arrays: One should be able to loop over an array and run callback functions for each individual item. Things like forEach() should be under firm grasp. Also, knowing immutability and why we shouldn't use .push() to add items to an array should be known.

- Array methods like map, filter, reduce and restructuring should be great to have. They come in really handy in React and knowing them makes the journey of learning React seem easier. At least that is how it was for me.

- Objects: Just like Arrays, one aspiring to learn React should also have good knowledge of Objects and manipulating data of an object. Things like Object literals, dot and square bracket notation with limitations should make things feel natural in the React world.

- Functions: A core concept of Javascript, it is not only necessary for learning React but for every other language too. It is one of the fundamental concepts & having a good grasp of it will help a lot in learning React. Also, ensure that you know how to write arrow functions!

Also, make sure that you know function expressions and how to write those in concise form using arrow functions. A lot of React code uses this and to be honest, it is just better!

- Classes: If you're going to work with legacy React code, then it is a must-have. However, if you're sure that you'll only work on modern apps then functional programming should be what you should focus on. Having this in your arsenal certainly helps but it isn't mandatory!

Also, if you're going to learn classes too, make sure that you know the 'this' keyword well. How it behaves under different circumstances should be known to you, in addition to 'bind' and how calling a method using a reference makes 'this' useless and how would you tackle that.

Of course, with everything I've said above, it doesn't mean that only when you completely know the mentioned topics can you learn React. It is rather meant to be a brief listicle of topics you should be familiar with.

You can also learn all these JS topics while learning React simultaneously but it'd be way easier to first brush up on your Javascript skills and then jump into the React world!
