💥Watch me build a micro-SaaS from scratch live in this thread this weekend💥 ⏰ 24 hours 🎯 1 feature 📝 Step-by-step instructions Tech stack: ⚛️ ReactJS 🐍 Python 🦾 GPT-3 API ☁️ AWS - 🤖 Lambda (serverless) - 📩 SES - 💾 DynamoDB (noSQL) The problem ⬇️

Problem 🤔 Last week I wanted a sweet cocktail, but I only had 2 ingredients at home: - vodka - champagne How do I come up with a tasty cocktail using these ingredients? 🍹

Idea 💡 An AI bartender that gives you cocktail recipes according to your input. Want a sweet drink with vodka as base? No problem Want a sour drink + proper cocktail name for your next dinner part? I got you fam Want a fizzy drink from 4 ingredients? Say no more Mockup ⬇️ https://t.co/cLxa7V2NOF

Let's come up with a fun name for our new cocktail business! I’ll use the free business name generator @NAMELIX2 I'm thinking something playful that includes the keywords: cocktail, bartender, drinks https://t.co/plD91Tewzh

I decided to go with the name BoozyBlend 🍹 ✅ sounds like a brand ✅ has an available •com domain https://t.co/EewyY360BN

First things first, securing the domain then creating a logo 🫡 I'm (overly) obsessed with •com domains and colorful logos. I don’t have time to design a new logo so I’ll go with a text logo from @LogoCorn (shameless plug) https://t.co/XAXrTRP1Jo

Any favorites so far? Poll in the next tweet 🗳 https://t.co/SemuWcgqoh

Help me pick a logo 🗳

While the domain is getting registered, let’s establish anti-goals. In short, anti-goals are things you DON’T want to happen. Final outcomes and along the way. What can you do to prevent the worst possible outcome? Here’s a great thread on the subject: https://t.co/1i72bA7hm1

Establish anti-goals: 1. Ask yourself: What does the worst possible outcome look like? For me, who hates maintenance, that would be: ❌ Manual content delivery ❌ Manual but repetitive tasks

2. Then ask yourself: What actions or systems would lead to that worst possible outcome? For me: ❌ Manually creating recipes ❌ Emailing cocktail recipes to users manually

3. Establish anti-goals ✋ ❌ Manually creating recipes ✅ Fully end-to-end automated cocktail recipe process ❌ Delay in delivering recipes ✅ Generate and send recipe to users immediately https://t.co/lZy9fBwjRg

Alrighty, we’re ready to generate the backend. I have a Python script that saves me 2h every time I need a new backend. I used to create every resource manually. Then I switched to full infrastructure automation, but it still took ~1 hour to add resources, deploy + debug

I finally wrote a script that generates ALL the code. Some of the most tedious tasks are now fully automated: - Generating YAML files - Creating repos + branches(Github API) - Creating CI/CD pipelines(Github actions) - Creating pull requests - Merging PRs Went from 2h➡️~10min https://t.co/pNczjeifSl

Essentially I built a simple landing page where I’m first picking the resources. Then using the GitHub API + CLI to create repos/branches/pr/merge. All with 19 (!) of Python scripts to orchestrate everything. I finally wrote a complete walkthrough ⬇️ https://t.co/0LtBFtDAAW
