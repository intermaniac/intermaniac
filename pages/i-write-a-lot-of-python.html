Input Validation is important. It deters against several vulns, ranging from Injection to XXE, etc. I like using Pydantic to get this done. I map the expected request POST/JSON to a Pydantic Model with validators and I am off to the races https://t.co/ECo4hdxW5q 2/

AuthZ is a big API security concern. I love using @casbinHQ to address this comprehensively. I use the pycasbin wrapper library to do this and use the appropriate Database API to store AuthZ policies https://t.co/xZdymu7MJO 3/

"Using crypto in your app shouldn't feel like juggling chainsaws in the dark" is 100% on point from the creators of my fave crypto library, Tink. Tink is secure-by-default crypto that now has a python version. Its great for encryption https://t.co/0tWPf4kYTQ 4/

You really shouldn't be handling and managing users and passwords. But if you have to, you should be using passlib. It gives you API for multiple key-stretching algos that you can use to store and verify passwords. It even gives you TOTP utils for MFA https://t.co/55msxlXifh 5/

I hate python's default logging module. I like loguru instead. Its super-easy to work with. I especially love it for security logging 'cause I can define a SECURITY log level and structured logging in seconds. https://t.co/JgsKw89dIc
