Having security vulnerabilities in #WordPress plugins is common. Many of them are low severity issues which are never exploited. If you are using a plugin which has become vulnerable, look at how the developer is approaching it. Here's 3 simple things to look at üßµüëáüèª

1) Is the developer active and is the issue fixed? - Quite often, plugins remain unpatched. This can happen for different reasons (discontinued, poor management, etc.). Keep an eye on how quickly the developer fixes the issue. Is the plugin removed from repo? Is dev ignoring it?

2) Is the developer open and transparent? - Unfortunately, many plugin devs see security fixes as something bad. Some may even try to hide the fact that there was a security issue at all. Great devs notify their users and recommend to update (even if the issue has low severity)

3) Does the plugin have vuln. disclosure program? - It's a great sign if a plugin developer has thought about security proactively. Some plugin devs have clear guidelines around how to report security issues to them, and some even reward those who do.
