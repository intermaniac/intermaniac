#JavaScript can be very weird sometimes because you'll have something like = but there's also something like == and === so what exactly is the difference between them??? In this thread I'll tell you the differences between them with some examples üìç #Thread #programmers

1. The "=" Unlike the normal equal to in maths this is known as an assignment operator in JavaScript because it is used to assign a value to a variable for example : Var x = 10 This means the value of 10 is assigned to the variable x but if we type something like: 10 = 10

This will result to a reference error as there is no variable that can be called "10"

2. The "==" This is called an equality operator in JavaScript and it is used to check if two values are equal *without* considering their data types because if the two operands are not of the same data type then the operator (==) will convert any of them into the other operand's

value and then compares their values. E.g Let x = 10 // this is a number data type Let y ='10' // this is a string data type Console.log(x == y) // output: true The (==) converted y to a number data type.

3. The "===" This is known as the strict equality operator and from the name I think you can take a guess on what this (===) operator does. The (===) operator is used to check if two values are equal *while* considering their data type that means if we have something like this:

Let x = '10' Let y = 10 Console.log(x === y) //Output: false We will get false because they are not of the same data type. It 's just like in an exam if the correct answer was 50 but you wrote 50.1 take the (==) as a kind teacher who'll pity you and give you a pass but the (===)

as a strict teacher who'll mark your answer wrong. I hope you get my analogy

In summary the = is an assignment operator used for assigning a value to a variable while the == operator is used to compare 2 variables without considering their data type while the === operator is used for comparing data as well but while considering their data type.
