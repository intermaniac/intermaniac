not able to reconcile stale state w/out full page reload should b a deal breaker for web apps in 2022. see @figma @Superhuman @getpostman unless you are abs certain that your biz needs are more doc than app.

most important is launch cadence. for new proj, full stack TS is quite appealing. offload backend heavy logic to lambda + edge + (react/svelte). cache data locally for fast, offline friendly UX.

more complicated decision for legacy apps. 1. decide if Doc or App 2. if doc, use templates, same lang. cache on edge. minimize state transition via design. simplify UI. 3. if app, refactor portion to TS (react/svelte). calc time to refactor entire app. assess ROI

there is no perfect to handle state transitions - moving it to server invites diff set of trade offs. 1 of reason React was created - displaying correct unread msg count on Fb messenger. rmb - all server bound renders assume user has fast + STABLE network.
