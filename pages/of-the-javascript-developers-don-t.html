I'm sure you must have used regular and arrow functions in JavaScript. Let me begin by asking a question. Do you think the arrow function is simply a more concise way of writing JavaScript functions?

You're right about that but there is more to it other than writing the function in short form. Arrow and regular functions differ significantly, and we cannot replace one for the other.

An arrow function does not care who calls it, while the regular function very much cares who the caller is. Regular function always binds the value of this keyword with its caller.

However, the arrow function does not care who calls it, it will close over the value of 'this' keyword of its scope at the time it was defined. Arrow functions give an easy access to it defining environment.

Let's understand this with an example. In any node module, the top level 'this' keyword is associated with its export object. I've created an object of a Regular and an Arrow function. https://t.co/nrneew3P3X

When a regular function is called, the 'this' keyword is associated with the caller which in this case is "regularFunction."

When the arrow function is called, 'this' inside an arrow function always refers to 'this' from the outer context. In our case it was module's export object to which we gave the label id: Anni Maan because by default it is empty.
