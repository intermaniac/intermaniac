1) ifconfig command: - It stands for interface configuration and it displays the network configuration of your primary network device, such as IP address and if it is currently up and running

2) ip command: - This command is coming up to replace the 'ifconfig' command as it is becoming deprecated on most Linux systems. At the basic level, they display almost the same information but the 'ip' command has more options when it comes to configuring network interfaces.

3) route command: - This command displays the routing table. It is a table that describes where network packages are sent. It is also becoming deprecated and being replaced by the 'ip route' command.

4) ping command: - A command used to check if another host on a network is reachable. It does this by sending packets and if we get back a response, then that host is reachable and we should be able to connect to it.

5) netstat command: - This command provides a large amount of network information. It can also display the routing table similar to the route command. It is typically used to listen to open ports on the host. using the -l option. - It is deprecated on most Linux distributions.

6) ss command: - Displays socket statistics. Meant to be a replacement for the netstat command and it has more features. Just typing ss will give you information similar to the netstat command.

7) dig command: - performs queries on the DNS server to determine if the information needed is available on the server. If the information is not available, it returns a connection timeout error.

8) host command: - This command associates a domain name to an IP address and vice versa. To view a comprehensive list of DNS information regarding a particular domain, use the -a option.

9) ssh command: - The ssh command allows you to connect to another machine across the network, log in and then perform tasks on the remote machine.

To know more about any of the commands mentioned above, including the options they provide, use man [name of command]
