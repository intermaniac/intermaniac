Components are at the very core of React. React is first and foremost a tool to build UI, components are the building blocks of that.

In the non React world, we use HTML to structure our content, CSS to style, and Javascript to embed functionality. Everything from a simple button to a sidebar to a hamburger menu is written like this.

React lets us combine it all into custom components that are written independently and can be reused as many times as needed. Think about creating a button component once and using it across the entire app with different values for different buttons, as needed.

This unlocks tremendous flexibility and helps do more with less code. Moreover, since the components are both independent as well as customisable, it saves a lot of time, especially in big apps

So, we know that components are reusable pieces of UI that React revolves around. Let's now take a look at a code sample to better understand it. We'll consider a greeter component.

In fact, if you think of any app, you'll find that every app can be broken down into components. From 'watch later' on a video streaming app to the end call button on Zoom, everything is a component. And it is here that React truly shines.
