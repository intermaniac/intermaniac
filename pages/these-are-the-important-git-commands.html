5. git log Used to view the commit history. 6. git commit --amend Used to amend changes to the previous commit. 7. git branch Displays list of existing branches in git repo.

8. git branch <branch> Creates a new branch based upon the current HEAD. Note: HEAD is just a pointer, pointing to the currently checked out branch(attached HEAD) or commit (detached HEAD). Or in simple words, HEAD answers the question "where am I right now in the current repo"

9. git switch <branch-name> Used to switch to the specified branch. You can also switch between branches using an alternative command git checkout <branch-name>, but "switch" command is the latest way of doing it. https://t.co/t9ZwP77JiM

10. git switch -c <branch-name> Use git switch with the -c flag to create a new branch and switch to it all in one go 11. git branch -v Used to view more information about each branch

12. git merge <branch-name> Used to merge changes from the specified branch into the current branch. Note: Before merging switch to the branch you want to merge the changes into (the receiving branch) https://t.co/EVZKJ9Mohz

13. git diff Lists the changes in our working directory that are not staged for the next commit.

14. git diff --staged Lists the changes between the staging area and our last commit. git diff <branch1-hash>..<branch2-hash> Lists the changes between tip of two branches git diff <commit1-hash>..<commit2-hash> Lists the changes between tip of two commits https://t.co/nPCEFCQxJP

15. git stash Used to stash all uncommitted changes (reverting changes in working directory) git stash pop Used to remove the most recently stashed changes in your stash and re-apply them to your current working copy

16. git checkout <commit-hash> Used to view the previous commits. We can use git log command to view the commit hashes. 17. git checkout HEAD <file> / git checkout -- <file> To discard any changes in that file, reverting back to the HEAD.

18. git reset <commit-hash> will reset the repo back to a specific commit. The commits are gone but the actual commit changes are still there. git reset --hard <commit-hash> will reset the repo back to a specific commit. Both the commit and commit changes are deleted https://t.co/7tVQfa3RdR

19. git revert <commit-hash> This is similar to git reset, but creates a new commit which reverses/undo the changes from a specific commit.

20. git clone <url> Used to clone a repo. Git will fetch all the files from the specified remote repo and will copy them to our local machine. https://t.co/RqFUgh1G5b

21. git remote add <remote-name> <url> Adds a new remote with specified name("origin" is the common name used for remote) to the git repository. 22. git remote -v Displays details of all the remote that is added to the repository.

23. git push <remote-name> <branch> pushes the committed changes in specific local branch to the specified remote repo. https://t.co/ykD7F23XuT

24. git branch -r Used to view the remote branches our local repository knows about.

25. git checkout <remote-name>/<branch> Used to checkout the remote branch (Now the HEAD is in detached HEAD state) If you want to work on any remote branch that is not in your local repo, then use this command -> git switch <remote-branch>

26. git fetch <remote> fetches branches and history from a specific remote repository. It only updates remote tracking branches. git fetch <remote> <branch> Used to fetch a specific remote branch.

27. git pull <remote> <branch> This will fetch the latest changes from particular remote branch and merge those changes to the current local branch. git pull is nothing but git fetch + git merge. https://t.co/EhIYY6vSCk

28. git tag Used to print the list of all the tags in current repository. Note: Tags are normally used for versioning the repo.

For more details on GIT commands check the git documentation. https://t.co/GMdZ3fJEhl
