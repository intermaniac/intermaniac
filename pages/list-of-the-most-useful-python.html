A list of the most useful #Python libraries you can use for #SEO right now. üêç This updated thread will tell you the main libraries for #DataScience and #NLP that you should consider. Use them in your workflow! üßµ

BeautifulSoup: a library to extract data from HTML/XML files, used in combination with scraping libraries to convert data into Python objects. One of the first ones you‚Äôll probably learn in your Python journey. https://t.co/b4akzyV5lL

Scrapy: the absolute peak of scraping. Nothing is better than this, even though the setup may be hard. You can carry out any scraping task with this library.

Matplotlib/Seaborn/Plotly: you need some sort of visualization and these libs are here to help you. You can start with Seaborn which is easier to use. DataViz is an important topic and you should value it.

NLTK/spaCy: work with human language to analyze text data and get insights into the nuances of our language. This is necessary to get your hands dirty with text data. The latter can be used to recognize entities and parts of speech.

Querycat: few functions but good quality thanks to association rule mining and BERT. It's one of my favorite libraries, but the installation may not be immediate. It's useful for visualizing losses in impressions over time. https://t.co/0t8l6gyNqV

Sklearn: A staple for Machine Learning. I don't think you really need it, but it's one of the first libs you will encounter. https://t.co/GBjKlXGwk2

Transformers: Pretrained models to handle a wide range of tasks. Essential for NLP! This library is crucial for the most advanced tasks and quite reliable too. I highly suggest you check my other thread: https://t.co/1qNYrlPIkJ

sentence_transformers: Python framework for state-of-the-art sentence, text, and image embeddings. Use it for keyword clustering and other text-related tasks. It's one of my most used libraries right now. https://t.co/44CQUiRTIZ

Trafilatura: download, parse and scrape web pages. If you work with content, look no further. Cleaning the HTML elements of a page is overrated, don't waste your life on it! https://t.co/srx4RtBPnd

Streamlit/Dash: interactive web applications. Useful for prototyping and communicating. Streamlit is one of the most favorite solutions for the SEO community.

Typer: create apps that you can run from your command line. Extremely powerful for personal uses and for running local scripts. A game-changer for automating your workflow. https://t.co/PzYZ5utcQx

networkx: the must-have graph theory library. I recommend you learn it once you have mastered the basics. Graph Theory is of great importance for analysts who want to level up their game. More on this in future threads. https://t.co/VHGXu40Xjv

searchconsole: Use this library to import your data from the GSC API. It's easy to set up and it's one of the most used libraries in my workflow. https://t.co/I7PjoDJyNe

BERTopic: one of my most used NLP libraries and for good reasons. I dedicated an entire thread on the topic: https://t.co/ABbuRHu4tg

scattertext: library for finding distinguishing terms in corpora and displaying them in an interactive HTML scatter plot. A short example from the official docs: (https://t.co/QVgZrdugTl).

openpyxl: if you have to work with Excel data and create spreadsheets. There are other libraries but I prefer to use this one. It's quite nice and it works well for most of the tasks. https://t.co/gtULBx8RCo

Start with scraping and data analysis. Then, you can move to NLP libraries and study topics like NER and Clustering.

Sticking to the mainstream libraries is necessary to get access to "better" documentation. My suggestion is to try alternatives and always look for new opportunities across the web. Be sure to always do your research, you could find the perfect library for your needs.
