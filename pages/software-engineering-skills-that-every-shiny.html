5 software engineering skills that every Shiny Developer should know: 1) Testing 2) System design 3) Writing robust and reproducible code 4) Using Sass and JavaScript 5) Consistency // üßµ

1) Testing: Coding is not just about pumping out new features. Ensuring the code is correct and resilient is crucial for production. Automated tests are perhaps the most important way to achieve that.

Typical blocker? Setting up a testing framework and ensuring tests are regularly run. üëâThat's whyü¶è {rhino} (https://t.co/DlOLl8eOWR) comes with a complete setup: 1. Unit tests with testthat 2. E2E tests with Cypress 3. All run on CI via GitHub Actions

2) System design Large app requires thoughtful design or developer productivity will plummet. Separation of concerns: organize code into modules with well-defined purpose and clear interface.

Typically languages support modularization at various levels. Example: Sass and JavaScript. Vanilla R is a bit lacking in this regard.

üëâ {rhino} supports using: 1. {box} modules for encapsulation 2. Logic and View separation: Logic can be expressed without Shiny, View defines the UI and wires its elements together with reactivity. 3. Shiny modules: ease of understanding, avoid ID clashes

3) Writing robust and reproducible code In R it is common to use library() call to import all names from a package into the global namespace. It is great for interactive use and quick prototyping. While tempting, this relaxed attitude towards dependencies has serious drawbacks:

1. Brittleness: update to your dependencies can change break your code 2. Lack of isolation: You cannot use different package versions in different projects 3. Name clashes and namespace pollution. Can be especially severe in long-lived projects with multiple collaborators.

{rhino} addresses these issues by bringing explicit dependency management to R: At project level: renv with its lockfile. At file level: box with its explicit imports/exports.

4) Using Sass and JavaScript Amazing things can be done with just R and its packages. Still, you‚Äôll face situations where they are not flexible enough to meet your needs. With JavaScript and Sass you get complete control over your application.

Rhino comes with project setup for JS and Sass out of the box. Both are compiled and minified to offer you the latest features and maximum performance.

5) Consistency Everybody has their preferences, but it is important to keep code style consistent across the project: 1. Makes the code easier to read. 2. Spend less time on code styling issues during code reviews. You should have a style guide, but you need to enforce it.

The best way to enforce style consistency is automation. Rhino comes with linters and formatters for R, JavaScript and Sass. They can automatically detect improperly formatted code and in many cases fix it. {rhino} lints all your code automatically via GitHub Actions.

Software engineering skills that every Shiny Developer should have: 1) Testing: correctness is crucial 2) System design: design for maintability 3) Robust code: use explicit dependency management 4) Sass and JavaScript: leverage full control over UI 5) Consistency: use linters
