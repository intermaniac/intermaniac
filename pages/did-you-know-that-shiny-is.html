Did you know that #Shiny is not the only way to create dynamic web apps with #RStats I created the following longevity risk model with OpenCpu using the three layers of the MVC concept: 1 Model : R OpenCpu 2 View: HTML/CSS 3 Controller: Modern Javascript. Thread: https://t.co/lOkK33aVNg

Two years ago I created a risk web-application that was implemented with ease and straightforwardness in R Shiny. Despite the easiness of Shiny,I didn't like the mixing of logic (Model) with HTML/JS/CSS (View) in the same R environment.The disadvantages of using Shiny are:

- Reusability: Can't reuse the R model in other packages - Seperation of concerns and decoupling: Can't tap into the almost unlimited existing Javascript, CSS and HTML frameworks/packages for building the front-end (View). - Scalability and openness: Vendor lock in.

While Opencpu has the following advantages: - Separation of concerns: Let R do where it has been built for, namely statistics and graphics. - Communicate with R from various clients that can speak and understand http. - Natural scalability

Building the Model: Setting up Opencpu on any cloud server (Azure/AWS/GC) is super easy: just a matter of copying and pasting few lines of code or use the public server. The nice thing about OpenCpu is that you can automatically hook your github R model "as is" to the server

Building the View: I used the bootstrap CSS framework to make a simple dashboard. https://t.co/rAMUKwkwZE

Building the Controller: here we define the control flow that can be triggered by one or multiple events,leading to the creation as well as retrieval of several R nodes/objects. I used the fetch javascript api to achieve this two way communication. https://t.co/6RnaxLhrzg

If you find this thread interesting/helpful, consider following me @datahabits_io for future updates on web development with #Rstats.
