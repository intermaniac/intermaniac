Take a look at this Python 🐍 dictionary that is case-insensitive 👇. This thread teaches you to implement it. https://t.co/s1yCy76vAz

To implement a case-insensitive dictionary we need four ingredients: 👉 the string method `str.casefold` 👉 the dunder method `__setitem__` 👉 the dunder method `__getitem__` 👉 the dunder method `__delitem__` Here is what the method `str.casefold` does:

The method `str.casefold` is THE method you SHOULD use when you want to do case-insensitive string comparisons. If you need to check if two strings are the same, up to casing, don't use `str.lower` or `str.upper`. It works with ASCII but fails with other languages https://t.co/dLxG3zQ0gK

I have a blog post that explains in detail why you should use `casefold` instead of `upper` or `lower`. Heck, you can even check the Python documentation and they agree with me 😁 But the blog post gives a detailed explanation, in case you're wondering. https://t.co/698IlFXS7E">https://t.co/698IlFXS7E

Now we know how to check if two strings are the same, up to casing. Now, we need to use that to implement the dictionary. When we add / access / modify / delete keys in a dictionary, Python 🐍 calls some “dunder methods” under the hood. https://t.co/staAVecRTW

In particular, dictionaries use the three dunder methods I talked about in the beginning: 👉 `__setitem__` is used to set values with `d[key] = value` 👉 `__getitem__` is used to access values with `d[key]` 👉 `__delitem__` is used to delete values with `del d[key]`

So, in order to implement a class that looks like a dictionary, we need to implement those three methods with those specific names. A simple implementation could look like this 👇 But how does it work? https://t.co/3b6pf8uS5e

We start by inheriting from `dict`. If we want to make a *case-insensitive* dictionary, might as well use as a starting point a fully functioning dictionary! https://t.co/5wpRtHjSgt

Then, we define the three dunder methods we talked about. Each dunder method is called IMPLICITLY by Python 🐍 in a specific situation: 👉 when setting an item 👉 when accessing an item 👉 when deleting an item But what's up with the body of those methods..? https://t.co/XvBy8sxyxr

In each dunder method, we do exactly two things: 👉 use `str.casefold` to change the key to its normalised version for case-insensitive comparison; and 👉 let the parent class `dict` actually do the heavy-lifting of handling keys and values! So, we essentially cheated:

We implement a case-insensitive dictionary in Python 🐍 by “intercepting” the regular behaviour of the built-in `dict`. Then, when we intercept the moments where items are set/modified/deleted, we just make sure those operations are done in a case-insensitive way.

By using this strategy, we also got A LOT of functionality for free, like 👉 pretty representation of the dictionary 👉 the methods `.items`, `.keys`, and `.values` 👉 the ability to iterate over the dictionary 👉 etc... But this class isn't perfect. (Yet!) Can you see why..?

You can take a look at this link with a more detailed explanation of what is going on.) https://t.co/698IlFXS7E">https://t.co/698IlFXS7E

In summary, if you want to prototype a case-insensitive dict in Python 🐍: 👉 inherit from `dict` 👉 use `str.casefold` for caseless string comparison 👉 intercept the dunder methods `__setitem__`, `__getitem__`, and `__delitem__` 👉 defer the heavy lifting to `dict`
