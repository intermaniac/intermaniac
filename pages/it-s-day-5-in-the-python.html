It's Day 5 in the Python functions series. It's time for, drum roll… **kwargs If you've not read yesterday's thread on *args, you may want to start there! Let's look at today's example to discuss **kwargs <code in ALT text> /1 https://t.co/0Nx5w5Xyi6

As was the case with *args, there's nothing special about the name `kwargs`. What makes a kwargs a kwargs (!) is the double asterisk before the parameter name, so you can use a more descriptive parameter name /2

In this example, I use `people` as the parameter name and it's preceded by the double asterisk ** in the function signature /3

Let's see what `people` is in this case by printing it out and printing out what data type it is, too /4 https://t.co/gzSlLipHX2

`people` is a dictionary When you call the function, you've used named arguments or keyword arguments in the function call, not positional arguments /5

You can see an earlier thread in the series to refresh your memory about positional and named arguments: /6 https://t.co/ofDCCGk2vx

The keyword in the named argument becomes the key in the dictionary The argument becomes the value in the dictionary associated with that key /7

You can include as many keyword arguments as you wish when you call a function which has **kwargs And incidentally, you may have guessed already, kwargs stands for KeyWord ARGumentS /8

We can now return to the original code and show its output, too /9 https://t.co/bN5PebY0bl

As `people` is a dictionary, you can loop through it using the dictionary method `items()` The last call, which has no arguments, doesn't print anything out since the dictionary is empty /10

In summary, you can add a **kwargs parameter when defining a function which allows you to add as many keyword arguments as you want when you call the function These keyword arguments will be collected in a dictionary. The keyword becomes the key and the argument the value /11

Tomorrow, we'll have a look at an example which uses both *args and **kwargs as well as other parameters /12
