#Day5 #onecyberthingadaywithcybersechaven A Thread 👇👇👇👇👇 Today, I learned about Process Management on Linux OS In my today's study, I learned that Linux has hundreds and sometimes thousands of processes running simultaneously on the machine, at a given time.

As a good Linux user, it is paramount to know these processes and how to manage them. This is important because it helps to adequately optimize system resources. I learned how to view and find these processes, how to discover which of the process is using the most resources,

How to run processes in the background, How to prioritize processes, kill processes ( stop the process from running), and how the process can be scheduled to run at a particular time and date.

I will briefly talk about each of the aforementioned below:

1) viewing and finding processes - The most basic tool/command for viewing processes on Linux machines is the “ps” command(process status). The ps command does not provide much information about the processes running in the system. https://t.co/coJzj8X9qJ

To get more information about all the processes running on the system and for all users, you will have to run the ps command(process status) with the“aux” options where a = show processes for all users u = display the process’s user/Owner x = to show processes not attached to ter

When these processes are displayed, Linux assign a unique ID to each of these processes sequentially, based on how they were created and this unique ID is shown as the “PID”(process ID) Running the“ps aux”command will give us numerous details about the processes that are running.

https://t.co/vVS5sVwp42

and some of the most important details are, the user who invoked the process, the PID,The % of CPU the process is using, the % of memory the process is using and the Command that initiated the process.

All of these are important to efficiently manage the processes for enhanced optimization of system resources.

2) How to discover which of the processes is using the most resource - To know the processes that are using the most resources, is where the “top” command comes in handy... The “top” command updates this list of processes every 3 seconds by default. https://t.co/rNbw4rveq1

3) How to run process in the background - some processes take time to run completely in the terminal and this will force the user to wait for it to be complete in the terminal, before doing something else. Such applications( processes) are best run in the background.

so you can have space to do something else. To start a process/application in the background, just append an ampersand (&) to the end of the command e.g “snort &” https://t.co/3QUKIzSbz1

4) How to prioritize processes - prioritizing processes on Linux is where the “nice” command comes in handy Using the “nice” command with a certain range of value, usually between -20 to +19 with zero (0) being the default value helps to prioritize a particular process.

In the range value, -20 represents the highest received priority while +19 represents the least priority.

5) Killing processes - Sometimes a process will consume too many system resources or exhibit unusual behavior, when this problematic process is noticed, you might want to stop it. This can be achieved by using the “kill command”

The kill command has 64 different kill signals which perform slightly different functions. For the sake of trying to be brief, I won't say much about the “kill command” in this post.

6) How processes can be scheduled to run at a particular time and date - As a good Linux user knowing how to schedule processes to run at a particular time is very important This can be achieved in two ways by using the “at” and “crond” commands.

This “at” is used to set up a daemon(background process) to run once at a particular date and time in the future while the “crond” is more suitable to schedule a process to run every day, weeks, or even months.

The difference between both commands is that “at” run once while “crond” is repetitive. You can refer to my screenshot below to see the acceptable time arguments to run these commands. https://t.co/zLHd1dA2oU
