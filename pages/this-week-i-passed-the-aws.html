This week I passed the AWS Architect Professional exam 🎉 Here are my hints on preparing, exam scope, understanding questions, and some pro tips! A thread 🧵👇 https://t.co/Z9SE3XUeEK

🧑‍🎓 Learning Video courses provide the best source of condensed knowledge for exams. My choice is always @StephaneMaarek courses on Udemy.

How to learn? Don't make it just passive watching. Make notes, mind maps, whatever suits you. Go to AWS Console and play with configurations yourself. You will remember much more this way. I wrote more about how I take notes when I passed my DevOps exam: https://t.co/bz3z8O0R4F

Another great learning technique is solving tests. Many psychological experiments prove it! There are sample questions available for the exam - go through them. Then find some more on the internet! This will also prepare you for the type of questions on the exam.

🗂️ Exam scope The exam guide lists dozens of AWS services in the scope. But those topics and services play the biggest role: 🔶hybrid cloud - connecting on-premise with AWS 🔹Direct Connect, VPN, Storage Gateway 🔹connection redundancy 🔹connecting with multiple regions

🔶migrating from on-premise to AWS 🔹Database Migration Service, Server Migration Service, Snowball 🔹no-downtime large DBs migrations (Snowball + DMS) 🔶multi-account VPCs 🔹connecting multiple VPCs 🔹using a single VPC to connect to the internet / on-premise

🔶managing multiple accounts 🔹Organizations, SCP, CloudFormation StackSets 🔹centralized logging 🔹enforcing tagging rules 🔹cost management and producing per-account cost reports

🔶DB scaling and resiliency 🔹read replicas 🔹migrating to Aurora 🔶High Availability and Disaster Recovery architecture 🔹DR strategies 🔹backup between regions 🔹Route53 failover 🔶cost optimization 🔹Reserved and Spot instances

🔶general, architecture-level knowledge of services 🔹EC2, RDS, ELB, AutoScaling, Systems Manager 🔹Elastic Beanstalk, ECS 🔹CodeDeploy 🔹S3, CloudFront, WAF, Shield 🔹EventBridge

🔶serverless solutions (maybe 5 questions) 🔹API Gtw., Lambda, DynamoDB 🔹SQS, SNS, Step Functions 🔹Athena, QuickSight

Thankfully not in the scope: - differences between EC2 instance types - VPC routing (except for multi-VPC)

🤔 Exam questions All questions are scenario-based. And they are long. For the first time, I did not finish after ~half time. The exam took me 2.5 of 3 available hours 🕒

Establish: - tech. requirements, e.g. NoSQL DB => will need DynamoDB, storing files => S3 - non-functional requirements, e.g. high availability, scalability - objective, e.g. reducing costs or complexity - whether re-architecture or minimal changes are expected

Then exclude non-matching answers and choose the best one from the rest.

Multi-choice questions usually require you to select the COMBINATION of steps. If you must choose 3 from 6 answers, there are probably 3 aspects of the question scenario and 2 answers for each. Identify the pairs, and choose 1 from each pair.

💡 Pro tips If the answer includes "use third-party solution", you can exclude it right away 😉 "Reducing management complexity" is the keyword for using managed services instead of EC2, for example serverless.

Migrating three-tier applications to AWS: - move the web tier to S3+CloudFront, - move DB to RDS, never to self-managed DB on EC2. Migrating data to AWS - if the question contains network connection speed, data size in TB, and time limit - often the answer is Snowball.

⚙️ Technicalities The exam is valid for the next 3 years. Passing it also prolongs the validity of the Architect Associate certificate for the next 3 years.
