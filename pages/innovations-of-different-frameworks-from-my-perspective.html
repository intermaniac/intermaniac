Innovations of different frameworks from my perspective. @angular, React, @sveltjs, @solid_js, @QwikDev ðŸ§µ

AngularJS was originally meant to extend the HTML tag/attr vocabulary. This meant that AngularJS need to be reactive to any JS object change. Dirty checking was the only way to do it. The upside was that it just worked.

The other thing to understand is that HTML was partially a source of the truth. HTML attrs would control the FW behavior.

React introduce few innovations: JS was the source of truth, unidirectional data flow, JSX, and setState. setState was especially important because it told the FW when it should do dirty checking on the vDOM. This put dirty checking control in the developers hands.

All of this was course grained reactive. A change would trigger a large amount of JS execution to reconcile the DOM. This would cause performance problems especially with animation. Fine grained reactivity was needed.

Everyone understood and wanted to do fine grained reactivity. But it was not obvious how to do it with good developer ergonomics. Svelt took a stab at it with the compiler. This was great for compiled code but for non compiled code Svelt had to introduce. Stores.

Single mental model is always preferred! Here I think @solid_js did great work! It introduce syntax which was easy and it worked everywhere not just in compiled code. The other innovation was no need to explicitly subscribe or unsubscribe.

I should also call out MOBX which popularized many of reactivity concepts. Fine grained reactivity is a win because it minimizes the amount of JS which needs to be executed on value change. Less JS faster app runtime.

The thing which fine grained reactivity does not solve is the amount of JS executed on startup. All components in the render tree need to execute on startup to build up the reactivity graph. Otherwise reactivity does not work.

This is where @QwikDev comes in. Resumability allows it to serialize the reactive graph such that nothing needs to execute on application startup. The application is reactive without any JS.

There are two wins. 1) resumability allows no JS executed on app startup 2) because reactivity graph is recovered most of the JS never runs. Hence more aggressive tree shaking and lazy loading. Often times only a fraction of JS needs to download.

There is a strong fine grained reactivity progress over the years. And each innovation stands on the shoulders of other giants.
