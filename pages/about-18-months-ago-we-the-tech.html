About 18 months ago, we (the tech team at Apteo) switched from creating custom Python models to using BigQuery for ML. For us, it was a godsend. üßµ Here's why.

For those that don't know, BigQuery is Google's all-in-one data warehouses (or data lake, for the consultants out there), ML trainer, and ML model host. It's extremely fast and handles huge amounts of data.

But it also takes care of a lot of the tedious tasks that need to be done when creating an ML model.

For example, it has a lot of built in transformations for data (one-hot encoding, standardization, etc)

It also provides multiple learning algorithms, along with the ability to tweak their parameters, so you as a data scientist can move quickly and test multiple models at once to get the results you need without waiting for hours or days

‚òùÔ∏è That last point above is crucial. BigQuery ML is *fast*. Whereas it used to take us 12 hours to build models on a large dataset using custom built Python models, it might take us 30-45 minutes on BigQuery. Less waiting = faster iteration.

It's also nice to be able to structure the data in the same system as where it's modeled. No data transfer delays or cost is a nice to have.

Finally, BQ ML also provides some nice and easy to understand evaluation functions, and they provide explainability metrics with just a single line of code. Wrap it all up in SQL (imo a very elegant language), and you've got a great data science system.

And while I love using it, I do with it had a few other things: - Easy-to-implement wide recurrent networks with configurable params - Better support for large training datasets (they might have changed this but the last time I ran a model I was limited to 50M examples)

There are a few other things as well, but those are all minor considerations. I've been fortunate enough to chat with the team that develops it, and they're smart, they're working hard to add features, and they understand how a tedious data science process slows down progress.

Highly recommend for all ML engineers. Now... let's see if they start supporting generative A.I. üòÄ
