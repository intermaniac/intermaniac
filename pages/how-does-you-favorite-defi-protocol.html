2/ What is an oracle? An oracle communicates between blockchains and any off-chain data source By combining off-chain data with smart contracts, you enable use cases of peer-to-peer transactions that were not possible before (i.e. sports betting without centralized sportsbook)

3/ The oracle gets the external data and records it on chain as a transaction so it's referenceable at any point in time However, off-chain data comes from a centralized source which introduces a single point of failure That's where a Decentralized Oracle Network (DON) comes in

4/ A DON is a distributed network that acts as an abstraction layer (abstraction layer is just a software structure that is easier to interact with) Chainlink is decentralized at 3 levels: data aggregators, node operators, and oracle network aggregation https://t.co/113rhKzRg2

5/ There are 2 main components to DON 1) Executables - programs that run continuously and performs deterministic operations 2) Adapters - links to external resources and may operate in both directions https://t.co/3S7eH2wvgB

6/ Important to call out, the executable processes transactions and external data and forwards the results This shifts the transaction processing off chain which reduces latency and fees and boosts throughput

7/ Chainlink is chain agnostic but is widely available on EVM chains and Solana

8/ Currently, the most popular use case for oracles is in DeFi Accurate pricing data with minimal latency is critical so DeFi protocols use Data Feeds to get real-time asset prices in their smart contracts (https://t.co/ZKgj2N3EJ8)
