Day #2 of learning @NEARProtocol In this thread, I am going to summarize AssemblyScript smart contract development on NEAR We will see a basic smart contract in Typescript (compiles to assembly) to store and retrieve data You can find the detailed article at @dacadeOrg

1/3 Storage on blockchain Following are the Storage types that can be used to store and retrieve data from the smart contract PersistentMap - a key-value data structure PersistentVector - an array PersistentDeque - a bidirectional queue PersistentUnorderedMap - an iterable map

2/3 Read and Write State There are 2 types of function calls, "view" and "change" "View" calls will only read data from the blockchain, while "change" calls will write data to the blockchain and modify its state View calls are free, while we need to pay gas for change calls

3/3 Smart contract example A basic smart contract that stores Products mapping from (productId => productName) and allows to retrieve productName by productId would look something like this https://t.co/iebBlWv2gy
