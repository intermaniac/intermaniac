There's no reason not to use TypeScript in 2022. It will make your development 10 times easier. It only takes 5 minutes to learn to use TypeScript with ReactJS, and you will benefit from it instantly. Here's how you can use TypeScript in ReactJS ðŸ§µ

If there's only one thing you use TypeScript for in React, make it this - component props. - You'll get autocomplete and static type checking in your IDE. - You don't need PropTypes anymore. Only 1 minute extra will save you hours in the future. https://t.co/wWArntZSrk

Sometimes you need to extend an element with extra props, by wrapping it in a component, that passes down the element props. 1. Hover on the element in your IDE to see its props type definitions. 2. Extend them in the component prop type or interface. https://t.co/YoecKwKF0J

Similarly, you may want to wrap other components. Here's the previously created component wrapped in another component. 1. Use the ComponentProps generic type to get the component props. 2. Extend them in the component prop types. https://t.co/9Mpa0kQiot

Other common use cases are: - Typing state - Typing event handlers - Typing refs Here's how you may use them ðŸ‘‡

Typing state. 1. Let typescript infer the value from default props if possible. 2. If not, the useState hook accepts a generic type, which describes the type (see User interface). When you use user and setUser you will get autocomplete and type-checking. https://t.co/DKYddC0aBm

Typing event handlers. Method 1: 1. Hover on the event handler attribute to see its type. 2. Use the type on the handler function. Method 2: 1. Hover the event parameter to see its type (usually same as event handler, without "Handler"). 2. Only set the type of the event param. https://t.co/Ew38YoaAl3

Typing refs. When using forwardRef on component, you can provide the generic type for the element type of the ref and component props. When using useRef, simply specify the element type of the ref and set null as the default value. https://t.co/vryLZ3qAlw
