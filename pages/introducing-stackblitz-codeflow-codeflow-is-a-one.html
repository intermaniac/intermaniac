Introducing: StackBlitz Codeflow! ♾ Codeflow is a one-click GitHub integration that keeps you in flow state by seamlessly weaving together: - creating & reviewing pull requests - fixing issues - contributing to open-source repositories in a way that wasn't possible before: 🧵 https://t.co/wEXPIr1R60

Codeflow eliminates the pain of context switching locally. For example: when a co-worker asks you to ship an urgent bug fix or to review a PR, but you have uncommitted local changes.😓 Wouldn’t it be great if you could spin up an instant copy of your environment just for that?

To solve this, we brought your desktop VSCode env into the browser with all of your settings, extensions, and toolchains. But unlike your local environment, these ones: - boot up to *10x faster* - are secure by default - can be disposed of as easily as closing a browser tab 🤯 https://t.co/2S8MDBqALq

How? Our team spent the last 3 years working to make this possible (and fast!): 🏎 pnpm runs 4x faster w/ @ZoltanKochan 🏎 Instant git clones & CLI in-browser 🏎 WebContainers boot Node.js in <200ms 🏎 Desktop VSC running natively on WebContainers And the result feels magical: https://t.co/VS1VIIGNld

1️⃣ To start a new PR, it’s as simple as clicking a button in your repo. Milliseconds later your desktop version of VSCode is running in your browser with your extensions, settings & toolchains, and you can open a PR in one click: https://t.co/kwqqEu9RQW

2️⃣ Doing a full PR review usually means running the toolchain (and stashing changes) locally. Now you can do that in one click, running the code live and see the changes, and reviewing directly in the live code: https://t.co/qdTcigJegN

3️⃣ Issue reproduction with one click means OSS maintainers don’t have to risk downloading and running the code locally. Also, Codeflow automatically links reproductions in a symlinked folder with your project using pnpm overrides so you can verify the bug is fixed:

4️⃣ But the best thing is, you can use Codeflow open & contribute to any repo you haven’t even worked on before. Simply add https://t.co/Cvlr62D2yg in front of any GitHub URL (repos, branches, pull requests, issues), and it magically transforms it into a live environment: https://t.co/Ca93fVbgxd

We’ve been using Codeflow for the past few months and it’s been a game changer. Now you can try it too! We’re incredibly excited to announce that Codeflow is now in Public Beta, and you can sign up now over at https://t.co/MhCugQPuBJ">https://t.co/MhCugQPuBJ ♾ Oh, there is one more thing…

We’ve also brought that seamless git-backed workflow into the documentarian world. ✍️ What if updating content in git repos felt like a real publishing experience? Introducing Web Publisher, a supercharged single file editing view that works with any framework: https://t.co/i0BgyvB3jQ

By lowering the barrier to entry for devs & non-devs alike, Codeflow is bringing development into the browser much the same way Figma did for design. On this note, we’re delighted to share that @Figma recently became a strategic investor in StackBlitz & part of our journey! 🎨⚡️ https://t.co/DXtKYuZia4

We’re excited about what these new development workflows unlock, and we hope you are too! ♾ Try out the beta at https://t.co/MhCugQPuBJ">https://t.co/MhCugQPuBJ 🚀 If you’re interested in making web development instant & seamless, we’d love to chat with you — we’re hiring! https://t.co/X7xlsKIE4E
