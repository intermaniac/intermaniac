Let's talk about Classes and Objects in Python üêç: A thread üßµ:

A class can be most aptly defined as a blueprint and the object instantiated from it as a product. The words 'object' and 'instance' are the same and are used interchangeably.

Python is an object oriented language and you work with objects of some built-in classes all the time while using it. https://t.co/SwUt839XdL

Every built-in class in python has some functions defined inside it that determine the behavior of its objects. If we try to implement a function on an object that is not defined inside its class we get an error. https://t.co/aR6gWKajbl

Procedural Programming which includes defining a bunch of functions operating on some data variables is pretty straightforward and you most probably learned first in your programming journey. However, as your programs gets larger and larger this approach becomes very inefficient.

The main purpose of object oriented programming is to streamline your code by removing interdependence among a lot of functions which makes your code vulnerable to breaking.

In object oriented programming, we group a bunch of related functions/methods and attributes/properties into one class and then we can create as many instances of that class as possible without reinventing the wheel each time.
