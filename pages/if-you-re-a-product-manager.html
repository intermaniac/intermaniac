If you're a Product Manager that can pull your own data, you can supercharge your impact. Master these 7 basic SQL functions and you'll be well on your way toward increasing your influence. ðŸ§µðŸ‘‡

The foundational functions of SQL queries: 1. Select 2. * 3. Count 4. From 5. Join 6. On 7. Where It's easier than you think. Let's dive in ðŸ‘‡

1/ Select Select = "Give me..." Virtually every query starts with "Select". Select is how you say "Give me the following...". Everything after "Select" is where the magic happens.

2/ * * = All "*" is commonly referred to as the "Wild Card". When you implement the "*" instead of column names, you will get every column in the table. When you use "*" with "Count" you will count all records. Speaking of "Count"...

3/ Count Count = Counting! When you leverage the "Count" function you will count the total of whatever attribute you request. For example: "Count(account_id)" will count every record in a table with an account_id. "Count" is a function commonly used to size your dataset.

4/ From From = The location of the data to query From is typically the table name. In most situations, data is structured as: 1. Database 2. Schema 3. Table ðŸ‘‡ database.schema.table So a sample table name would be: "enterprise_db.accounts.enrollment_records"

5/ Join Join = Connect data from two tables together Data is typically structured across multiple tables. Sometimes you will need to join one dataset to another. As long as they have a common identifier, you can do so. Example: From Table A Join Table B on account_id

Those familiar with SQL will note that I'm simplifying Joins for the purposes of this thread. There are 4 basic types of Joins: 1. Inner Join 2. Left Join 3. Right Join 4. Full Join Typically "Join" defaults to "Inner Join" as "Inner Join" is the most commonly used.

6/ On On = How you identify how to join two tables As mentioned with Joins, you must have a common identifier to join the datasets. The "On" function is where you specify the common identifier. In the example above, Table A is Joined with Table B using "account_id".

7/ Where Where = Specify "Where" is used to specify specific conditions the data must meet to be selected. For example, if we wanted accounts created in after January 1st, 2022, we could say: Where account_creation_date > '2022-01-01'

8/ Putting it together So let's review what an example query might look like using: - Select - * - From - Where We will pull all records from Table A where the account creation date was after January 1st, 2022. SELECT * FROM Table A WHERE account_creation_date > '2022-01-01'

9/ Summary As a PM, learning the basic SQL function is enough to make you dangerious. 1. Select 2. * 3. Count 4. From 5. Join 6. On 7. Where There is so much you can do with SQL. Let me know if it'd be helpful for another thread showing examples of using "Count" and "Join"!
