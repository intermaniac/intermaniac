SQL is one of the most sought after languages in data science... And sometimes, most of the keywords are often mistaken for one another.... Here's a short thread summarizing the different types of languages in SQL and the difference between some similar keywords....

SQL is grouped into 4 types of languages, based on what they do.... We have: 1. Data definition language (DDL) 2. Data manipulation language (DML) 3. Data control language (DCL) 4. Transaction Control Language (TCL) .....

the ddl statements include: - CREATE: Used to create a new table or database. - ALTER: used to alter of data in the database tables TRUNCATE: Used to delete data from the table without deleting the table itself. DROP: Used to drop a table. RENAME: to rename a table....

The DML statements include: - SELECT from - INSERT with conditions like INTO, WHERE - UPDATE with parameters like SET, WHERE - DELETE with parameters like FROM, WHERE... one good thing about SQL is that it's not case sensitive, so you can use any lettercase....

But best practices suggests that you write the keywords and functions in UPPERCASE.... so it can be more readable and easily understandable... The DCL statements include: - GRANT - REVOKE to either grant or revoke access or privileges to certain users....

The TCL statements include: - COMMIT: To save changes you have made to the database tables automatically... - ROLLBACK: to go back to the last change you saved.... People often confuse the TRUNCATE, DELETE and REMOVE statements.... Because they sound similar.....

Here's the difference: With the TRUNCATE statement, you're literally cleaning up the table..... It removes the data, but keeps the table... But with the DELETE statement, you can specify the exact data you want to delete using the WHERE clause......

While the DROP statement can be used to drop a whole table or database and that means totally removing it..... This is a brief summary.. Hope it helped you get familiar with the language or helped in refreshing your memory. I'll drop more detailed threads in the near future.
