🌠 Here are 4 Python causality libraries to learn in 2023 (a thread) (1/6) #causaltwitter #causality #causalinference #Python #MachineLearning

Python causal ecosystem grows rapidly. While writing my book on causality I analyzed it thoroughly, looking for pkgs that have stable support, enthusiastic contributors and simply do their job. (2/6)

⭕ 𝗗𝗼𝗪𝗵𝘆 (https://t.co/JVOb6j0nFl) This package originally from #Microsoft. Provides a unified interface for causal inference using structural causal models, graphical models & more. Great candidate for the Sci-kit Learn of Causality! Learn? -> https://t.co/sYYUj1LI4N (3/6)

⭕ 𝗘𝗰𝗼𝗻𝗠𝗟 (https://t.co/XMeTQcqWZV) DoWhy’s sister package built with conditional treatment effect estimation in mind. Supported by Microsoft. Integrates smoothly with DoWhy’s casual flow, whichj is a great advantage! (4/6)

⭕ 𝗴𝗖𝗮𝘀𝘁𝗹𝗲 (https://t.co/CVIDazfDpm) A causal discovery super-hero! Supported by Huawei, it’s the only package out there that implements the most recent causal discovery methods, including gradient-based & RL-based stuff! More? -> https://t.co/DfvmDh1xbw (5/6)

⭕ 𝗖𝗮𝘂𝘀𝗮𝗹𝗣𝘆 (https://t.co/XO2zvOQ56q) A new kid on the block. CausalPy is based on @pymc_labs PyMC. Developed by PyMC’s prolific @inferencelab with contributions from @juanitorduz . CausalPy implements a bunch of methods to work with quasi-experimental data. (6/6)
