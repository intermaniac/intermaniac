How to integrate an #LLM with a custom #API (e.g. a python function)? I did some experiments creating programmatic prompts that instruct text-da-vinci-003 to call specified functions ("back-commands"). ðŸ§µ thread below https://t.co/mou9QAqYOD

The idea is to have a controller python program that intercept the function call and run it; the returned text text is added to the DATA section. By example in a chatbot application, I foresee these steps: 1. the user ask about weather forecasts

2. LLM returns Call weather('Genova, Italy', 'Jan 14, 2023', '5:52:01 PM') 3. the controller insert in the DATA section weather forecasts: It's sunny all day. 4. LLM reply to user the updated info https://t.co/pENTcGbxX1

This seems to work, but there are 2 big cons: 1. the "program" written in the prompt is pretty unstable/sensitive to the natural language instructions; just minor changes in the lexicon, inhibit the generation of correct back command.

2. all the procedure consume almost 2X prompt token count, because you need to ask a forst completion to generate the back command and a second completion to submit the prompt that include the DATA section update.

QUESTION: Is this the a right way to go? There is any alternative approach? See also: https://t.co/q4INS8LxV0 How to using a @LangChainAI tool? https://t.co/4fJC85fryq Any suggestion/gist is welcome!
