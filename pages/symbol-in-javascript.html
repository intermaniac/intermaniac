A thread on Symbol in #javaScript 👇 https://t.co/KIZUWop7iW

🤔 What is Symbol? ☝️Symbol is one of the primitive data types that was introduced in ES6 ☝️used to generate a unique value but the fun part is that value is never known

🤔 How to create a Symbol? ☝️write Symbol() or ☝️optional description can also be passed -Symbol("desc") *new keyword is not used, results in TypeError https://t.co/g0alsccTxC

🤔 When two Symbols are created with the same description, do they contain the same values? ☝️NOOO!!! A new Symbol is created each time, even if a similar description is passed. https://t.co/4M0EGzrqov

🤔 But how to use the same Symbol value again? ☝️JS has a global Symbol registry, Symbol.for(<key>) is used to add values to registry. ☝️First, it is checked that the key exists in registry or not. If it doesn't exist, new Symbol is created otherwise existing one is returned. https://t.co/d4fpXS8ba0

🤔 How to access the key associated with Symbol in global registry? ☝️use Symbol.keyFor(<symbol-variable>) method https://t.co/3WkgFqhqgD

🤔 What can be the use case for Symbols? ☝️ to add unique property keys to an object, so that it never gets overridden ☝️it enables a form of weak encapsulation or a weak form of information hiding https://t.co/1G8upPDcKw

🤔 How to get the Symbol properties of an object? ☝️use Object.getOwnPropertySymbols(<object>) https://t.co/3R8IuTBwW2
