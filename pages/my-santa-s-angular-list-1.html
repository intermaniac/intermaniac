My Santa's #Angular list! ðŸ§µ 1/10

1. Make it truly reactive! Make ChangeDetection.OnPush not necessary to improve performance while fields are completely reactive.

2. Choose Jest over Karma in order to reduce testing times. Add complete support with Jest to avoid using weird APIs like process.nextTick and similar. Find a way that fixture.detectChanges resolves promises.

3. Make ControlValueAccesor way simpler.

4. Make things like ChangeDetectorRef unnecessary. The framework should know when things need to be re-rendered!

5. Make standalone components the way to go, improving their testing support.

6. Make providedIn: 'root' the default behaviour, perhaps be able to configure it in the angular.json.

7. Make HttpClient return Promises, not Observables. RxJs, while a great tool, should be optional. Maybe a truly reactive system would make RxJs not needed for the components and services.

8. Make Cypress the default E2E testing solution.

9. Add full eslint integration without needing to add the builder and dependencies.

10. Simplify the framework in general. I find myself sometimes doing the same thing with less code and complexity in other frameworks.

What would you like Angular to improve?
