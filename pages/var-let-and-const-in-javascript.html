Var,let and const in #Javascript: You should always use let or const in your Javascript code. I will explain you why in this thread:

1.Var is globally scoped when declared outside a function and function scoped inside a function. This will lead to many errors.

2.Var variables can be re-declared: You can literally do this all the time. But this is not viable coding.Sooner or later you will get bugs,errors and mess. https://t.co/hixYBh7TAC

3.Let and const are block scoped.If they are declared in a function they are "available" only in this scope.

4.Let can be updated but not redeclared. So this works https://t.co/HZldIC2hFr

5.A let variable can be re-declared but only in different scopes. This will work fine https://t.co/8tAlc68ZtS

6.Const variables have similarities with let variables.Both are function scoped and cannot be re-declared.

7.Just like let, const declarations are hoisted to the top but are not initialized.

8.Just like let, const declarations are hoisted to the top but are not initialized.

9.Const variables(origin from constant) cannot be re-declared.NEVER!

10.If your const is an object you can re-declare your object properties.The one and only time you are allowed to do that. Like this: https://t.co/agnp5ZiMiK

These are the key takes on Javascript variables. Hope it helped a little. :)
