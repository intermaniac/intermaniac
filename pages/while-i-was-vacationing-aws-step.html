While I was vacationing 🌴, AWS Step Functions got new intrinsic functions. Let's check them out! permanently delete They added 14 new functions, making it 18 in total. A thread 🧵👇 (with code examples!) https://t.co/6aO4N4Vwov

A quick Step Functions recap: https://t.co/vun9UM0Nvp

❓ What are intrinsic functions? ➡️ Operations you can make on the states' payloads to convert/generate data directly in Step Functions, without using additional Task states and doing it in Lambda functions. Intrinsic functions = less states = simpler (and cheaper) workflows

❓ How to use them? ➡️ You end the property key name with ".$" and then put the function name as a value. The function name starts with "States." You can use intrinsic functions in 𝗣𝗮𝗿𝗮𝗺𝗲𝘁𝗲𝗿𝘀 and 𝗥𝗲𝘀𝘂𝗹𝘁𝗦𝗲𝗹𝗲𝗰𝘁𝗼𝗿 fields. https://t.co/MQgE4qiMgL

1️⃣ "States.Array" returns a JSON array with given elements. When it's needed? When you want state input values (accessed by "$.") to be put in an array. https://t.co/dNydLciL6e

2️⃣ "States.ArrayPartition" splits the array into smaller chunks. When it's needed? When you want to process elements in batches. For example, DynamoDB BatchWriteItem operation accepts up to 25 items. You can split the array into chunks and iterate over them with Map state. https://t.co/uqs5w465BD

Side note: You can provide state input values (accessed by "$.") as any intrinsic function parameters. In some examples, I put hardcoded values for brevity. https://t.co/UVGmDCdxIt

3️⃣ "States.ArrayRange" generates an array of integers in a given range. Function parameters: start, stop (inclusive), step. When it's needed? 🤔 Maybe when you want to run Map state x times, where "x" is a dynamic value. You can create an array and iterate over it with Map. https://t.co/FgC4XwC6fo

4️⃣ Other "States.Array*" functions: - "ArrayContains" - true/false if element is present in the array - "ArrayGetItem" - returns the item (can pass "$." parameter as the index) - "ArrayLength" - returns array length - "ArrayUnique" - removes duplicates https://t.co/IqGtJNOxr5

5️⃣ "States.Base64Encode" and "States.Base64Decode" allow, you wouldn't guess it, encoding and decoding from Base64. https://t.co/034jClraDP

6️⃣ "States.Hash" generates value hash with a selected algorithm: - MD5 - SHA-1 - SHA-256 - SHA-384 - SHA-512 https://t.co/tJSx2qYWpK

7️⃣ "States.JsonMerge" merges two JSON objects. It takes two JSON objects as arguments. The third argument is whether to make a deep merge. Right now it must be "false" - only supports shallow merges. https://t.co/IJAd93uMqz

8️⃣ "States.StringToJson" and "States.JsonToString" convert between a JSON object and a stringified (escaped) JSON value. When it's needed? When you pass inputs or read outputs from Lambdas and other tasks that operate on stringified JSONs. https://t.co/I3ebF8EDHY

9️⃣ "States.MathRandom" generates a random number. It takes 3 arguments: - start (inclusive) - end (exclusive) - seed (optional) 🔓 Generated values are not cryptographically secure random numbers! https://t.co/uaPlQD5Dfp

1️⃣0️⃣ "States.MathAdd" adds numbers. 'Nuff said. https://t.co/SxD9jPYBnD

1️⃣1️⃣ "States.StringSplit" splits the string by a given delimiter. https://t.co/RiuielXlhY

1️⃣2️⃣ "States.UUID" generates a v4 UUID. https://t.co/opmtKn7joU

1️⃣3️⃣ "States.Format" builds a string from a given template and values. "{}" in the template are replaced with provided values. https://t.co/1MaHUpzyVe
