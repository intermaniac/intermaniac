1/ With 2021 behind us, easy access to capital has disappeared. 😶‍🌫️ 2022 is about managing costs and tightening engineering spend. 💸 🎬 Presenting 5 tips from the #Neeva infra team to control your #AWS bill. Read on... 🧵 https://t.co/zmlZAUkhax

2/ We use #AWS today for most of our storage and compute needs (with #kubernetes & #karpenter for provisioning compute resources). While AWS Cost Explorer is great, we realized you must explicitly enable tags [1] in order to slice and dice the numbers by tags you assign. ✂️🎲

3/ Tip 1️⃣ Watch your notebooks Helpful rule of 👍 = "notebooks aren’t to be left unattended” Data scientists are known for expensive overnight computations. So we: ☑️ Have scripts to monitor how long notebook instances run ☑️ Reach out to engineers that have them up for 1+ day https://t.co/I7uyXrpEFo

4/ Tip 2️⃣ Jobs over notebooks With #databricks, notebooks are 3x the cost of jobs[2]! Long running tasks (e.g,. training deep learning models, data processing on the index) should use jobs (kubernetes jobs, databricks jobs) and not notebooks. 🙅 📒

5/ Tip 3️⃣ Use S3 Storage Lens[3] Using S3 Storage Lens, we learned that we had a ton of space wasted by multipart uploads from canceled/incomplete jobs. We then switched to the lifecycle policy which cleans-up old versions and old multipart uploads. https://t.co/sQDH9oBtvu

6/ Tip 4️⃣ ⚠️ Use S3 storage classes (can be utilized to reduce costs for infrequently used data) ⚠️ We once set up a policy to transition data to the “IA” tier, but then one job that happened to analyze old data made it more expensive than leaving the data in the standard tier.

7/ FYI: We’re experimenting with S3 intelligent tiering now. 🙂

8/ Tip 5️⃣ Analyze data transfer costs Enabling flow logs and analyzing them helps, but requires manual analysis to correlate logs with the corresponding processes.

9/ We’ve caught egregious mistakes (e.g., a job running in one region reading large amounts of data from an S3 bucket in another), but would love some better tooling here. https://t.co/ynFw2uG0UA

10/ References: 🔎 https://t.co/Xb21y3uPhu 🔎 https://t.co/Chp1oFCDvh 🔎 https://t.co/542FZP9ZER

11/ We'd love for you to share any learnings or techniques you have found helpful in making sure the back-of-the-envelope calculations and the bills are consistent! 👇 https://t.co/nvvZRWcR1l
