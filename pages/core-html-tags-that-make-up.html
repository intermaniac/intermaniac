Core HTML tags that make up most websites on the internet (or at least should!) üßµ üëáüèª

<html> - top level 'root' element that contains all other tags within it <head> - not visually displayed but contains metadata that crawlers can interpret. Also contains scripts and styles for loading <body> - container for all visual content on a page, nest content tags here

<nav> - the section of the page that allows users to navigate to internal/external links <main> - tells the browser what the most important section of the body is <section> - container for standalone content that should contain a header tag to explain what is in it

<div> - extremally generic content container that can be styled <p> - used to represent sections of text (paragraphs) <img> - allows you to display images located either on your server or externally <ul/ol> - unordered/ordered list tags for showing bullet/numbered form content

<h1 - h6> - header tags that can be used by crawlers to determine what is on the page *note only use one h1 tag per page

HTML tags are important for screen readers to determine what is on your page and accurately index as well as describe content to visually impaired users. Take some time to learn when and what to use for the websites you're creating. https://t.co/PRPxKbef3e
