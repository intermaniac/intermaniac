Standard Queues (default) 1. Standard queues are available as the default queue type in SQS. 2. It guarantees that a message is delivered at least once. However, sometime, more than one copy of a message might be delivered out of order. 2/7

3. You can perform as many transactions per second as you like. 4. It offers "best-effort ordering," which guarantees that messages are typically delivered in the same order as they are sent, but it makes no guarantees. 3/7

FIFO Queue 1. The standard Queue is enhanced by the FIFO Queue. 2. It ensures that they are received in the same sequence in which they are sent, which is known as ordering. 4/7

3. A message is delivered only once and is available until the consumer processes and deletes it. This is known as FIFO Queue and exactly-once processing. 4. Duplicates are not permitted to be added to a FIFO queue. 5/7

5. FIFO Queues offer all the features of regular queues but are restricted to 300 transactions per second. 6. Additionally, it enables message groups, enabling the existence of many ordered message groups in a single Queue. 6/7
