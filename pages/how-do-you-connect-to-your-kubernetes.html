How do you connect to your #kubernetes applications database from your local #SQL client, which is located in a private subnet, like an #AWS RDS instance? A thread ðŸ§µ

What you need is a pod, which serves as a bridge between your local sql client and your private rds instance. So in the end you have: client<-->pod<-->db

To create the pod, type the following: kubectl run --restart=Never --namespace %namespace% --image=alpine/socat %podname% -- -d -d tcp-listen%local-port%,fork,reuseaddr tcp-connect:%rds-endpoint%:%rds-port%

Now the pod is running and should be listening to your db instance. So the connection pod<-->db is ready. Now we need the connection between our local sql-client and the running pod: kubectl port-forward %podname% --namespace %namespace% local-port:db-port

If you're using #aws you could merge these two commands in one single command using AWS Systems Manager: aws ssm start-session --target %rds-endpoint% --document-name AWS-StartPortForwardingSession --parameters "{\"portNumber\":[\"1234\"], \"localPortNumber\":[\"1234\"]}"

Now you can open your favorite sql client and create a connection on localhost:local-port and voilÃ , you should see your database.
