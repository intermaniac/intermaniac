Install #mlxtend in #Anaconda on Windows & use with #VSCode & #Juypter Notebooks. To use with VSCode, the Jupyter extension needs to be installed in VSCode. #Python #DataAnalytics #DataScience #OpenSource #code #CodeNewbies #gistlover #gischat #Analytics #QGIS #ArcPro Thread:ðŸ§µ https://t.co/ciXIlH7Dmy

First go to "Type here to search" on Windows, which is normally at the bottom of the screen to your left. In the search, type Anaconda Prompt. Select the Anaconda Prompt(anaconda3) or (anaconda2) depending on your version https://t.co/7ESins7mDG

After selecting the Anaconda Prompt, the prompt will open. You should see the following for your Anaconda Prompt but with your user. https://t.co/UL05TZIg6r

After opening the prompts. Type the following to update. You do not need to do this if you do not want to update your Conda. The following image show I do not need to update but if you have updates, the console with prompt you to enter y(yes) or n(no) to update. https://t.co/OKwIoAGpbP

After update, if this is the route you took, it is time to install mlxtend. To install a package in Anaconda, I recommend using conda. Type the following in the Anaconda Prompt: conda install -c conda-forge mlxtend and hit your Enter button. https://t.co/lLiFNhbfu7

Once the Enter button is pressed, you might get Prompted to proceed like in the following Image. Type y for yes and processed. https://t.co/Z6mtu44wqO

Now mlxtend should be in your #Anaconda and can be utilized now. Next, open #VSCode and try mlxtend out. https://t.co/pYpW0w6Zyl

With VSCode open & Juypter Notebook loaded, let's try out mlxtend. Note, make sure the correct kernel is selected. If the correct one is not select, it will not work. View the 2 images below. If you are like me, you could have multiple kernels. https://t.co/aXdPmwlM6u

Now that you have the correct kernel selected, add the sample code from mlxtend in your notebook to test. The sample code can be found at: https://t.co/iqKfNcxRkK. With that said, I hope this is helpful to someone and happy coding and exploring data. https://t.co/UGc7VmEIZT
