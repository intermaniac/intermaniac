Off-chain infra projects enable cross-chain communication, automation and bridging among other benefits. While there are many varieties of infra projects, this thread looks at the oracle, keeper, and bridge protocols in @avalancheavax, @BNBCHAIN, @NEARProtocol and @0xPolygon. https://t.co/fzv3udify6

@deBridgeFinance - deBridge helps power secure & seamless cross-chain apps, and have just announced their deSwap Liquidity Network, a new paradigm for limitless, zero slippage cross-chain value transfers.

As Avalanche, Binance and Polygon are EVM blockchains, they share a number of bridges. But there are some that are unique to each chain.

Polygon Bridges: @mtpelerin - Building Bridge Wallet, a non-custodial, multi-chain mobile wallet. Swap any Polygon token, buy and cash out MATIC, USDC and USDT with zero fees fiat on-ramp and off-ramp.

@gravity_bridge - A decentralized Cosmos blockchain for bridges between the Cosmos and Ethereum ecosystems. @MapProtocol - Omnichain layer for Web3 with provably secure cross-chain communication built on light-client & zk-technology.

@BNBCHAIN Bridges: @TeleportChain - Building TeleWallet, their multi-chain, multi-identity wallet allows for token swaps across any two chains, currently supporting EVM and Cosmos networks, adding Polkadot and Solana soon.

@Orbit_Chain - Orbit Chain is a cross-chain project that challenges the limitations of blockchain scalability, supporting token conversion between mainnets and supports 15 blockchains and 58 tokens.

@LayerZero_Labs - An omnichain interoperability protocol designed for lightweight message passing across chains.

Avalanche Bridges: @RangoExchange - The First Multichain DEX/Bridge Aggregator. @atlas_dex - A DEX where cross-chain swaps with 0% fees. @ScottyBeamIO - The first cross-chain NFT teleporter and INO platform!

The use of blockchain oracles allows external data to be passed on-chain in a trust minimized way. https://t.co/cdIdtiCbFp

The oracle space is dominated by Chainlink, with it being available across all 4 ecosystems. The only other oracles that are shared across these chains are:

@API3DAO - a DAO leading the movement for first-party oracle solutions that deliver more security, efficiency, and regulatory compliance. @BandProtocol - Secure, scalable cross-chain decentralized oracle - connecting smart contracts to external data and APIs.

Avalanche Oracles: @cartesiproject - The Blockchain OS. A Layer-2 solution integrating Linux & standard programming environments to blockchain. @db_dao - The first community curated blockchain-based database for Web3 applications.

BNB Oracles: @BNBChain‚Äôs Oracle - A price feed for the DeFi ecosystem provided securely through Binance. @GomuWeb3 - Offers all the API data and development tools you need to start building, beginning first with non-fungible tokens (NFTs).

Polygon Oracles: @Blockpour - Multi-chain DeFi data analytics & insights. @UMAProtocol - An optimistic oracle built for Web3, used by over 10 projects with over $60M TVL.

Near Oracles: @DIAdata_org - An open-source, financial information platform utilizinng crypto economic incentives to source and validate data for smart contracts. @fluxprotocol - The trustless data layer for web3 - providing access to economically secure data feeds on anything.

Although each of these ecosystems have diverse off-chain projects, there are still many gaps to fill. What is missing is a generalized platform for building custom decentralized off-chain services.
