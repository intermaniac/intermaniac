•A server which is a type of long-running program called a daemon process. •The REST API is used to specify interfaces that programs can use to talk to the daemon and instruct it what to do. •A command line interface client is used to execute docker commands. 5/23

what is Docker image? •Docker Image can be compared to a template which is used to create Docker Containers. They are the building blocks of a Docker Container. command- docker images above command will list all locally stored images. 6/23

•These Docker Images are created from Dockerfile using the below docker build command. •docker build -t imagename:tagname dockerfilepath •Docker images are used for creating containers by using the below docker run command. •docker run -it -d imagename 7/23

•We can either use a ready-made docker image from DockerHub or create a new image as per my requirement. 8/23

What is Docker Container? •Containers are the ready applications created from Docker Images. •Docker Container is a running instance of a Docker Image and they hold the entire package needed to run the application. 9/23

What is Docker Registry? •Docker Registry is where the Docker Images are stored. •Docker Registry can be either as private repository or a public repository. •DockerHub is a Docker registry allowing multiple users to collaborate in building an application. 10/23

•Multiple teams within the same organization can exchange or share containers by uploading them to the Docker Hub. •DockerHub is Docker’s very own cloud repository similar to GitHub 11/23

Docker Features: •Easy and Faster Configuration •Increase productivity •Application Isolation •Swarm •Routing Mesh •Services •Security Management 12/23

What is Docker Swarm? •Docker swarm is a clustering and scheduling tool for Docker containers •Swarm uses the Docker API as its front end, which helps us to use various tools to control it •It also helps to control a cluster of Docker hosts as a single virtual host 13/23

•Docker swarm is a self-organizing group of engines that is used to enable pluggable backends. 14/23

Routing Mesh : •It routes the incoming requests for published ports on available nodes to an active container. •This feature enables the connection even if there is no task is running on the node. 15/23

Services: •Services is a list of tasks that lets us specify the state of the container inside a cluster. •Each task represents one instance of a container that should be running and Swarm schedules them across nodes. 16/23

Docker Architecture: Docker follows client-server architecture. Its architecture consists mainly three parts. •Docker Client •Docker Host •Docker Registry 17/23

Docker Client: Docker provides Command Line Interface (CLI) tools to client to interact with Docker daemon. Client can build, run and stop application. Client can also interact to Docker_Host remotely. 18/23

Docker Host: It contains Containers, Images, and Docker daemon. It provides complete environment to execute and run your application. 19/23

Docker Registry: It is global repository of images. You can access and use these images to run your application in Docker environment. 20/23
