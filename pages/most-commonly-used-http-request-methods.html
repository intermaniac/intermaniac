There are officially 9 HTTP Methods, however only the following 5 of them are used frequently: ðŸ”· GET ðŸ”· POST ðŸ”· PUT ðŸ”· PATCH ðŸ”· DELETE

ðŸ“Œ GET The GET method is used to request data from a specified resource. GET requests are cacheable and idempotent, meaning they do not change the state of the server (sending multiple requests has no effect).

On success, GET requests return 200 (OK). The GET method is the most widely used and can be used to verify endpoints and make sure the data they are returning is as expected.

ðŸ“Œ POST POST is used to create new resources on a server, and on success, will return a 201 (Created) response code. The request contains the new data in the body and it is created under the URI defined on the request-line.

Because POST requests submit new data to the server, they change its state and are therefore not idempotent. Sending repeated POST requests can create multiple of the same resource.

ðŸ“Œ PUT The PUT method modifies resources and updates the entire resource. Thus, updating a single field within a resource using PUT requires you to send the entire resource in the body (payload) of the request.

If the resource already exists, it is overwritten and will typically return 200 (OK) or 204 (No Content). If the resource does not exist, it will be created and return 201 (Created). PUT requests are idempotent and successive calls just produce the same result.

ðŸ“Œ PATCH PATCH is similar to PUT because it is also used for updating resources, however PATCH is for partial modifications. You do not need to send the entire resource, only the field you are changing. On success, PATCH can return various 2XX status codes.

PATCH is not necessarily idempotent. Most PATCH actions are idempotent, such as renaming an attribute value. However, adding a new item to a resource list using PATCH is non-idempotent.

ðŸ“Œ DELETE DELETE request removes a specified resource. They are idempotent. Once a resource has been deleted, you can verify this using a GET request for the deleted resource, which should return a 404 status code.
