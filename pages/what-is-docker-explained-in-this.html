Docker is a platform or ecosystem around creating and running containers. In other words: Docker is an advanced version of virtualization. But what is virtualization? #devops #kubernetes #devopscommunity #terraform #aws #azure

In simple words: Accessing or using your computing resources remotely. Have you worked with VirtualBox ever? if yes then you can easily understand it. In virtualization, we use Hypervisor to create virtual machines. What are virtual machines?

A Virtual Machine (VM) is a compute resource that uses software instead of a physical computer to run programs and deploy apps. ... Each virtual machine runs its own operating system and functions separately from the other VMs, even when they are all running on the same host.

Now if you want to run multiple operating systems or multiple servers on one machine, you have to create virtual machines for that. But what if you want a lot of different servers on the same machine, it can be massive because you've to preallocate your resources to VMs.

Also, let's see another scenario. You've developed an app on your local machine, assume that app is written in JavaScript(nodejs) Your nodejs version on your local machine is 14. But you want to share your app so you shared your code to your friend, he's using node v15

So there could be problems with running the application on your friend's local machine. So what are these problems? 1. Running your application on other machines(servers) and getting dependencies/version problems. 2. Running multiple servers on your local machine

So the above problems can be solved by Docker. How?? Let's see that
