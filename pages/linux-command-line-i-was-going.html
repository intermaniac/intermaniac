WHY THE SHELL? Interacting with the operating system via command line gives the user more fine-grained control than what can typically be achieved using a Graphical User Interface (GUI). So instead of a GUI we have what is called a Command Line Interface (CLI). 21/

Given the composite nature of the CLI, programs can interact with each other in ways that are not possible with GUIs. Another advantage of a CLI over a GUI is the ability to automate actions. What can be typed at the CLI can also be made into programs via scripting. 22/

There are many different shells available to work in and depending on the particular *nix operating system will determine the default shell. 23/

The most ubiquitous shell in modern Linux operating systems is Bash which stands for the "Bourne Again SHell" and is a play on words as it incorporated, updated and extended the original Bourne Shell developed in the 1970s. 24/

IT IS ABSOLUTELY RELATIVE TO WHERE YOU ARE *nix operating systems use a hierarchal directory structure which means that files are contained in a tree-like structure. Directories can contain files and/or other directories. 25/

There are many system directories in a *nix operating system and depending on which distribution you are using will determine the directory structure, however, suffice it to say that the two most common you will come in contact with are the home and bin directories. 27/

When navigating around the filesystem we move between directories. We move between these directories by typing commands. If we type the directory path using the forward slash then we are using absolute paths, i.e., absolute from the root directory. 28/

If we leave out the forward slash when moving between directories then we are accessing the directory via a relative path, i.e., it is relative to where we currently are located in the file system. 29/

ERR, WHAT IS STANDARD INPUT, OUTPUT AND ERROR? Every time you type a command at command line terminal, you're sending an input stream to shell which interprets this input and does something. That something is sent out as output stream to a special file called standard output. 31/

And if something goes wrong in processing the command then there is an error stream displaying the error. 32/

* Standard Input provides input to command, is called stdin and has a file descriptor 0 * Standard Output provides output of command, is called stdout and has a file descriptor 1 * Standard Error displays error on command, is called stderr and has a file descriptor 2 33/

The default for stdout and stderr is the terminal screen but they can be sent to file on disk for long term storage or re-use. 34/
