Javascript is a single-threaded language that means the whole code will be executed line by line. And this technique of running code is known as Synchronous Programming. #100DaysOfCode #javascript

let's see an example - const a =10; const b =20; console.log(a); console.log(b); console.log("Vaibhav"); what do you think the output will be? the output will look like this - 10 20 Vaibhav And why the output will look like this? because javascript executes code line by line.

As no code consumes time for the execution we've got the sequential output, but if there is a line of code that consumes time for the execution, then it blocks the execution of code in that line which is called blocking code. (1/2) #100DaysOfCode #javascript

This delay in execution may be due to callback functions, fetching data from API, etc, but containing such blocking code may take a lot of time for the execution. So to avoid such behavior we use a technique called Asynchronous programming. (2/2) #100DaysOfCode #javascript

what is Asynchronous programming? Asynchronous programming is like starting now and finishing later. It is a technique that makes the program start running the time-consuming task but it doesn't block other events. (1/2) #100DaysOfCode #javascript

The program or a function that consumes time will be removed from the thread or scope of the execution and the remaining code will get executed. Due to this, the code will not be blocked. (2/2) #100DaysOfCode #javascript

Example- console.log(1); setTimeout(()=>{ console.log("Hello") },10000) console.log(2); So now we know that setTimeout is a higher order function it takes a callback function and time duration as a parameter. (1/2) #100DaysOfCode #javascript

So here setTimeout() function has a time delay of 10 seconds which means the output of the function will be printed in the console after 10 seconds. so the output will be 1 2 hello hello, will be the last output because it will be executed after 10 seconds. (2/2) #100DaysOfCode
