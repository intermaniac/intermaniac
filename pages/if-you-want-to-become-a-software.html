If you want to become a software developer, you must know version control tools like Git. Here are 11 basic commands to get you started. Thread üßµüëá

1. git init: This command initializes the version control flow within your project once executed within a folder on your PC. Both for a new project or existing projects.

2. git status: Allows you to check the current status of your git repository. For example, we can track which files are new, have been modified, and also the files which aren't being tracked by git.

3. git add <file/directory/. (everything)>: This command is used when adding files into what we call a staging area. As all files are held in this temporary location until they are committed.

4. git commit -m "Update play button": It's a step after git add which takes a message as an argument in describing the changes being committed. An approach I use in writing a commit message is by adding the prefix "This commit will..." before every message; Excluding the prefix

5. git pull: This command is used to fetch and download content from a remote repository to update the files on your local repository (your local computer); making sure you're in sync with the latest changes being made.

6. git push: This command is used to push commits made from your local computer to a remote branch/repository which other developers can track and see what was changed from your side.

7. git branch: This command has multiple purposes. It can be used to perform the following operations, which are to create, list, rename and delete git branches. 8. git checkout: It's used when you want to switch between existing branches available in a git repository.

9. git merge: Remember the story I gave about multiple developers working on the same file, when you want to join two or more commits on a public repo together, this command ensures those changes are safely joined on separate branches.

10. git log: Even though many modern IDEs have this integrated within their software, when you want to show all the logged commits, this command displays exactly that for you in descending order; based on the time of the committed message.

11. git clone: Once you find a project online on GitHub, you don't necessarily have to create a new project locally and begin with the hustle of copying those files. You can simply clone the entire project to your local PC and either contribute to it or make something out of it.
