What the Hell is GIt? Git is a version control system used for tracking changes in computer files. It is generally used for source code management in software development. Git is used for tracking changes in the source code. PS: it's easy, just follow along.

The distributed version control tool is used for source code management.

Initializing git Initializing a git repository starts with 'git init' command. If you make any changes, you can see that change using 'git status' command. ‚è¨

Adding and committing Committing means you're making a change in your repository, and you can tell git which files are going to be part of the current change by adding. ‚è¨

'git add [FILENAME]' is used to add one file to the current commit. And 'git add -A' will add all changed files into the current commit. And finally you can push a change into a repository using 'git commit -[MESSAGE]'. A message is usually what changes are included in a commit.

Creating a branch When you're going to make a change and not sure if it'll work or not. You can create a branch. In this case if your code fails, your main repository will not be affected but that created branch. To create a branch use 'git branch [BRANCH_NAME]'. ‚è¨

And if you've more than two branches, you can switch between using 'git checkout [BRANCH_NAME]. PS: It is mostly used when you're working on a new feature or solving a critical problem.! PS: DON'T FORGET TO RT. üôè

Merge When you've done with a newly created branch and everything's working as expected. You will merge that into the main repository. To do that, first checkout to main branch. And then use 'git merge [BRANCH_NAME]'. PS: It may sound like woah. But it's easy usually.

PUSH Push is used when you're uploading your commits into a cloud repository on Github. To push, use 'git push' simple?. Yeah.
