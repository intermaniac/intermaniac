I wanted a good setup for running multiple services and a devnet in docker. Here is what I came up with ðŸ§µ

The repo is an NX/PNPM based monorepo. Nx is a package manager for doing caching and incremental builds https://t.co/ZEClWWLQZN. Pnpm is an npm/yarn alternative that is much faster because it's smarter about how much file system it uses when bundling https://t.co/iM7Fvy175d

Docker images usually start by using another docker image as it's base starting point. First I tried using the official forge docker image. https://t.co/9GjWTW3qkr This had issues though. Getting nx to work in docker container was not easy

Docker has an OS called Alpine. It is a really light and small OS to start with. Foundrys image is alpine based. Node seems to go off the rails fast with alpine unless we stay close with the herd. So I switched to using an official node image https://t.co/BvdWxbeRpN

Additionally, installing a very specific version of node other than the supported one on Alpine is kinda complicated. Installing forge onto any image is pretty straightforward though

Rather than use alpine, I made my life easier by using node:18.12.1-bullseye. Bullseye is heavier, but easier to use as a Docker noob. It's debian based

I build my entire repo so I can run anything via sending different commands into this one image. Later I may use multistage builds to make them smaller https://t.co/TlQsRqcyYx

The last thing I have to debug is setting `ENV NX_DAEMON=false` and I finally have a working dockerfile building my monorepo to run forge and typescript services https://t.co/myKW8mjpin

Now I want to spin up every server plus a devnet in one easy command. For that I use docker-compose. Docker compose allows us to specify how all our services "compose together" in a simple yaml file https://t.co/6Bqr3WPHM1

Most of my services share configuration. I put the shared configuration in a seperate yaml file https://t.co/dlzB3rLbHD

Configuring a service is pretty simple and readable after pulling out the common config into `base-app`. We simply pass in a command to run in our monorepo and what ports to map to outside ports https://t.co/2FbMex3BFA

For anvil, we add this special healthcheck method (later I will add a healthcheck to other services too. We want anvil to be up and healthy before we deploy the contracts https://t.co/pYntNz64w4

Lastly we deploy the contracts https://t.co/SJ9wUoSJ1l

Now we just have to run `docker-compose up` and it spins everything up! https://t.co/5QyP5LiKZZ

My next step is to get another docker image running e2e tests with Playwright. Playwright is what I believe is the best e2e test framework for web3 apps today https://t.co/wlewnS4Lb7

I'm a complete docker noob and learned a lot here. Let me know if there is anything I could do better. The code can be viewed here https://t.co/HzWiHnTHIr https://t.co/nwtTDYPWWy https://t.co/LPpNjh3Mb2
