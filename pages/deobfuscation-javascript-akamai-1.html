Thread: Deobfuscation javascript (Akamai 1.75)

Je vous présente un petit thread d'introduction à la deobfuscation de code javascript via l'analyse d'AST (Arbre de syntaxe abstrait)

Depuis début 2021 la version 2.0 d'akamai est sortie. La plus part des gros sites aujourd'hui utilisent la version 2.0. Aujourd'hui nous nous attarderons sur la version 1.75

Qu'est-ce qu'un AST ? Un AST pour Arbre de syntax abstrait, est une structure de donnée permettant de récupérer le control flow du programme sous une forme d'arbre. Exemple: + 1 * 2 2

En effet si on lit le programme sous forme arithmétique on obtient: 1 + (2*2) Vous allez me dire mais c'est quoi le rapport ? Il faut comprendre que tout langage à son AST. Un langage passe par plusieurs étapes de compilation.

Plan de compilation: ------------------------------------ | Front end | Middle | Back end | ------------------------------------ Lorsque l'on développe un obfuscateur nous modifier le front end afin que le programme ressemble à un tas de code désoptimiser.

Finalement on peut comparer l'art de l'obfuscation par le faite de désoptimiser un code source afin de le rendre illisible. La déobfuscation au contraire est l'art de réoptimiser un code source obfusqué.

Il existe plusieurs technique d'obfuscation: - Control flow graph flattening - VM obfuscation ... Je vous ai donné les deux grosses techniques utilisé dans le monde réel aujourd'hui. Bien entendu il en existe des tas.

Il est possible d'obfusquer n'importe quel langage. Il existe plusieurs type d'obfuscateur. Il y a ceux qui vont obfusquer directement l'assembleur LLVM du langage en question. Et il y a ceux qui vont directement parcourir l'AST du langage.

Aujourd'hui nous allons apprendre à développer un déobfuscateur pour Akamai 1.75. Pour ce faire nous allons utiliser la librairie Babel. Babel est une librairie permettant de parcourir ou encore de modifier l'AST d'un code source javascript.

La librairie Babel vous sembles familière ? C'est tout à fait normal, en effet typescript utilise babel pour transformer l'AST typescript en AST javascript.

Il m'est déjà arrivé d'entendre des personnes disant que le typescript était plus performant que javascript. Je vous laisse relire ma phrase ci-dessus pour comprendre que cette phrase n'a aucun sens. Typescript ne fait que transpiler du code typer en javascript.

Akamai 1.75 est vraiment très simple à déobfusquer dynamiquement. Nous avons un tableau de string qui va stocker chaque nom de fonction... https://t.co/rvK7rUfcxO

Ce que vous voyez c'est simplement les string en ascii correspondant aux noms des fonctions. Deuxième chose à repérer c'est qu'il y a certaines opérations mathématique. Nous allons donc faire du constant folding afin de simplifier cela.

Le constant folding c'est une étape de l'optimisation des compilateurs qui consiste à évaluer à la compilation les calculs mathématique. dans les variables.

Troisième point nous allons simplement replacer les index dans le script par leur valeur directement correspondant à la position dans le tableau de string. C'est les trois step pour rendre Akamai 1.75 lisible voici un code source que j'ai développé pour faire ceci.
