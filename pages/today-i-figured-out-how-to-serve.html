Today I figured out how to serve an AWS Lambda function from a custom domain! It took me, no kidding, five hours. My issue thread of notes is 77 comments long. I was going to write it up, but I'm exhausted so I'm sharing my research issue thread instead: https://t.co/fE5m9W2acN

AWS Lambda functions can have Function URLs now, which is great! But giving them a URL from a custom domain/subdomain is not straight-forward at all: I ended up figuring out a way to do that using CloudFront Demo at https://t.co/IV10bsUlDt

Of course you also need to issue a certificate to do HTTPS for that custom domain, which means figuring out AWS Certificate Manager (ACM) And you have to verify ownership of your domain If your DNS is with Vercel, there's an edge-case you have to handle https://t.co/uawoxIkRfF

And finally... Datasette needs an x-forwarded-host: header to generate internal URLs, but CloudFront can only provide that if you write a custom CloudFront Function (NOT the same as Lambda@Edge functions) to set that header! https://t.co/3zljJSD2Zk https://t.co/R8PHZLNwdC

This issue thread is an extreme example of how I handle all sorts of different research tasks: I open a research issue, then drop in a stream of comments with URLs, notes, quotations, code samples and screenshots until I figure out the solution

Here's my write-up of my previous adventures figuring out how to deploy a Python ASGI application as an AWS Lambda function https://t.co/cwgPlW3vTF
