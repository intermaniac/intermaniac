👀 It's been a long time since the last scripting thread, so here we go... ⌨️ (not) Daily #Roblox scripting thread 7! ✨ FUNCTIONS, SCOPE AND RETURNING 🔙 A #RobloxDev thread! 👇

What are functions? 🤔 In simple terms, they're pieces of code that are just saved somewhere in the computer. 🖥️ The important part about them is that you can CALL them whenever you want - you just need to have a REFERENCE to them. 📲

Now, before you go like "but Actulurus i don't know what a reference is!", 🤫 In this case, "reference" means a variable - it means to have "access" to the function. Can't call it if you don't know its phone number!

So, how do we call a function❔ Well first, we need to DEFINE it. 🤔 There are two ways of doing this and both are explained below 👇

The first way is the "very cool" way 😎 Use the 'function' keyword followed by the function Name - this will be the function's "phone number" when you want to call it 📲 Then, type an opening and closing bracket right after the function name - no spaces, otherwise I'd be angry!

In those brackets goes what we call "parameters" 🤔 These are used to give info to the function, for example if we have a function that should destroy a car, it needs to know which car to destroy! 🚗 And lastly, don't forget to close it with the 'end' keyword. For example 👇 https://t.co/766ns16Dly

The second way of defining a function is writing it directly to a variable - the value of that variable will be the function! 🤓 We don't include the function name like we did previously - the function's name is now the variable name. 🤔 So that looks like this 👇 https://t.co/khAdfxJv4e

Now, how do we call the function? 📞 Pretty straightforward, we just type the function name followed by those brackets again. 💭 Guess what's in the brackets? That's right! The parameters! Well, except for that they're called 'arguments' now. 🤔

Why? I don't know. In the function, you call them 'parameters'. When you call that function, they're 'arguments' 🫤 It's been like this for ages so you better stick to that! ‼️

Still here? No? Come back! That was the easy part. 🔙 Okay so now what does "scope" mean? 🔭 I've already covered this in one of my previous threads, but without a lot of detail. 👍

Basically, it's all the variables that the code that's currently being run can currently access. ✅ If you define a variable inside a function, something that's outside of that function has no idea that variable even exists. 🤔

This is where GLOBAL variables come in. As suggested by their name, they completely ignore this scope stuff. ❌ You can define them in a small for-loop inside 45 functions and you'll still be able to read them outside. They just have to exist when you read them, of course 🤔

Learned something new today? Make sure to drop a foll- but wait! There's more! ✋ How do we access variables that were declared in a function, from outside of the function? 🤔 Returning. It's a very cool thing that let's the functions give us some info back. 🌴👍

For example, if we have a function that calculates my monthly revenue, it'd return 0. 📉

It's simple and works similarly to parameters. 👍 We use the 'return' keyword to signify that we are returning a value. After that can be multiple values separated by a comma. ✅

But we still need to retrieve those values outside of the function. 🤔 This is how it's done 👇 We declare a local or global variable with any name we want. Then, we set its value to the function, which we call with any arguments. 🤔 Hard to explain, but this is how it looks: https://t.co/r3xnwvh7A8

Keep in mind that if you're returning multiple values, you also need multiple variables - separated by a comma 👇 https://t.co/yJ0KtWj6gL
