1/ @rootsecdev published a blog post where common misconfigurations inside the Conditional Access Policies in Azure are discussed. In an Azure Tenant from a customer, the following CA policy was implemented: Require MFA for administrative users. ðŸ§µ https://t.co/GqppjLaMB6

2/ However, within the Directory roles checkbox, not all the roles were selected (see the picture below). In Azure Assessments, I use, among others, the script Get-MsolRolesAndMembers.ps1 to find users which are part of such different roles. [2] https://t.co/UhYnklVUZO

3/ The users or roles found with the mentioned script must be cross-checked with the CA (the checked roles from the menu above) to find possible users which could log in without MFA, resulting in a security gap. ðŸ”Ž

4/ "Blocking Linux operating systems used to not be available in Conditional Access. It's worth a review though because there have been times when I have gained unintended access simply by using a Linux operating system such as Ubuntu to log in." [1]

5/ Searching for MacOS and Linux operating systems in the User Agent strings is one of the hunts I regularly perform in IR cases or compromise assessments. This simple search could find compromised accounts, mainly if the company only uses Windows systems.

6/ As you can see in the picture below, the user used a Windows operating system for the logins in the last weeks, but two logins were performed with a Linux (Android) system. This could be legitimate but needs to be clarified further (in our case, the login was not legitimate). https://t.co/wYT7Bv1ziF

7/ "By default, every user in your organization has access to Azure Active Directory. If an end-user is compromised with a weak password, then an attacker has access to all your email addresses and phone numbers." [1]

8/ In the Tenant from the first screenshot above, we also find the CA Policy below, which is precisely the same as outlined in the blog post and is another hurdle for attackers. Implement as much such controls as you can. https://t.co/2kE3Nuk4SY

9/ Unfortunately, we see the additional recommendation "Restrict access to Azure AD administration portal" repeatedly not implemented in our assessments. Restrict this access if you can (even if the Azure Active Directory data could still be read via PowerShell). https://t.co/Bft688TGGX
