When I started React I didn't know how much JS I should've known. What if I miss out on some concept that is super important for React? What if I learn something that is irrelevant now? These questions troubled me. If you're in the same boat, open this thread.

First, let's start with the very basics. Javascript has changed a lot in the last few years. One of these recent changes was the introduction of keywords like const and let. They provide new ways of declaring variables.

Const creates a variable, the identifier of which can't be assigned to something else afterwards. So, if we say const num = 12, we can't change num to some other value. But there's a catch here!

If we declare an object using the const keyword, we can modify the object even if it was declared using const. For ex - const arr = [1,2,3] we can still do arr.push(4) and arr will become [1,2,3,4]

This happens because the arrays in Javascript are of object data type and since we're not changing the reference here, it is legal to do so. Didn't get it? Let me explain it a bit more. üëáüèΩ

const arr = [1,2,3] creates a reference to the memory location where the array is stored. Our identifier 'num' can't be reassigned to something else. When we modify the array, we are changing the contents of the array, while the identifier is still assigned to the same array.

On the other hand, if we write const num = 123 and try num = 456, it won't work as we're reassigning the identifier to something else entirely. When learning React, always use 'let' and 'const' to declare variables. Use of the old keyword 'var' isn't recommended.

Now, let's see what else in Javascript is needed to become a good React dev. After variables, one should also know how to play with Arrays. Make sure you have a firm grasp of the following array methods: - push() - pop() - map() - filter() - reduce() - forEach()

In addition to these you should also know about: - callback functions and how they work, - Arrow functions, function references, function expressions, - Promises, async-await, try-catch, - destructuring assignment

Now, let's move to the Objects concepts: - Object literals, - bracket and dot notation and the limitations of both - Constructors and object methods, - the keyword 'this' and how its scope changes and makes it behave differently - using 'bind' along with 'this' keyword

Then, if you want to work with older React code, classes become super important so study that too. But if you're only going to be working with 'new' react code then you'll be using hooks so classes won't be of that great use. However, studying never hurts, so pick wisely!
