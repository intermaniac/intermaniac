Unlike the EVM where the smart contract language is Solidity, on @tezos, their smart contracts are expressed in Michelson a low-level Turing-complete stack-based language that includes common and blockchain-related features.

However, since Michealson is not very easy for humans to read and write. A number of high-level languages are available on Tezos, which allows developers to write smart contracts in very expressive, easy-to-read forms, these languages then compile down to Michealson.

The @pykampala is a python community and what better way to build on the @tezos #blockchain than through smartpy, a python library that is an intuitive and powerful smart contract development platform for @tezos.

Lets explore a basic calculus smart contract that allows us, to add, multiply, divide and get the factorial of a given value. And because security is really key, well we have to run tests. https://t.co/oe255kcroG

With that simple script, you'll have yourself your very first tezos smart contract. The smartpy IDE provides you with a virtual environment that you can use to test out your code and deploy it to the blockchain./ https://t.co/LwM58SuUA2

Now that you have written your first contract, let's check out what an NFT contract would look like on the @tezos #blockchain using Smartpy. https://t.co/gmP6QnzcKa
