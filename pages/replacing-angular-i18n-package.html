Replacing #Angular i18n package. About 2 months ago, we started out a journey, to replace the not-so-bad i18n package in Angular. First week, we created the Translation pipe, replaced the Plural and Select features, and added Locale Id token. A thread üëáüèº https://t.co/RqhYgJyDto

Second week, we moved the language scripts, and the injected locales to provide externally, to the same build, then we created a stand alone #expressJs server to serve different languages driven by cookies. https://t.co/fSzDGDjx43

Third week, we added rules in #ExpressJS for a URL driven app, and created a express template engine to replace the base href, also turned it into SSR, and used `ngExpressEngine` to replace the base href. https://t.co/8f0Dwk00C6

Forth week, we experimented some more with the template engines, to replace the whole index file after build, and found out about an undocumented feature of the ngExpressEngine "inlineCriticalCss" https://t.co/4tp9ySbwgn

Fifth week, we created the UI component to switch language, whether by setting a cookie, or redirecting, and we goofed off a bit with the Woolong currency to see how far we can manipulate Angular locales. https://t.co/xiDjCTq298

Sixth week, we created an Angular builder, and a Gulp task, to generate the different index.html files, to serve them statically in our Express app. https://t.co/Nwie6FWsCV

In the seventh week, we threw it in #Netlify, and figured out a better way to serve languages, using APP_BASE_HREF token in #Angular. https://t.co/CBBA4sXbu1

Eighth week, we locally hosted on #Firebase and #surge, we had to write a serverless function, or work around client redirects for friendly URLs. https://t.co/UrnKm8wfQw

And today, we created an extract #Angular builder, and a #Gulp task, to extract translation terms before build. And we made a final conclusion, well, at least I did. What did you think? was it worth it? https://t.co/axAJE5tbo0
