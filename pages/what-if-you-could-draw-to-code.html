This is a very early conceptual exploration, and by no means a commitment to include it as functionality in Henosia. It's meant to demonstrate how everyone, including designers and non-coders, can interact with code in an intuitive, productive and playful way.

The components used here are from @MUI_hq. They are the real production components. Henosia will support using your own design system, and will have out-of-the box support for a number of popular open source design systems.

The reason I used Play/Interact to change the rating at the beginning was to demonstrate how the Henosia editor and code engine works in harmony with existing logic. In this example the Rating component uses a React hook to update and store the rating state.

On the right is my IDE with the file open that Henosia is editing visually on the canvas. For this project, code is formatted using Prettier. Henosia writes code of a quality that is on par with traditional IDEs. Code that's ready to be used in your product via Git and PRs.

Henosia reads code as much as it writes code. To write code, Henosia has to understand what's already there. This read-write flow happens in real time while you design, and allows code changes written by developers to be used in Henosia via a Git pull. True two-way collaboration.

To extend on what "Hensoia reads code" means in terms of value-add: Henosia is designed to easily open and read your existing code bases (uses Git clone). The goal is to enable you to continue work on existing coded design systems and your full digital product inside of Henosia.

Bottom line is that we've just only begun to explore code-as-data and code-as-executable in what it means for UX Design and collaboration in digital product team as a whole. https://t.co/fg4QGQJnl2

We're still an early Alpha, but we'd love to learn from your feedback to make it the most useful tool it can be. We have the Henosia User Group (HUG) for this very purpose, and HUG'ers will be among the very first people to get access to Henosia. https://t.co/voPTZI7zWr
