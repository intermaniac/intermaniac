Did you know you can build a Wordle using *only* #TypeScript types? Here is how. Explanations in thread 🧵👇 https://t.co/DhgtLxceok

First, let me tell you a secret 🤫 TypeScript’s type system is much more than just type annotations. It’s a full-fledged programming language! I call it Type-level TypeScript and I’m currently working on an online course about it.

Now, how does this work? There are 3 main features at play here: 👉 Conditional Types 👉 Template Literal Types 👉 Recursion Let’s dive into it 🤿

1. Conditional Types are just Pattern Matching. `A extends B` checks if A is assignable to B. But you can think about it as asking the question “does A match the pattern B?” `infer` declares local variables by destructuring the type A. https://t.co/VOGZujKe71

2. Parsing strings with Template Literal Types Inside a conditional type, template literal types can: 👉 Split a string in several parts 👉 check if a letter is included in a string https://t.co/m0U4PnnoD5

3. Loops with Recursive types. Conditional types can be recursive! This enables us to write loops of arbitrary complexity. I told you the type system was a programming language 😄 https://t.co/yu3ieX53c1

Granted, this is pretty advanced stuff. 🧠 Hard to come up with this kind of types without a good understanding of the fundamentals of this language. But it's fun, isn't it? 😅 Here is an online playground with the final code 👉https://t.co/ACHaF0aAa3
