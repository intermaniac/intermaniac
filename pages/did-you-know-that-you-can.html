Did you know that you can configure different .gitconfig instances for different file directories in Git? I'll try my best to make this thread beginner-friendly but you're free to ask any questions if you don't understand any of the details. Let me explain üëá

Note: I'm using macOS but this can be done in any operating system if you have Git v2.13.0 and above installed. Let's get started.

First you need to find your global .gitconfig file. This file is placed in root directory (~/.gitconfig). Use your favorite editor to open it. The command "code ~/.gitconfig" to opens it via VS Code if you have "code" command installed correctly.

You'll see something like this if you configured your credentials before: https://t.co/5pBjEoogbg

Now we'll add some lines that conditionally adds another .gitconfig file with different settings: [includeIf "gitdir:~/customer-a/"] path = .gitconfig-customer-a "customer-a" is a directory that I created under the root directory. https://t.co/IXeQNSuMQw

I want to use a different email in Customer A's projects, therefore I create a different .gitconfig file called ".gitconfig-customer-a" in the root directory as well. This file can contain any setting available in here: https://t.co/gfS0Zz2VK9. I'll only change my user settings. https://t.co/GoApZ1UseC

That's it. Now my config changes depending on the directory I'm in. If I create a test repository under the "customer-a" directory and I run this command to check what is my configured email, I'll see that my customera email is being used. https://t.co/61LEkTTL8o

All the other directories still use my personal config settings. You can think it as a fallback mechanism. Feel free to ask any questions, I'll do my best to help. Happy coding üë®‚Äçüíª
