regular expressions (regex) is an object that is used to match a sequence of characters in strings. this is used to define search patterns, input validation and so on. regex are executed using the exec() and test() methods in javascript.

the code below defines a regex pattern. the pattern is: any five letter string starting with b and ending with s. https://t.co/lkwtvenjbd

there are two ways of creating regex in javascript: - using a regular expression literal (pattern enclosed between forward slashes) - using the regexp() constructor check codes below https://t.co/mwgzohetdl

metacharacters to specify regular expression patterns, we use metacharacters such as the ones below: [] . ^ $ * + ? {} () \ |

square brackets [] this is used to specify the set of characters you wish to match. for example: [xyz] will match if the string contains "x, y, or z"

period . we use the period to match any single character except a new line. for example: given two periods .. a -----> no match ab -----> 1 match abc -----> 1 match (contains 3 characters) abcd -----> 2 matches (contains 4 characters)

caret ^ we use a caret to check if a string begins with a specified character. for example: ^ca cat -----> 1 match ‚úÖ carrot -----> 1 match ‚úÖ corn -----> no match ‚ùé

dollar $ we use a dollar to check if a string ends with a specified character for example: g$ g -----> 1 match ‚úÖ dog -----> 1 match ‚úÖ doge -----> no match ‚ùé

star * the star symbol * matches zero or more occurrences of the pattern at the left hand side of it. for example: ba*n ban -----> 1 match ‚úÖ banner -----> 1 match ‚úÖ barn -----> no match ‚ùé ( a is not followed by n)

plus + the plus symbol + matches one or more occurrences of the pattern at the left hand side of it. for example: ba+n bn -----> no match ‚ùé (a is missing) ban -----> 1 match ‚úÖ bann -----> 1 match ‚úÖ barn -----> no match ‚ùé ( a is not followed by n)

question mark ? the question mark symbol ? matches zero or one occurrence of the pattern left side of it. for example: ba?n bn -----> 1 match ‚úÖ ban -----> 1 match ‚úÖ baan -----> no match ‚ùé ( more than one "a" character) barn -----> no match ‚ùé ( a is not followed by n)

braces {} this contains 2 parameters, where the first one specifies the least, and the second one the most repeatitions of patterns at the left of it. for example: b{2,3}. this means at least 2, and at most 3 repetitions of the pattern b at the left side of it.

alternation | we use the vertical bar to match specified characters located at alternating positions for example: a | b ade -----> 1 match ‚úÖ ( which is a) acdbda-----> 3 matches ‚úÖ ( a..b.a)

parentheses () parentheses () is used to group sub-patterns. for example, (a|b|c)xyz match any string that matches either a or b or c followed by xyz

backslash \ a backslash is used to bypass or escape characters and metacharacters. \$a is used to escape the $ and a in this case, the interpreter does not treat $ as a special character.

asides the regexp() methods, we can also use different string methods that allows us pass regexp as parameter. some of these methods include: - match() - split() - replace() - search()

flags there are optional flags that allow for functionality like global searching and case-insensitive searching. they are part of regex and can be used together or separately.

g -----> global search i -----> case-insensitive search m -----> multi-line search etc
