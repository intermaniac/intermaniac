Data Engineers are primarily tasked with ETL - One of such tools on AWS is Glue, which is a server less , you do not need to worry about provisioning servers ,installing libraries etc. Now how do we run a glue job using Terraform.

Step 1 Define your terraform script start with setting up your AWS creds and region you want to create your ETL workflows https://t.co/BUIkcuwe8X

Step 2 Setup your storage - A S3 bucket to hold your python scripts and cloud watch to store your logs from the Glue jobs https://t.co/KVunWCXxiU

Step 3 Setup IAM roles to provide proper interface between how these different services communicate with each other. https://t.co/3luUVuCIqg

Step 4 Final step , write your Glue job and point it to the script you want to run In this example it will run hello_world.py from S3 bucket we defined earlier. https://t.co/v0u4aXP1YV

Now, you are all set to provision your infra on cloud , before you run the job - Manually upload a simple script to S3 bucket defined earlier and kick off your job. Congratulations, you just finished coding up your first Glue job. https://t.co/5JRZIPsRWL
