Recently I was discussing some PR's and ChatGPT with @rom1504 and he thought of a cool idea - have it summarize changes in pull requests. We'd like to present GPTReview (https://t.co/5CF20mSk9k) - a GitHub bot that summarizes PR's and suggest improvements! ðŸ§µ(1/N)

here's a few examples of GPTReview in action: 1. https://t.co/MrSOlMsd6X - adding a new training mechanism for open_clip 2. https://t.co/mfK3yKmT93 - support for different precisions in @karpathy minGPT (uh oh GPT reviewing GPT) (2/N)

3. https://t.co/xZWP4j6GQd - model parallelism in @huggingface transformers 4. https://t.co/3K8kdMBl8H - AltDiffusion in @StabilityAI StableDiffusion 5. https://t.co/2JX4aNH56l - adding PlotJuggler VideoViewer in @comma_ai openpilot (3/N)

the basic idea is to detect a comment on a pull request, retrieve the .patch file of the PR where the comment was posted, and input that into the model along with some prompt like "Can you summarize this PR for me and suggest some improvements?". (4/N)

the README has a 3-step tutorial in the README on how to integrate this bot into your repository and there are plenty of improvements that could be made. These days I'm pretty busy so if you're interested in the idea, submit A PR! (5/N)

although the inspiration was ChatGPT and it's impressive ability to summarize code snippets, I decided to use the OpenAI API and summarize the changes with text-davinci-003 which ChatGPT is based on. (6/N)

while working on the project I actually used ChatGPT for consultation quite a bit since I had never made a GitHub bot before and it was pretty useful for some tasks and a bit misleading for others. (7/N)

in particular it's very useful for high-level ideas to get the gist of a task but if you need detailed code it might get things wrong and be very confident in it's predictions which is even more misleading for someone who hasn't worked with a specific technology. (8/N)

also, when I was discussing creating the bot with ChatGPT, it kept insisting that I should give it write access to the repository because this is considered a "best practice"... might want to keep an eye on this one @sama. (N/N)
