Tutorial for beginners in Open source for free: Thread 5: Basics: What is a GitHub repository, how to create one & what are its types? OR How to upload your project/notes/cheat-sheet from your laptop to GitHub ? https://t.co/sAiUeLjmqo

-For creating a GitHub repository, you first need to have a GitHub account. -Follow the below steps to create/sign up to a GitHub account:

Create a GitHub account Go to https://t.co/F4XgEASqRd -> sign up-> Enter your email with which u want your GitHub account to link-> Create a strong password Then enter a username(it's a name for your GitHub account), u can give any name as u like. https://t.co/T8sqjoNtxT

Solve the puzzle-> Create account. Then, u will get a code on your mail that u signed up with, enter that code. Now u get a page(Welcome to GitHub) -> skip personalisation. Congratulations !! you have signed up with your mail on GitHub. https://t.co/PUN536puYJ

What is a repository? It contains all your project’s files. If you upload a project that you have on your laptop/PC to GitHub (open source platform), you save the files & folders of your project in a repository on GitHub.

Types of repository: 1.Public: publicly accessible to everyone 2.Private: You choose who can access your repository https://t.co/cGQFmBh0qG

How to create a repository on GitHub: 1.Go to https://t.co/bhTjVNzc5F & after signing in with your email id, go to ‘New repository’ (as in screenshot). https://t.co/vNwnZ4rWRS

2. You will get something like this. 3. Give a name to your repository, and then choose whether you want the repo (repository) as public or private. Then select the “Add a Readme file” checkbox, it will create a Readme file. https://t.co/rgBKRZ6YUI

Readme is a file that contains the description of your project such as what is your project all about, what tech stack you used etc.

Note: If you select the type of repo as private, then it is not open source project. To make your Open source project, make sure your repo is public & has a beginner-friendly Readme file so that others can easily understand your project.

https://t.co/lO6UrwFqj4 on ‘Create Repository’ https://t.co/pyrqNqpoZr will get something like this https://t.co/kmRvDLnJWd

https://t.co/jbVYrqBU3l on the settings icon that is at right of “About”, you will get a window like this. Fill in the short description of your repository & also fill in the topics. Topics mean which subject areas are your repo related to. https://t.co/BeypACfRgK

EG: If you have made a portfolio website project using HTML, CSS, JavaScript, then your topics will be HTML, CSS, JavaScript, Portfolio, Website & any other keywords related to your project.

What is the use of Topics ? It makes your project/repo easy to search for others. EG: If someone wants to contribute to a JavaScript project, they will search for JavaScript topic, then it will show your project also in the search results. https://t.co/10Fu2JXkK4

-To search for a topic: Just go to GitHub-> Topics or simply go to https://t.co/Db6sNSEf5O (as in screenshot). Here, you will get “All featured topics” (a list of all topics) & some “Popular topics” or you can simply search for a topic in the Search bar. https://t.co/7jV3apKZeD

7.After filling in the topics in step-6, just click on Save changes. You can now see these details in the About section. https://t.co/Lrde05lkNi

8. Now, edit the Readme file & add a detailed description about your project by clicking on ‘pencil’ icon (as in screenshot). 9.After editing Readme file, Commit changes. When you commit, the changes get saved as well as creates a record of who made the change. https://t.co/YN6AlnoQRz

10.After creating Readme file, you need to add your project files from your laptop/PC to the repo. For that, just go to “Add file” -> “Upload files” You get something like this, just go on “choose your files” & drag and drop your files there. Then click on ‘Commit changes’. https://t.co/hULvo6sCGD

Congratulations !! you have uploaded your project on GitHub. That's a wrap for this thread. Thanks for reading !!
