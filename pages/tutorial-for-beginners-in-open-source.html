Tutorial for beginners in Open source for free: Thread-2: Basics: What is Git, situation before & after Git & it's benefits

Situation before Git was introduced: 1. There were developers who worked on a project & each of those developers submitted their code on the central server. But they themselves didn’t have the copy of that code on their local system.

2. The developers could not interact with each other: so, developer 1 was not aware of the changes done by developer 2 & developer 3 and so on. https://t.co/rwYCmaU7U2

Limitations faced by this system: https://t.co/Ow9dlYk9wQ interaction among the developers https://t.co/wFlT7vNIcj tracking of code changes 3.Code stored on central server & if it fails then the whole system fails https://t.co/uc1kSrtXS6 backup

Situation after Git was introduced: 1. There is a distributed server (instead of centralized). 2. Each developer can work on different tasks as well as tech stacks at the same time with the help of branches. For EG: Developer 1 is working on Java, Developer 2 on python & so on. https://t.co/R7eKOg01xt

Branches are like the branches of a tree where each developer working on different task has a different branch & when they complete their work, their branch/ their work is combined into the distributed server (main branch).

3. All developers can interact with each other, so they are aware of the code changes done by each of them. 4. All developers have a copy of the code stored on their local machines as well.

What actually is Git ? • Git is a version control system (VCS): VCS is used to track changes that are done to the code over time. • Git is a distributed VCS. • Allows multiple developers to work on different tasks at a time (using branches) & also interact with each other.

Benefits of Git: • Tracks history/ code changes • Is free & open source • Supports parallel development (as multiple people can work on tasks simultaneously) • Creates backups (as all developers have the copy of the code on their local)

• Supports collaboration • Distributed environment • If the server fails, there is no loss of code (as there are backups)
