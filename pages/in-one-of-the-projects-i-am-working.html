In one of the projects I am working on, I found a very interesting thing /bug about safari browsers on iOS and iPadOS. I'll share the whole story. A Thread ðŸ§µ #web #javascript #FrontEnd #webdevelopment #mdn #iOS #bug #softwaredevelopment

So, I am building this app called Task Manager which is inspired by the Jira board. The app is simple, you can create a few projects, and inside projects, we can create tasks. Thread (1/n)

And you can interact with the task by choosing if it's in todo, in progress, in development, in review or completed, and also an option to delete the task. So projects are the parent container and tasks are the child containers of it. Thread (2/n)

So, since tasks are created dynamically, and there can be many child tasks created, it's not advised to have event listeners on each and every child task, and what is advised is to use event delegation. Thread (3/n)

In brief, event delegation is a technique to handle event listeners for similar elements effectively, what we do here is, add an event listener to the parent element instead of adding it to all similar elements. Thread (4/n)

And this is all possible because of capturing and bubbling of events in javascript. After building the app to a certain phase, I checked on my iPhone and realized that events are not triggered and there is no action happening. On diving into this ðŸ‘‡ Thread (5/n)

I found out that Safari Mobile 7.0+ suffers from a bug where click events aren't fired on elements that aren't typically interactive (e.g. <div>) and which also don't have event listeners directly attached to the elements themselves (i.e. event delegation). Thread (6/n)

As of now, the know workarounds are : - Set cursor: pointer; on the element or any of its ancestors. - Add a dummy onclick="void(0)" attribute to the element or any of its ancestors up to but not including <body>. Thread (7/n)

- Use a typically interactive element (e.g., <a>) instead of one that isn't typically interactive (e.g., <div>). - Stop using click event delegation. Thread (8/n)

I am attaching a link to MDN docs where the above are listed. Also, do let me know if you have faced this situation and how did you try to solve this issue. Thread (9/n)
