In JavaScript, the this keyword refers to an object, and that object depends on how this is being invoked (used or called). Sounds interesting? Let's dive more into it..

The this keyword refers to different objects depending on how it is used: https://t.co/ZhWDr3mulz

1. this in a Method: When used in an object method, this refers to the object. In the example on top of this page, this refers to the person object. Because the fullName method is a method of the person object. https://t.co/2nuIisNVs0

2. this Alone: When used alone, this refers to the global object. Because this is running in the global scope. In a browser window the global object is [object Window]: https://t.co/sxzWy9Aw1y

3. this in a Function (Default) In a function, the global object is the default binding for this. In a browser window the global object is [object Window]: 4. this in Event Handlers In HTML event handlers, this refers to the HTML element that received the event: https://t.co/LHAr3XWV8E

5. Object Method Binding In these examples, this is the person object: i.e. this.firstName is the firstName property of this (the person object). https://t.co/xjpHfJirXO

6. Explicit Function Binding The call() and apply() methods are predefined JavaScript methods. They can both be used to call an object method with another object as argument.

The example below calls person1.fullName with person2 as an argument, this refers to person2, even if fullName is a method of person1: https://t.co/LejDQjtUVi

7. Function Borrowing With the bind() method, an object can borrow a method from another object. This example creates 2 objects (person and member). The member object borrows the fullname method from the person object: https://t.co/VHbjRQ2aHt
