2/ Regulators After the "cold summer" of 2022, many exchanges (DEX & CEX) failed to deliver on their promise of stability, causing many to lose significant amounts of capital. This caused a response from regulators to step up their involvement in the field.

3/ Tornado Cash - with @WilliamSantiago As a result, addresses related to Tornado Cash was blacklisted, and all USDC funds were frozen. Their Github repo was also deleted, and contributors were banned and/or arrested.

4/ The Slippery Slope Code is speech, thus obligated to be protected by the First Amendment. Banning addresses & Github accounts involved with a neutral tool is a slippery slope that can lead to leeways for regulators to soft ban speech "legally"

5/ Sanctions The point of blockchain encryption is to communicate and act freely, anonymously. Sanctions on Tornado Cash is only the start, showing regulators they can intervene & limit this ability.

6/ Consequences Regulators aiming to make ETH transactions non-fungible, traceable & controllable? ETH 2.0's PoS structure must be questioned. If 51% of validators choose not to validate a transaction, it does not go through What happened to censorship resistance & anonymity?

8/ Solution II: Decentralized Domains Nowadays everyone has a device that can potentially be used to run a node. This opens up the possibility to host decentralized domains. This allows apps & services to have full uptime, without needing to worry about being shut down.

9/ Centralized Github Github is the arbiter of accessing your own code. There has been instances of false DMCAs taking down projects, i.e. the popular Youtube Downloader. Not only can you lose your code, you can lose all important metadata that comes with it.

10/ Restricted Countries Github is restricted in some countries. Key repos and code is not available for use by everyone, everywhere. By traveling to a restricted country (even temporarily), you risk getting locked out of your account.

11/ Decentralized Github Github can instead be run on a distributed network of nodes. This guarantees that: Even if the main server goes down, you'll still have access to your code. No central authority has monopolistic control. Allows ability to access from anywhere.

12/ @giteaio's Motivation A decentralized code sharing platform will allow users to control their data themselves. Users will able to build projects without having to worry about getting cancelled overnight, or any of the other negative effect of centralization
