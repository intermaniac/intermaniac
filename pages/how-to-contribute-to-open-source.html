How to Contribute to open source with the GitHub Web Interface: GitHub's web interface is excellent for contributing when you are just getting started with OSS. Check out this step-by-step thread with screenshots üëá

WARNING! Please don't go to the repository I showed here and make the exact commit. This thread is meant as an overall guide for using the GitHub web interface and not how to contribute to `awesome-for-beginners` repo. Use your common sense.

Fork the repo you want to contribute to by clicking the fork button. This will create a copy of the repository in your GitHub profile. https://t.co/ow439GWzhO

Go to your version of that project. Make the changes like file edits by clicking the pencil button ‚úèÔ∏è, you can also add or delete files from the UI. By making changes to your own copy, you can keep the original project untouched. https://t.co/ioQ0bWEU9o

After making the changes, commit them by filling out the form, - Write a compelling title for your PR. - Add a detailed description explaining what you did and why you did it. - Create a new branch name to identify your change like: pr/your-change. https://t.co/J4zSIHGdV5

Now go back to the original project by clicking the small link under your project url: https://t.co/dcXvrdDzCg

Now over here, you will see a new button for you to Compare & Pull Request. https://t.co/04EKHw9Yxb

You will be presented with another window to commit your changes. This time, it will be auto-filled with your commit info. If you wish, you can edit your commit message and description. After you are done, click the green "Create pull request" https://t.co/w10Zs9zufe

Now you can go to the pull requests tab and see your PR. https://t.co/MsZXrHTCLM

Within a day or two, you will receive an email from GitHub about your PR. If your PR gets accepted, it will be merged into the code. If the maintainer needs some changes, she will give you further comments and instructions. https://t.co/dC0jSrsEiJ

In October 2022 I participated in the annual #Hacktoberfest. This was my first time getting involved in open source. If you would like to take part in open source even after the event, check out my full article with my journey https://t.co/0FuCZ5l0Dw
