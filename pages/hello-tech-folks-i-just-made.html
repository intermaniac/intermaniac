Hello Tech folks I just made my first Java Spring Microservice project that combines the knowledge I gained mainly from two different teachers: @Java_Brains and @In28Minutes. Following is a thread : #java #javabrains #Microservices #SpringBoot

@Java_Brains @In28Minutes Inspired by IMDB, proactively designed, implemented, and deployed a Movie-Catalog Microservice application that provides REST resources of movies watched by users and ratings given to them. @DailyCodeBuffer

@Java_Brains @In28Minutes @DailyCodeBuffer The application returns consolidated data by communicating to other applications in the microservice architecture via Spring Cloud Feign Client. Used an external API provided by the "Movie Database" to fetch movies via Spring RestTemplate. @springframework @springboot

@Java_Brains @In28Minutes @DailyCodeBuffer @springframework @springboot Created an API gateway for cross-cutting concerns and implemented Netflix's Eureka client-side service-discovery server, which registers all the services in the system by their names and thus helps in auto-scaling, and fault-tolerance. @springcloud

@Java_Brains @In28Minutes @DailyCodeBuffer @springframework @springboot @springcloud Implemented Circuit Beaker pattern, which breaks inter-service communication based on past requests, checking faulty service's health and thus reconnecting when it is ready. @springcloud

@Java_Brains @In28Minutes @DailyCodeBuffer @springframework @springboot @springcloud BulkHead pattern, which effectively handles a slow or unresponsive microservice by assigning a limited number of Threads using Netflix's Hystrix configuration to make the application Fault-tolerant, Resilient, and Scalable. #SpringCloud #NetflixHystrix

@Java_Brains @In28Minutes @DailyCodeBuffer @springframework @springboot @springcloud Implemented config server that fetches service-specific configurations from a GitHub repository and distributed tracing using Sleuth, which gives a unique ID to each request, and used RabbitMQ as an intermediary for asynchronous communication between services and the Zipkin srvr

@Java_Brains @In28Minutes @DailyCodeBuffer @springframework @springboot @springcloud Containerized the application using Docker and deployed it on Google Kubernetes Engine, which provides auto-scaling, fault tolerance, service discovery, health monitoring, and zero downtime deployment. #SpringCloud #SpringBoot #Docker #Kubernetes #GKE #Cloud #Microservices

@Java_Brains @In28Minutes @DailyCodeBuffer @springframework @springboot @springcloud Technologies used: Java 8, Spring Boot, Spring Cloud, MySQL, Docker, Kubernetes, and Google K8S Engine. #java #SpringCloud #SpringBoot #Docker #Kubernetes #GKE #Cloud #Microservices #SoftwareDeveloper

I am really excited about Microservices. I want to thank @crichardson for his awesome tech talks related to Data Consistency in Microservices and the SAGA pattern on YouTube. I learned so many advance topics from his talks.

I was hesitant about writing this thread but thanks to @kunalstwt for inspiring me to show my work to people and gain more insights from the industry.
