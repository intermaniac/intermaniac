Ahead-of-time (AOT) compilation is a method of compiling a program into machine code that is executed directly by the hardware, rather than being interpreted or run in a virtual machine.

So, when you execute `rustc https://t.co/IbWgnS45bQ` in the command line, it directly produces machine executable file as opposed to the file which runs on virtual machines.

Just-in-time (JIT) compilation is a method of compiling a program into machine code as it is being executed, rather than ahead-of-time (AOT).

So, when you execute `javac https://t.co/VmPTYAIuhQ` in the command line, it produced bytecode which doesn't run directly into machine but is supposed to be run under virtual machine and virtual machine is responsible for converting bytecode to machine code.

As solidity is following same procedure to compile and execute like Java and uses virtual machine, my assumption and research says that it is also JIT language. Pls correct me if I am wrong @leonardoalt @_hrkrshnn

This leads to the following question. Why do we say that `solc` compiles the solidity file? Shouldn't we be saying that it converts the solidity file to bytecode which can be ran into EVM because it is the EVM who is converting bytecode to machine executable and compiling it.
