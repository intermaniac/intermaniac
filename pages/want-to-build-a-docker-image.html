Want to build a Docker image inside a Kubernetes pod or Docker container? This open-source utility can create container images without compromising security. I am talking about Project Kaniko. Let's learn more ðŸ§µ

kaniko is an OS container image-building tool created by Google. Before understanding kaniko, here is why you should avoid docker dependency in building images. (2/2) ðŸ‘‡

- The Docker build containers runs in privileged mode. It is a big security concern, and it is an open door to malicious attacks. (3/3) ðŸ‘‡

- Also, Kubernetes removed Docker from its core. So, mounting docker.sock to host will not work in the future unless you add docker host to all the Kubernetes Nodes. Kaniko resolved these issues. (4/4) ðŸ‘‡

Here is how Kaniko works, 1. There is a dedicated Kaniko executer image. It is responsible for building container images. 2. kaniko accepts three arguments. A Dockerfile, build context, and a remote registry to push the build image. (5/5) ðŸ‘‡

3. When you deploy the kaniko image, it reads the Dockerfile and extracts the base image file system using the FROM instruction. 4. Then it executes each instruction from the Dockerfile and takes a snapshot in the userspace. (6/6) ðŸ‘‡

5. After each snapshot, kaniko appends only the changed image layers to the base image and updates the image metadata. It happens for all the instructions in the Dockerfile. 6. Finally, it pushes the image to the given registry. (7/7) ðŸ‘‡
