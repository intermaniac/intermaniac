Some advanced Git commands :- ðŸ§µ

1. git log Shows a set of history of all the commits that were recently made in the active branch.

2. git reset <log-hash-code> Undoes the changes that were made after that particular log.

3. git restore --staged <filename> Resets the added/staged file to unstaged. (checked using 'git status')

4. git stash Takes the recently added files to stash area. Stash is a stacking place where any file changes can be kept and can be called later when required. This is used when you dont want to commit a change and also dont want to lose it.

5. git stash pop Calls back all the changes from stash back-area. Now they are again ready to be added/staged.

6. git stash list List all the stash files.

7. git stash clear Deletes all the stash files.

8. git stash drop Deletes the top/recent stash file.

9. git stash drop <stash-reference> Delete the specific stash file. the reference goes like stash@{1}. The reference can be obtained from 'git stash list' command.

10. git push <alias> <branch> -f Force push the changes from local to remote repository. This is used in case if there are some deletions of commits in local repository which are not updated in remote repository's commit history.

11. git fetch --all --prune Gets all the changes from upstream branch (primary branch/original branch from forked). Prune is used to update even the ones that were deleted.

12. git reset --hard upstream/main Resets your origin branch in local repository to the one in the upstream. This fetches all the updates that were made in the upstream branch and the local repository is up-to-date.

13. git push <alias> <branch> Pushes all the updates from local repository to the remote repository. Required to make forked repository even/equal with the original one.

14. git pull upstream <branch-name> gets all the updated from upstream branch to the local repository. (11+12) Shortcut

15. git rebase -i <commit hash-id> Opens an interactive file to edit from where we can merge multiple commits into one. Here, 'pick' means take it as the main commit. In place of 'pick', one can write 's'/'squash' which means merge these squash commits into the above picked one.
