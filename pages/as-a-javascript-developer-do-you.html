As a JavaScript Developer, do you understand the terms ‘transpiler’ and ‘polyfill’? If not, here’s a 🧵 for you-

First of all, we need to understand the difference between backwards compatibility and forwards compatibility. To be backwards compatible means that any future updates to the language should not make the old syntax invalid, e.g., ES2019 still doesn’t make ES2015 invalid. 👇🏻

Whereas, forwards compatibility is a little different. It encourages the idea that including a new addition to the language in a program should not cause it to break if it were run on an older JS engine. 👇🏻

JS is backwards compatible, not forwards compatible. For example, we can still use ES6 syntax due to backwards compatibility of the language; however, if we run an ES6 program on a JS engine older than 2015, the program will most likely break. 👇🏻

But we still need to support older browsers to run our ‘modern’ JavaScript code, for which, we use a JavaScript transpiler such as Babel. A transpiler converts the newer version of JavaScript code into an equivalent older syntax. 👇🏻

If, however, the issue is not related to a newer syntax but a recently added API, we can write a custom definition of that API in a backwards compatible form of code. That custom definition can then be included in the program to be executed. This is what a polyfill is.

So that is it for today! I hope it helped you, and if it did, then share this thread to others!
