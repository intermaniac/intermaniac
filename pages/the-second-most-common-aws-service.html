The second most common AWS service to invoke Lambda functions is SQS Learn the fundamentals, quirks and solutions 🧵 👇

𝗪𝗵𝘆 𝘄𝗼𝘂𝗹𝗱 𝘆𝗼𝘂 𝗦𝗤𝗦 𝘄𝗶𝘁𝗵 𝗟𝗮𝗺𝗯𝗱𝗮? SQS queue acts as intermediary between producer(Service A) and consumer (B) So, when consumer goes down - your messages will not get lost. When consumer comes back up again - it will pick all those messages in the queue https://t.co/0Mpkd2NBDI

𝗢𝗿𝗱𝗲𝗿 𝗼𝗳 𝗺𝗲𝘀𝘀𝗮𝗴𝗲𝘀: If you're using standard sqs queue, the order of messages cannot be guaranteed. If you're sending messages 1,2,3 - you may be getting as 1,3,2 or some other order https://t.co/QXI1rdrdts

When `Service A` is producing lot of messages - then, lambda service(not your lambda function) will scale out `Service B` by making many replicas for `Service B` - many lambdas would be processing the messages, in our case https://t.co/7xQ4jC3FIM

This begs the question - if there are many replicas of `Service B`. 𝗪𝗼𝘂𝗹𝗱 𝗲𝗮𝗰𝗵 𝗿𝗲𝗽𝗹𝗶𝗰𝗮 𝗽𝗶𝗰𝗸 𝘁𝗵𝗲 𝘀𝗮𝗺𝗲 𝗺𝗲𝘀𝘀𝗮𝗴𝗲 𝘁𝗼 𝗽𝗿𝗼𝗰𝗲𝘀𝘀? `visibilityTimeout` property determines how long a message becomes invisible when it is delivered to a consumer.

If a consumer picks a message, the message would be invisible for specified number of seconds - as dictated by `visibilityTimeout` property. During this period all the other consumers would not be able to see that message (even though the message still exists in the queue)

How the consumer lambda receives message from SQS queue? When SQS is used with lambda, lambda service (not your lambda function) will poll SQS queue for every few seconds and if queue contains message - the message is delivered as part of event to the consuming lambda.

𝗛𝗼𝘄 𝗳𝗿𝗲𝗾𝘂𝗲𝗻𝘁𝗹𝘆 𝗱𝗼𝗲𝘀 𝘁𝗵𝗲 𝗹𝗮𝗺𝗯𝗱𝗮 𝗽𝗼𝗹𝗹 𝘁𝗵𝗲 𝗾𝘂𝗲𝘂𝗲? There are 2 types of polling - short polling and long polling.

In short polling, it will poll only subset of servers which may result in false positives - messages will be there in the queue but you might get empty response. It is always recommended to use long polling.

In long polling - you can specify the frequency (in terms number of seconds) with a maximum value of 20 seconds. For example, if you use long polling of 20 seconds, lambda service will poll your queue (all the servers for the queue) every 20 seconds to check for messages.

If you're using CDK, `receiveMessageWaitTime` property determines the polling frequency. If the value of this property is greater than 0 seconds, it is considered as long polling.

𝗪𝗵𝗼 𝗽𝗮𝘆𝘀 𝗳𝗼𝗿 𝘁𝗵𝗶𝘀 𝗹𝗮𝗺𝗯𝗱𝗮 𝗽𝗼𝗹𝗹𝗶𝗻𝗴 𝗶𝗻 𝗦𝗤𝗦 𝗾𝘂𝗲𝘂𝗲? You'll not be charged for polling . However, it is not free. Since polling involves calling SQS APIs, you'll be charged for calling SQS APIs but it is pretty cheap.

𝗛𝗼𝘄 𝗺𝗮𝗻𝘆 𝗺𝗲𝘀𝘀𝗮𝗴𝗲𝘀 𝗱𝗼𝗲𝘀 𝗹𝗮𝗺𝗯𝗱𝗮 𝗽𝗼𝗹𝗹 𝗶𝗻 𝘆𝗼𝘂𝗿 𝗾𝘂𝗲𝘂𝗲? Lambda service will poll your queue in batches and each batch will contain specific number of messages as mentioned by the batch size (configurable). Default value is 10 messages. https://t.co/096kRu5fDK

𝗗𝗲𝗹𝗲𝘁𝗶𝗼𝗻 𝗼𝗳 𝗺𝗲𝘀𝘀𝗮𝗴𝗲𝘀 Another advantage of using Lambda with SQS is that as soon as the messages are successfully processed, the messages in that batch would be automatically deleted from the queue. You don't need to do this manually.
