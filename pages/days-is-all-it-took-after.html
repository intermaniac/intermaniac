Linus never wanted to work on SCM tools and found them uninteresting, but BitKeeper changed his perspective. He knew that the biggest advantage of distributed SCM is that it avoids politics around who can make changes.

However, they found that BitKeeper had its own problems, including technical choices and the fact that it wasn't open source. These problems came to a head when someone started reverse engineering the BitKeeper protocol, and Linus was unable to mediate between the two parties.

As a result, Linus decided to write his own SCM tool to use to develop Linux.

It took about a day to have git to be "self-hosting", so Linux could start committing things into git using git itself. After we can look at the commit history of git itself to see how the rest unfolded.

He worked mostly during the day, so it wasn't like he stayed up all weekend to write it. The most interesting part is how quickly the fundamental git concepts came together.
