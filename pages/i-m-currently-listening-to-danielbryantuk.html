Developer experience comprises of the "Inner dev loop" and the "outer dev loop". The inner dev loop is where the individual developer writes the code for the new feature or bug fix and then goes further to test the impact of their changes, refactor if necessary and retest again. https://t.co/cQ3JZZRj6C

While the outer dev loop is everything else that happens leading up to release. This includes things like code merge, canary release, observation of results, etc. https://t.co/x5XVtAe2Vy

Adopting cloud-native technologies has altered the developer experience. Developers now have to perform more steps in the inner dev loop, which makes the feedback loop slow. https://t.co/dFJQSs8w6Q

There are a couple of ways to resolve this; running everything locally or running everything remotely. But they both come with their own challenges. Utilizing the removal approach means enjoying the pros of the local and remote instead of using only one route. https://t.co/hocueL2XnS

Telepresence is one of the extensions available on Docker Desktop. It speeds up your inner dev loop by allowing you to rapidly develop & test Kubernetes services. You can use hot reload, your IDE, & run integration tests locally instead of waiting on a remote deployment. https://t.co/y3flBMzFRK
