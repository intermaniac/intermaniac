Its @AWS #reinvent week and I am feeling serious FOMO by not being there. But I have been pretty excited by the new security releases. Let's dive in. A ðŸ§µ

I am a big AuthZ and Access Control proponent and there's a lot of good that has happened in that space offlate. Managed Access Control services and @OpenPolicyAgent are some examples. I am stoked about AWS getting into the Access-Control-as-a-Service Game for 2 reasons

1. Keeping AuthZ and Access Control close to apps is always useful 2. Integration of AuthZ and Admission Control into API Gateway, etc provides awesome scale I am stoked for AWS Verified Permissions. Looking forward to working with it https://t.co/C8wGt5Zf9x

The new feature where AWS Inspector scans #lambda for flaws sounds great and a much needed feature. I need to test it out, but I am looking for the following capabilities: 1. Source code scanning for security issues 2. IAM Analysis and Recommendations 3. Lambda Layer Scanning

4. Source Composition Analysis I personally intend to use this capability a lot and use Eventbridge to automate the crap out of it. Hopefully it lives up! https://t.co/MhuLydzrUq

Security Analytics is a big need. Especially for multi-account AWS orgs. It also consumes a great deal of time to build, automate and maintain. Which is why I am hopeful for AWS Security Lake. AWS Security Lake is a service that centralizes security data from $everywhere

What is $everywhere? - AWS Services like VPC, Cloudtrail, GuardDuty, Security Hub - On-Prem Services - Custom sources and integrations using the OCSF (Open Cybersecurity Schema Framework) Data is gathered into S3 buckets and normalized into OCSF for querying and analysis

What if a VPC and Service Mesh had a baby? I think that's what VPC Lattice is all about. VPC lattice is a new service that adds a whole bunch of service-to-service comm capability incl: service discovery, advanced traffic management, security & workload authN & AuthZ

I think this will be especially useful for orgs running multiple types of workloads (containers, faas and VMs) in VPCs. I am intrigued by the Contextual AuthN/AuthZ part of this product and look forward to digging in https://t.co/TMaoqA7OE3

Needless to say, we @AppSecEngineer have immediately started to build labs and content around some of these new developments. They impact AWS security and give you more options from a security standpoint. You should see some of this appear in labs and courses very soon
