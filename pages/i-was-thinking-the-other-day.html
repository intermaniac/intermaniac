1/ I was thinking the other day, AWS Simple Email Service is a great tool to send out any kind of email for your web app, but there must be a simpler way to use it with an @AWSAmplify project? And so I built the @sendEmail graphQL directive! Let me tell you all about it! üßµ https://t.co/eXR1ut0jZi

2/ You can now send out emails by adding a simple @sendEmail directive to your graqhQL model definition. What that directive does behind the scenes is deploy a lambda function and attach it to the event stream from that models db table. https://t.co/XrsrQEce0Z

3/ In other words when a model is created, updated or deleted, the lambda fires and sends out an email with the template you defined in the directive. The {{placeholder}} fields are filled from the event data. https://t.co/9t0jLNyX2W

4/ Just remember to set up your SES beforehand, by validating the sender email, and if you are in SES sandbox mode also verifying the recipient email. https://t.co/o7SUpt7R8W

5/ There are many more features that could be added in the future! Like loading templates from a URL, adding array of data to the template, and filtering db events by field change as well as mutation type. Find more installation instructions on Github: https://t.co/drxHkixTHd
